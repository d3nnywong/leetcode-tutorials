二叉树最经典的入门递归题。看完你会明白：「深度」其实就是一个会自己往下问、 再把答案一路带回来的递归定义。

🧩 第一步：「深度」到底是什么

给你一棵二叉树，求它的最大深度——也就是从根节点走到 最远的叶子节点，这条路径上一共经过几个节点。

root[3,9,20,null,null,15,7]

从根节点 3 走到叶子 15（或 7）：3 → 20 → 15，一路经过 3 个节点， 所以最大深度是 3。

💡 第二步：把「深度」拆成更小的「深度」

直接去数「最长路径」不好下手，但换个角度看：一棵树的深度，其实可以用它 左右两棵子树的深度推出来——

  • 空节点（树都没有）：深度规定为 0。
  • 非空节点：它的深度 = 左子树深度和右子树深度里 更大的那个，再加上自己这一层，即 depth = 1 + max(左, 右)。
这就是递归：要算「整棵树」的深度，先去算「左子树」和「右子树」各自的深度—— 而左、右子树本身又是更小的树，可以用同一套逻辑继续往下拆，一直拆到空节点（深度 0）为止， 再把答案一层层带回来。

⚡ 第三步：看递归怎么「下去再回来」

后序遍历天然契合这个过程：先递归处理完左子树、再递归处理完右子树， 最后才轮到自己用 1 + max(左, 右) 算出答案，交还给上一层调用它的节点。

还在递归栈里等结果的祖先 正在处理 / 刚算完的节点 已经算出深度的节点 根节点的最终答案

每个节点上方会冒出它算出来的深度；点「下一步」看深度怎么从叶子一路汇总回根节点。

☕ 第四步：写成 Java 代码

// LeetCode 已经定义好 TreeNode：
// class TreeNode { int val; TreeNode left; TreeNode right; }
class Solution {
    public int maxDepth(TreeNode root) {
        if (root == null) {
            return 0; // 空节点，深度为 0
        }
        int leftDepth = maxDepth(root.left);   // 左子树的最大深度
        int rightDepth = maxDepth(root.right); // 右子树的最大深度
        return 1 + Math.max(leftDepth, rightDepth); // 较深的子树 + 自己这一层
    }
}
时间复杂度 O(n)

每个节点恰好被访问一次（n 是节点总数）。

空间复杂度 O(h)

h 是树的高度，来自递归调用栈：平衡树约 O(log n)， 最坏（退化成链）O(n)。

🚀 小结

套路：树相关的问题，很多都可以问自己一句——「能不能用『左子树的答案』 和『右子树的答案』，推出『整棵树的答案』？」能的话，递归（分治）几乎就是现成的解法： base case 处理空节点，递归式负责合并左右两边的结果。最大深度是这个套路最干净的入门例子。
