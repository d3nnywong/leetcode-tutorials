二叉树最经典的入门递归题。看完你会明白：「深度」其实就是一个会自己往下问、 再把答案一路带回来的递归定义。

🧩 第一步：先搞懂「二叉树」和「深度」是什么

题目给你一棵二叉树，没接触过也没关系——把它想象成一张倒着画的家谱图： 最顶上那个人是根节点（root），每个节点（就是树上装着一个数字的 小方框）最多能往下「生」出两个孩子：画在左边的叫左子节点，画在右边的叫 右子节点。一直往下传，直到某个节点没有孩子了，这种「断了后」的节点就叫 叶子节点（leaf）。

题目要求的最大深度，就是从根节点出发，一路走到最远的那个叶子节点， 这一趟路（包括起点和终点）一共经过几个节点。

root[3,9,20,null,null,15,7]

这里的 null 表示「这个位置没有孩子，是空的」——LeetCode 常用这种数组写法来表示树： 按层从上到下、从左到右列出每个节点，没孩子的地方就写 null。

从根节点 3 走到叶子 15（或 7）：3 → 20 → 15，一路经过 3 个节点， 所以最大深度是 3。

💡 第二步：把「深度」拆成更小的「深度」

直接去数「最长路径」不好下手，但换个角度看：一棵树的深度，其实可以用它 左右两棵子树的深度推出来——这里的「子树」是指：把某个节点当成新的根， 连同它下面挂着的所有节点，单独拎出来看，就是一棵「缩小版」的树。

  • 空节点（这个位置没有孩子，什么都没有）：深度规定为 0。
  • 非空节点：它的深度 = 左子树深度和右子树深度里 更大的那个，再加上自己这一层，即 depth = 1 + max(左, 右)。
这就是递归：所谓递归，就是一个函数自己调用自己， 去解决一个「更小的同类问题」——有点像照镜子时镜子里还套着一面镜子， 或者拆开一个俄罗斯套娃，里面永远是同样造型、更小一号的娃娃。 要算「整棵树」的深度，先去问「左子树」和「右子树」各自的深度—— 而左、右子树本身又是更小的树，可以用同一套问法继续往下拆，一直拆到空节点（深度 0）为止， 再把答案一层层带回来，汇总成整棵树的深度。

⚡ 第三步：看递归怎么「下去再回来」

我们按「先处理左边、再处理右边、最后才轮到自己」的顺序走一遍所有节点——这种走法有个名字， 叫后序遍历。具体来说：先递归处理完左子树、再递归处理完右子树， 最后才轮到自己用 1 + max(左, 右) 算出答案，交还给上一层调用它的节点。

还没算完、正在等孩子返回结果的上层节点 正在处理 / 刚算完的节点 已经算出深度的节点 根节点的最终答案

每个节点上方会冒出它算出来的深度；点「下一步」看深度怎么从叶子一路汇总回根节点。

☕ 第四步：写成 Java 代码

// LeetCode 已经定义好 TreeNode：
// class TreeNode { int val; TreeNode left; TreeNode right; }
class Solution {
    public int maxDepth(TreeNode root) {
        if (root == null) {
            return 0; // 空节点，深度为 0
        }
        int leftDepth = maxDepth(root.left);   // 左子树的最大深度
        int rightDepth = maxDepth(root.right); // 右子树的最大深度
        return 1 + Math.max(leftDepth, rightDepth); // 较深的子树 + 自己这一层
    }
}
时间复杂度 O(n)

每个节点恰好被访问一次（n 是节点总数）。

空间复杂度 O(h)

h 是树的高度，来自递归调用栈——每往下递归一层，就像在一摞盘子上 再摞一个新盘子，记着「这一层还在等下面算完」；等下面返回结果了，这个盘子才被拿走。 树越深，这摞盘子就摞得越高。平衡树约 O(log n)， 最坏情况（退化成一条链）O(n)。

🚀 小结

套路：树相关的问题，很多都可以问自己一句——「能不能用『左子树的答案』 和『右子树的答案』，推出『整棵树的答案』？」能的话，递归（这种「先拆成小问题分别解决、 再把结果合并起来」的思路，也叫分治）几乎就是现成的解法： 先写好「最简单、拆不动的情况」（这里就是空节点，直接返回 0）， 再写「怎么把左右两边的结果合并成当前节点的答案」。最大深度是这个套路最干净的入门例子。
