一串只含数字的密文，可能藏着好几种不同的解码方式。这道题不是要你找出某一种解码， 而是要你数清楚——一共有多少种。一维 DP 里「同一格可以由两条路径凑出来」的经典写法。

🧩 第一步：题目到底要什么

字母 A~Z 分别对应数字 1~26（"1"→'A' … "26"→'Z'）。 给你一串数字 s，你要把它切成一段一段，每段是 1 位或 2 位， 每段对应的数字都要在 1~26 之间，这样就能还原出一串字母。

因为切法可能不止一种（比如 "2" 和 "5" 既能分开算，也能合成 "25" 一起算），题目要你算出总共有多少种合法的切法， 而不是具体切成什么样。如果一种合法切法都没有，返回 0。

s"226"

可以切成 2 | 2 | 6（BBF）、22 | 6（VF）、2 | 26（BZ）， 一共 3 种合法切法 → 答案是 3。

坑点：两位数必须落在 10~26 之间才合法。 像 "06" 这种「前导零」的两位数是不合法的（因为它和单独的 "6" 意义重复，规则上不允许），所以单独一个 "0" 也不能解码成任何字母。

🐢 第二步：暴力递归 + 它的毛病

最直接的想法：从字符串开头出发，每一步「要么切 1 位、要么切 2 位」，递归地把剩下的部分接着切， 数一数能切到底（用完整个字符串）的方案有多少种。

慢在哪：每一步都在「切 1 位」和「切 2 位」之间分叉，最坏情况下方案数会 指数级增长（很像爬楼梯每次走 1 步或 2 步）。但仔细看会发现：递归到「剩下后面这一段字符串」时， 很多分支其实是同一个子问题——子问题只跟「还剩多少个字符要切」有关，被反复算了无数次。

关键洞察：设 dp[i] 表示「前 i 个字符一共有多少种合法切法」， 那么 dp[i] 只可能来自两个「最后一步」：

  • 最后一步切了 1 位（第 i 位单独解码）——要求这一位不是 '0'，能贡献 dp[i-1] 种方案；
  • 最后一步切了 2 位（第 i-1、i 位合在一起解码）——要求这两位组成的两位数在 10~26 之间，能贡献 dp[i-2] 种方案。

两条路互不冲突，能走的都要加上：dp[i] = dp[i-1]（若成立） + dp[i-2]（若成立）。 从左往右把 dp[0..n] 都算一遍，重复子问题只算一次，整个表 O(n) 就填完了。

⚡ 第三步：一维 DP 表逐格点亮

下面这条 dp 数组落在字符串的「缝隙」上：dp0 在最前面（空串）， dpi 在第 i 个字符的右边。算 dp[i] 时，看它能不能从左边 1 格 （dp[i-1]，单字符）或 2 格（dp[i-2]，两字符）转移过来。

正在算的 dp[i] 单字符来源 dp[i-1] 两字符来源 dp[i-2] dp = 0（死路）

☕ 第四步：写成 Java 代码

class Solution {
    public int numDecodings(String s) {
        int n = s.length();
        // dp[i]：s 的前 i 个字符一共有多少种合法解码方式
        int[] dp = new int[n + 1];
        dp[0] = 1; // 空串只有「什么都不解码」这一种方式

        for (int i = 1; i <= n; i++) {
            // 把第 i 位单独解码成一个字母：要求不是 '0'
            if (s.charAt(i - 1) != '0') {
                dp[i] += dp[i - 1];
            }
            // 把第 i-1、i 位合在一起解码成一个字母：要求两位数在 10~26 之间
            if (i >= 2) {
                int twoDigit = (s.charAt(i - 2) - '0') * 10
                    + (s.charAt(i - 1) - '0');
                if (twoDigit >= 10 && twoDigit <= 26) {
                    dp[i] += dp[i - 2];
                }
            }
        }
        return dp[n];
    }
}
时间复杂度 O(n)

从左到右把 dp[0..n] 各算一次，每格只做常数次判断。

空间复杂度 O(n)

用了长度 n+1 的 dp 数组；因为每格只依赖前两格，也能压缩成 O(1) 的两个变量。

🚀 小结

套路：当「当前结果」可以由有限几种「最后一步」转移过来时 （这题是「切 1 位」或「切 2 位」），就设 dp[i] 表示「前 i 个能凑出多少种合法方案」， 把每种「最后一步」能贡献的方案数加起来。这和爬楼梯（每次走 1 阶或 2 阶）几乎是同一个模型—— 只是这里多了一条「这一步合不合法」的校验。
