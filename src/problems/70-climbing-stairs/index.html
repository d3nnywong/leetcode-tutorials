全网最经典的动态规划入门题。看完你会发现：它换了个壳，其实就是小学就听说过的 「斐波那契数列」——而动态规划，不过是把递归里重复算的部分「记下来」而已。

🧩 第一步：题目到底要什么

有一架 n 阶的楼梯，你每次只能往上迈 1 阶或 2 阶。问：一共有多少种不同的「迈法组合」能正好走到第 n 阶？

n3

三种走法：1+1+1、1+2、2+1 → 返回 3。

🐢 第二步：先想最后一步，而不是第一步

直接从第一步往后想，分支会越想越乱。换个角度—— 盯着终点「第 n 阶」往回看：不管前面怎么走， 走到第 n 阶的最后一步只可能有两种情况：

  • 最后一步迈了 1 阶：说明在那之前，你已经站在第 n−1 阶。
  • 最后一步迈了 2 阶：说明在那之前，你已经站在第 n−2 阶。

这两种情况互不重叠（最后一步要么迈 1 要么迈 2，不会同时是）， 所以「走到第 n 阶的方法数」就是「走到第 n−1 阶的方法数」加上「走到第 n−2 阶的方法数」。

如果直接写递归：f(n) = f(n-1) + f(n-2) 能算对， 但 f(n-2) 这种子问题会在不同分支里被反复重新计算， n 一大，递归树指数级爆炸。动态规划要做的，就是把每个子问题只算一次、记下来。

🧠 第三步：动态规划三件套

1

状态：dp[i] 表示什么？

dp[i] = 爬到第 i 阶一共有多少种不同的方法。

最终答案就是 dp[n]。

2

转移：dp[i] 怎么由更小的问题推出来？

照上面的洞察，最后一步要么从第 i−1 阶迈 1 阶上来，要么从第 i−2 阶迈 2 阶上来：

dp[i] = dp[i-1] + dp[i-2]（i ≥ 3）

3

边界：最小的情况是什么？

dp[1] = 1（只有「迈 1 阶」这一种走法）； dp[2] = 2（「1+1」或「直接迈 2 阶」）。 这两个地基定下来，后面全靠它们往上叠。

下面这个动画一阶一阶往上算：当前在算的台阶会高亮，它依赖的「前一阶」和「前两阶」 会被标成来源色，算完把方法数填到台阶上方。

当前正在算的 dp[i] 依赖的 dp[i-1] 与 dp[i-2] 最终答案 dp[n]

试试把 n 改成 1，看「地基」单独一步是怎么直接给出答案的； 再改成 8、9，看台阶怎么越爬越高、方法数怎么像滚雪球一样变大。

☕ 第四步：写成 Java 代码

动画里我们把每一阶的方法数都存了下来，但仔细看转移方程会发现： 算 dp[i] 时只用到了最近的两个值 —— 没必要存下整张表，用两个变量滚动更新就够了，空间从 O(n) 降到 O(1)。 

class Solution {
    public int climbStairs(int n) {
        if (n <= 2) return n; // dp[1]=1, dp[2]=2，和 n 本身正好相等

        int prev2 = 1; // 滚动变量：dp[i-2]，初始对应 dp[1]
        int prev1 = 2; // 滚动变量：dp[i-1]，初始对应 dp[2]

        for (int i = 3; i <= n; i++) {
            int cur = prev1 + prev2; // dp[i] = dp[i-1] + dp[i-2]
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
}
时间复杂度 O(n)

从第 3 阶算到第 n 阶，循环一遍。

空间复杂度 O(1)

只保留最近两个值，不需要整张 dp 表。

🚀 小结

套路：爬楼梯本质就是斐波那契数列（移了一位）。它的价值不在题目本身， 而在于把「动态规划三件套」——状态、转移、边界——讲得最干净的一道题： 先盯着最后一步把大问题拆成小问题，再发现「重复子问题」该被记下来而不是重算， 最后观察到「只依赖最近几个状态」时，还能把空间从 O(n) 压到 O(1)。 这套思路是后面所有一维 DP 题（打家劫舍、解码方法……）共用的起点。
