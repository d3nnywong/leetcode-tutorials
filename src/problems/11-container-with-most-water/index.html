这题是「双指针」最漂亮的一课。难点不在写代码，而在想通一件事： 为什么每次只把矮的那根指针往里挪，就一定不会错过答案。

🧩 第一步：先把「面积」想清楚

每根竖线是一块挡板，任选两根当容器的左右壁，水能装多高由矮的那块决定 （高的那块上面会漏）。所以：

面积= min(左高, 右高) × (右下标 − 左下标)

一个是「矮边的高度」，一个是「两根之间的宽度」，两者相乘。

🐢 第二步：笨办法与它的浪费

枚举所有两根线的组合算面积取最大，O(n²)。n 能到 10⁵，平方就是 10¹⁰，太慢。

能不能更聪明？先把两根线放在最左和最右—— 此时宽度最大。接下来只要想：怎么挪，才有机会换来更大的面积。

⚡ 第三步：双指针，只移矮的那根

左右指针从两端开始。当前面积被矮边卡死，宽度又已经最大。这时：

  • 移高的那根？宽度变小，矮边还是那个矮边 → 面积只会更小，没意义。
  • 移矮的那根？宽度也变小，但矮边有可能换成更高的 → 才有机会变大。

所以每一步都把矮的指针往里移，扫一遍就够，O(n)。

当前左右指针 L / R 最终最大面积的两条线

蓝色阴影就是当前容器装的水；点「下一步」看矮指针怎么一步步往里挪、面积怎么刷新。

☕ 第四步：写成 Java 代码

class Solution {
    public int maxArea(int[] height) {
        int l = 0, r = height.length - 1, max = 0;
        while (l < r) {
            int area = Math.min(height[l], height[r]) * (r - l);
            max = Math.max(max, area);
            // 移动矮的那根，才有机会换来更高的矮边
            if (height[l] < height[r]) {
                l++;
            } else {
                r--;
            }
        }
        return max;
    }
}
时间复杂度 O(n)

左右指针总共向中间走 n 步。

空间复杂度 O(1)

只用了几个变量。

🚀 小结

双指针的核心：每一步都能确定地排除一个不可能更优的选择 （这里是「移矮指针、丢掉的组合一定不会更大」），于是 O(n²) 的搜索塌缩成一条 O(n) 的路径。 很多「首尾向中间逼近」的题都靠这个论证。
