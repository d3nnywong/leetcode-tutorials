一个字母网格、一个目标单词：能不能像走迷宫一样，沿着相邻格子一步步拼出它？ 这道题是「网格上的 DFS + 回溯」最经典的入门题——走错了就退回去，换条路再试。

🧩 第一步：题目到底要什么

给你一张 m x n 的字母网格 board，和一个单词 word。 问：能不能从网格里的某一格出发，每一步只走到水平或垂直相邻的格子， 依次拼出 word 的每一个字母？同一个格子在这一趟路径里不能重复走。

board[["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]]
word"SEE"

从下标 (1,3) 的 S 出发，往上走到 (0,3) 的 E 是个死路（再往下走不到下一个 E， 而且 (1,3) 已经用过了），退回来改成往下走到 (2,3) 的 E， 再往左走到 (2,2) 的 E —— 正好拼出 "SEE"，返回 true。

🔑 第二步：为什么是 DFS + 回溯，而不是简单遍历

单纯遍历网格找第一个字母没用——找到 word[0] 之后，还要在它的邻居里继续找 word[1]，再在 word[1] 的邻居里找 word[2]…… 这是一条沿着相邻关系往前探索的路径，天然适合用深度优先搜索（DFS）： 一条路走到黑，走不通就退回上一步换路。

走不通有三种情况：① 越界了；② 这个格子在当前路径里已经用过了；③ 这个格子的字母对不上 word 的下一个字符。只要踩中任何一种，这条分支就要回溯—— 把刚才标记的「已用」状态撤销，退回上一格，换下一个方向继续试。

「标记 + 撤销」是回溯的灵魂：进入一个格子时标记它「已用」（防止本趟路径重复走）， 这条分支失败、要退回去尝试别的可能性时，必须把标记撤销回去—— 因为同一个格子在另一条候选路径里完全可以重新被用到。忘记撤销，是这道题最常见的 bug。

⚡ 第三步：看 DFS 怎么走，又是怎么回溯的

从网格里每一个等于 word[0] 的格子出发，按「上、下、左、右」固定顺序尝试相邻格： 能接上 word 的下一个字符就走过去，接不上 / 越界 / 已用过就跳过； 四个方向都试过仍然走不通，就回溯到上一格，换其他方向（或者换下一个起点）继续。

当前正在尝试的格子 已经走过、留在路径里 刚被回溯撤销的格子 已匹配上的字母

☕ 第四步：写成 Java 代码

class Solution {
    private int m, n;
    private char[][] board;
    private String word;
    private boolean[][] visited;
    private static final int[][] DIRS = {{-1,0},{1,0},{0,-1},{0,1}};

    public boolean exist(char[][] board, String word) {
        this.board = board;
        this.word = word;
        m = board.length;
        n = board[0].length;
        visited = new boolean[m][n];
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // 只有这一格的字母等于 word[0]，才值得从它开始尝试
                if (board[i][j] == word.charAt(0) && dfs(i, j, 0)) {
                    return true;
                }
            }
        }
        return false;
    }

    // dfs(i, j, k)：当前走到 (i, j)，它已经匹配了 word 的第 k 个字符
    private boolean dfs(int i, int j, int k) {
        if (k == word.length() - 1) {
            return true; // 连最后一个字符都匹配上了，整个单词拼成功
        }
        visited[i][j] = true; // 标记：这一格在当前路径里已经用过
        for (int[] d : DIRS) {
            int ni = i + d[0], nj = j + d[1];
            if (ni < 0 || ni >= m || nj < 0 || nj >= n) continue; // 越界
            if (visited[ni][nj]) continue; // 这趟路径已经用过
            if (board[ni][nj] != word.charAt(k + 1)) continue; // 字母对不上
            if (dfs(ni, nj, k + 1)) {
                return true; // 这条分支成功了，一路把 true 传回去
            }
        }
        visited[i][j] = false; // 回溯：四个方向都走不通，撤销标记，退回上一格
        return false;
    }
}
时间复杂度 O(m·n·3^L)

m·n 个起点，每个起点最坏要往 L = word.length 层深搜； 除了来路方向，每层最多还有 3 个方向可选，所以是 3 的 L 次方量级。

空间复杂度 O(m·n + L)

visited 网格占 O(m·n)，递归栈深度最多到单词长度 L。

🚀 小结

套路：在网格 / 树 / 图上「沿着某种相邻关系，一步步拼出一条满足条件的路径」， 就是 DFS + 回溯的典型场景。核心永远是那一对动作——进入一个状态前先标记， 这条分支彻底失败、要退回去时一定要撤销标记。 掌握了这一对「标记 / 撤销」，后面遇到的「岛屿数量」「全排列」「N 皇后」都是同一个骨架的变体。
