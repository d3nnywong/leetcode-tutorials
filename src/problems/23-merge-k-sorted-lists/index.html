k 条已经各自排好序的链表，怎么把它们拼成一条更长的有序链表？ 看完你会明白：一个小顶堆，怎么把「每次都要从 k 个候选里挑最小」这件事做到飞快。

🧩 第一步：题目到底要什么

给你 k 条链表，每一条自己内部都已经是升序的。 要求把它们全部合并成一条链表，合并后的结果也必须是升序的。

lists[[1,4,5],[1,3,4],[2,6]]

三条链表各自有序：1→4→5、1→3→4、2→6。把它们「拧」成一条： 1→1→2→3→4→4→5→6。注意里面可以有重复值（两个 1）。

如果只有两条链表，这就是经典的「合并两个有序链表」（每次比较两条链表的头节点， 小的接到结果末尾，往前走一格）。现在难点是 k 可以很多——怎么把这个思路扩展到 k 条？

🐢 第二步：直接的笨办法，慢在哪

最容易想到的：把 k 条链表两两合并——先合并链表 1 和链表 2， 结果再去和链表 3 合并，再和链表 4 合并……依次做 k − 1 次「合并两个有序链表」。

慢在哪：设所有节点总数是 N。每一次「合并两个有序链表」最坏要扫完当前已经合并出的那条 （越往后越长），k − 1 次合并下来，总代价是 O(N·k)。 k 一大（题目里能到 10⁴），就会很慢。

换个角度想：在任意时刻，我们其实只关心一件事——k 条链表「当前最前面那个还没用掉的节点」里， 谁的值最小。每次把这个最小的拿出来接到结果末尾，它所在的链表就往前挪一格， 换出一个新的候选节点。这正是小顶堆（优先队列）最擅长的事： 维护一堆候选值，随时能 O(log k) 地拿出最小的那个。

⚡ 第三步：用小顶堆做 k 路归并

先把每条链表的头节点都放进一个小顶堆（按节点值排序，堆顶永远是当前最小值）。 然后反复：从堆顶弹出最小的节点，接到结果链表末尾；如果它在原链表里还有下一个节点， 就把那个节点放进堆里继续候选。堆空了，说明所有节点都合并完了。

每个节点入堆、出堆各一次，堆里最多同时存在 k 个节点， 所以每次堆操作是 O(log k)，总共做 N 次，整体就是 O(N log k)——比两两合并的 O(N·k) 快得多。

堆中候选节点（各链表当前最前面的节点） 堆顶最小值（即将弹出） 已接入结果链表的节点

试着改成 5,6|1,2,9|3 或者带一条空链表 1,4,5|， 看看堆是怎么始终维护「每条链表当前候选」并挑出最小值的。

☕ 第四步：写成 Java 代码

// 单链表节点定义（LeetCode 已给出）
class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) return null;

        // 小顶堆：按节点值从小到大排，堆顶永远是当前最小候选
        PriorityQueue<ListNode> heap =
            new PriorityQueue<>((a, b) -> a.val - b.val);

        // 把每条链表的头节点放进堆（跳过空链表）
        for (ListNode head : lists) {
            if (head != null) heap.offer(head);
        }

        ListNode dummy = new ListNode(-1); // 哨兵节点，简化「结果链表为空」的边界
        ListNode tail = dummy;

        while (!heap.isEmpty()) {
            ListNode min = heap.poll();   // 弹出当前全局最小
            tail.next = min;                  // 接到结果链表末尾
            tail = tail.next;
            if (min.next != null) heap.offer(min.next); // 它所在链表还有下一个，补入堆
        }

        return dummy.next;
    }
}
时间复杂度 O(N log k)

N 是所有节点总数，k 是链表条数。每个节点恰好入堆、 出堆各一次，堆大小不超过 k，单次堆操作 O(log k)。

空间复杂度 O(k)

堆里最多同时存放 k 个节点；不算输出结果链表本身占用的空间。

🚀 小结

套路：当你要从「k 个有序来源」里不断挑出当前最小（或最大）的元素时， 不需要每次都把 k 个候选重新比一遍——用小顶堆把「挑最小」这件事从 O(k) 降到 O(log k)。这个套路不只适用于合并 k 个有序链表， 也是「外部排序」「Top K」「多路归并」这类问题的通用解法。
