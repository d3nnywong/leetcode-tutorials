一张无向图，什么时候才算是一棵「树」？这题是并查集（Union-Find）最经典的应用之一： 看完你会明白，判断「成不成树」其实只需要两个条件——边数刚刚好、并且不能有环。

🧩 第一步：题目到底要什么

给你 n 个节点（编号 0 到 n - 1）和一些 无向边 edges。问：把这些边连起来，能不能正好拼成 一棵树？

n5
edges[[0,1],[0,2],[0,3],[1,4]]

5 个节点、4 条边，把它们连起来发现没有任何「绕回去」的边，并且所有节点都连在一起 → 是一棵合法的树，返回 true。

回忆一下「树」的定义：连通（所有节点之间都能走到），并且无环 （任意两个节点之间只有唯一一条路径）。这道题就是要写代码去验证这两件事。

🔑 第二步：先用一个数字过滤掉大部分情况

一个很有用的图论事实：n 个节点的树，边数永远恰好是 n − 1。 直觉上很好理解——一棵树是「逐个节点用一条边接上去」长出来的，n 个节点要接 n − 1 次。

反过来想：
  • 边数 多于 n − 1 → 一定有节点被「连了不止一次」，必然出现环，肯定不是树。
  • 边数 少于 n − 1 → 边不够用，必然有节点连不进来，图肯定不连通，也不是树。
所以第一步直接检查 edges.length !== n - 1，不满足直接返回 false，省掉后面所有麻烦。

但边数对了不代表一定是树——比如 5 个节点、4 条边，也可能是「0-1-2 围成一个三角形， 3-4 单独连一条边」，边数刚好是 4，却既有环又不连通。所以还要真正检查有没有环， 这就要用到并查集（Union-Find）。

⚡ 第三步：用并查集一边合并一边判环

并查集维护「每个节点属于哪个连通分量」，每个分量有一个代表元素（根）。两个核心操作：

  • find(x)：顺着 parent 指针一路往上找，找到 x 所在分量的根。
  • union(a, b)：把 a、b 两个分量合并成一个（让一个根指向另一个根）。

逐条处理边 [a, b]：先 find(a)、find(b)。 如果两个根相同，说明 a、b 早就在同一个分量里了——这条边纯属多余， 连上去就会绕出一个环，直接判 false；如果根不同， 就把两个分量合并，继续看下一条边。

边数已经在第一步确认是 n − 1 条了，只要这 n − 1 条边 一条都没触发「同根」，就说明全程没有环，而且每条边都成功合并了两个分量—— n 个节点最少合并 n − 1 次才能并成 1 个分量，次数刚好够，图必然全部连通成一整块。

正在处理的边 / 端点 不同色圈 = 不同连通分量 触发环的边 / 端点 全部通过，判定为合法树

试试把 edges 改成 [[0,1],[1,2],[2,3],[1,3],[1,4]]（边比 n−1 多，秒判 false）； 或者 [[0,1],[1,2],[2,0],[3,4]]（边数对，但 0-1-2 围成一圈）， 看并查集在哪条边上抓到环。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean validTree(int n, int[][] edges) {
        // 树的边数必须恰好是 n - 1，不满足直接排除
        if (edges.length != n - 1) {
            return false;
        }

        int[] parent = new int[n];
        int[] rank = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i; // 一开始每个节点自成一个分量
        }

        for (int[] edge : edges) {
            int rootA = find(parent, edge[0]);
            int rootB = find(parent, edge[1]);
            if (rootA == rootB) {
                // 两个端点已经在同一分量里，这条边会围成环
                return false;
            }
            union(parent, rank, rootA, rootB);
        }

        // 边数恰好是 n - 1 又没有环，整张图必然全部连通
        return true;
    }

    private int find(int[] parent, int x) {
        while (parent[x] != x) {
            parent[x] = parent[parent[x]]; // 路径压缩，把查找树压扁
            x = parent[x];
        }
        return x;
    }

    private void union(int[] parent, int[] rank, int rootA, int rootB) {
        // 按秩合并：矮树挂到高树下面，避免树越长越高
        if (rank[rootA] < rank[rootB]) {
            parent[rootA] = rootB;
        } else if (rank[rootA] > rank[rootB]) {
            parent[rootB] = rootA;
        } else {
            parent[rootB] = rootA;
            rank[rootA]++;
        }
    }
}
时间复杂度 ≈ O(n)

恰好 n − 1 条边，每条边只做一次 find + union；加上路径压缩与按秩合并， 单次操作均摊接近 O(1)（严格说是 O(α(n))，反阿克曼函数，几乎是常数）。

空间复杂度 O(n)

parent 和 rank 两个长度为 n 的数组。

🚀 小结

套路：「判断一张无向图是不是树」= 「边数恰好 n − 1」+「无环」。 无环这件事天然适合用并查集来判断：处理每条边前先问一句 「这两个点是不是已经连通了」，连通就是环，不连通就合并。 并查集是处理「动态连通性」问题的标配工具，冗余连接、账户合并、岛屿数量这些题都是同一套路。
