一堆区间躺在数轴上，互相有重叠该怎么合并？答案出奇地简单： 先排序，再扫一遍。这题是「排序预处理 + 贪心算法」最经典的入门例子—— 「贪心」这个词听着唬人，其实就是每一步只根据眼前看到的情况，做当下最合理的选择， 一旦选了就不回头。后面你会看到，为什么这样走一步看一步，最终也能拼出正确答案。

🧩 第一步：题目到底要什么

给你一堆区间 [start, end]，有些区间之间有重叠（比如 [1,3] 和 [2,6] 在 2~3 之间是重叠的）。你要把所有互相重叠的区间 揉成一个大区间，最后返回一批互不重叠的区间，并且要把原来所有区间 覆盖到的范围都覆盖住。

输入[[1,3],[2,6],[8,10],[15,18]]

[1,3] 和 [2,6] 重叠 → 合并成 [1,6]；[8,10] 和 [15,18] 各自独立，不和谁重叠，原样保留。 最终输出 [[1,6],[8,10],[15,18]]。

🐢 第二步：笨办法为什么麻烦

如果区间在输入里是乱序的，你想知道某个区间到底和哪些区间重叠，得拿它去和 所有别的区间逐一比较，而且合并之后还可能继续和别的区间产生新的重叠—— 需要反复扫描，逻辑绕来绕去，容易漏判。

关键洞察：如果先把区间按起点从小到大排好序， 那么会互相重叠的区间，在排序后一定是挨在一起的——不可能出现「中间隔着别的区间」 还能重叠的情况。于是问题就从「到处找重叠」简化成了「从左到右扫一遍，看相邻的要不要粘起来」。

⚡ 第三步：排序 + 扫描，一遍搞定

排好序后，维护一个结果列表，手里始终攥着「列表里最后一段」。依次看接下来的区间 cur：

  • 如果 cur 的起点 ≤ 最后一段的终点，说明重叠（或刚好首尾相接） → 把最后一段的终点更新成两者的较大值，继续往后看。
  • 如果 cur 的起点 > 最后一段的终点，说明没有重叠 → 把 cur 当作新的一段，单独放进结果列表。

因为已经按起点排过序，所以这样从左到右扫一遍，绝不会漏掉任何重叠。

注意我们从头到尾只往前走了一遍，从来没有回头改主意： 每看到一个新区间，就当场决定「粘上去」还是「单独开一段」，决定完就不再重新考虑。 这种「每一步只看眼前信息、做当下最合理的选择，一条路走到底」的思路，就叫 贪心算法。能贪心成功的关键，正是第二步那条「关键洞察」—— 它保证了排序之后，眼前的选择就是全局都不会出错的选择，所以我们才敢不回头。

当前正在处理的区间 已经处理完 / 结果中已固定的段 刚单独开的新一段 最终合并结果

试试改成 1,4;4,5;7,9，看 [1,4] 和 [4,5] 刚好首尾相接也会被合并； 而 [7,9] 因为起点大于上一段终点，会单独开一段。

☕ 第四步：写成 Java 代码

class Solution {
    public int[][] merge(int[][] intervals) {
        // 按起点从小到大排序，重叠的区间排序后一定挨在一起
        Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

        List<int[]> merged = new ArrayList<>();
        for (int[] cur : intervals) {
            if (merged.isEmpty() || cur[0] > merged.get(merged.size() - 1)[1]) {
                // 和结果列表最后一段没有重叠，单独开一段新的
                merged.add(cur);
            } else {
                // 有重叠，把最后一段的终点延长成两者的较大值
                int[] last = merged.get(merged.size() - 1);
                last[1] = Math.max(last[1], cur[1]);
            }
        }
        return merged.toArray(new int[merged.size()][]);
    }
}
时间复杂度 O(n log n)

瓶颈在排序；之后从左到右扫一遍只要 O(n)。

空间复杂度 O(log n) ~ O(n)

排序所需的额外空间，加上存放结果的列表（不计入返回值本身的话是 O(log n)）。

🚀 小结

套路：遇到「一堆区间/线段，要判断重叠或合并」的题，第一反应就是 先按起点排序，把无序的重叠关系变成有序的相邻关系，再用一次线性扫描 搞定。这个「排序预处理，把复杂关系简化成相邻比较」的思路，在区间类问题里反复出现。
