不让用除法，怎么知道「除了我自己之外，其他人乘起来是多少」？ 答案是：把「我左边的乘积」和「我右边的乘积」分别算出来，再相乘。

🧩 第一步：题目到底要什么

给一个数组 nums，要你返回一个新数组 answer， 其中 answer[i] 是 除了 nums[i] 自己， 其余所有元素乘起来的结果。并且不准用除法，还要 O(n) 时间内做完。

nums[1, 2, 3, 4]

answer[0] = 2×3×4 = 24（跳过自己 1）；answer[1] = 1×3×4 = 12； answer[2] = 1×2×4 = 8；answer[3] = 1×2×3 = 6 → [24, 12, 8, 6]。

🐢 第二步：最直接的笨办法

最容易想到的：先把所有数乘起来得到总乘积 total， 然后 answer[i] = total / nums[i] 就完了——可惜题目明确禁止用除法， 而且如果数组里有 0，这个除法直接就崩了。 举个例子：nums = [1, 0, 3]，total = 1×0×3 = 0。 对 i = 1（也就是那个 0 自己）来说，我们想要的答案其实是 1×3 = 3， 可是 total / nums[1] 是 0 ÷ 0，根本没法算； 对其他位置来说，total 本身就是 0，随便除出来也全是 0，答案就错了。

那就老老实实来：对每个 i，把数组里除它以外的所有数再扫一遍乘起来 （也就是每个位置都单独跑一次「从头乘到尾，但跳过自己」）。 这样里外两层循环，行话叫 O(n²)（数组每多一个数，要做的运算量就大约翻着法儿地增长）， n 能到 10⁵ 就会跑到超时。

关键观察：把数组想象成一排站好的人，你就是下标 i 那一个。 「除了你自己之外，其他所有人乘起来是多少」，其实就是「你左边所有人的乘积」 乘上「你右边所有人的乘积」——因为这两拨人合起来，正好就是「除了你之外的所有人」。 如果能把「每个位置左边的乘积」和「每个位置右边的乘积」都预先算好、记下来， 那对每个 i 就只剩下把两个记好的数乘一下，不用再重新扫一遍数组了。

⚡ 第三步：前缀积 × 后缀积，三遍扫描搞定

顺着上面这个「左边乘积 × 右边乘积」的想法，我们准备两个帮忙用的数组， 专门记「每个位置左边／右边分别乘到多少了」：

  • left[i]：下标 0 到 i-1 这些位置乘起来的结果（也就是 i 左边所有数的乘积， 不包含 nums[i] 自己）。规定 left[0] = 1—— 因为 0 号位置左边什么数都没有，「什么都没有的乘积」按数学习惯记成 1 （就像什么都不买、账单是 0 元一样，乘法里「空」记成 1 才不会把后面的乘法结果搞乱）。
  • right[i]：下标 i+1 到最后这些位置乘起来的结果（也就是 i 右边所有数的乘积）。 同理，最后一位右边没有数，right[n-1] = 1。

left 怎么一遍就算完：从左往右走，left[i] = left[i-1] × nums[i-1]—— 「i 左边的乘积」就是「i-1 左边的乘积」再多乘上 nums[i-1] （因为往右挪一格，i-1 号这个数也被纳入「左边」了）。right 同理， 只是方向反过来，从右往左走：right[i] = right[i+1] × nums[i+1]。 两个数组各自只需要扫一遍（专业说法叫「线性扫描」，就是老老实实从头走到尾或从尾走到头）， 各自都是 O(n)。

nums[1, 2, 3, 4]

left = [1, 1, 2, 6]（left[1]=1×1=1，left[2]=1×2=2，left[3]=2×3=6）
right = [24, 12, 4, 1]（right[2]=1×4=4，right[1]=4×3=12，right[0]=12×2=24）
逐位相乘 answer[i] = left[i] × right[i] → [24, 12, 8, 6]， 和第一步手算的结果完全一致。

最后 answer[i] = left[i] × right[i]，再扫一遍 O(n)。 全程没有出现除法，三遍线性扫描，加起来总共还是 O(n)。

当前处理的下标 i 这一步用到的「来源」格子 本步刚算出的结果

先看第一遍怎么从左往右把 left 填满，再看第二遍从右往左把 right 填满，最后第三遍逐位相乘得到 answer。 试试把数组改成 -1,1,0,-3,3，看看出现 0 时答案怎么变化。

☕ 第四步：写成 Java 代码（顺手把空间也省掉）

动画里用了 left、right 两个数组方便讲清楚原理， 但真正写代码时其实不用真的开两个数组，还能再省一点内存： 第一遍从左往右，直接把「前缀积」（也就是 left）写进 answer 数组里， 让 answer 先临时兼职当 left 用； 第二遍从右往左，不再需要一整个 right 数组，只用一个变量 （代码里叫它 suffix）就够了——它一边往左走，一边把「目前为止乘到的后缀积」 随身带着更新，每走一步就用它去乘一下 answer[i]，再把自己也乘上 nums[i] 准备下一步用。这样一个变量就顶替了整个 right 数组的作用。 算下来除了输出数组，只多用了这一个变量，额外空间降到 O(1)。 

class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] answer = new int[n];

        // 第一遍：从左往右，把「前缀积」直接写进 answer
        answer[0] = 1;
        for (int i = 1; i < n; i++) {
            answer[i] = answer[i - 1] * nums[i - 1];
        }

        // 第二遍：从右往左，用一个变量滚动维护「后缀积」，边乘边算最终结果
        int suffix = 1;
        for (int i = n - 1; i >= 0; i--) {
            answer[i] = answer[i] * suffix;
            suffix *= nums[i];
        }

        return answer;
    }
}
时间复杂度 O(n)

两遍线性扫描，每遍都是 O(n)。

空间复杂度 O(1)

除了输出数组 answer，只用了一个 suffix 变量（输出数组不计入额外空间）。

🚀 小结

套路：当某个位置的答案依赖「它左边一段」和「它右边一段」各自的某种累积值 （乘积、和、最值……）时，往往可以拆成从左往右和从右往左 两次线性扫描分别预处理，再 O(1) 合并出每个位置的答案——这正是「前缀和 / 前缀积」类技巧的核心， 能避免除法、避免重复计算，把 O(n²) 压成 O(n)。
