不让用除法，怎么知道「除了我自己之外，其他人乘起来是多少」？ 答案是：把「我左边的乘积」和「我右边的乘积」分别算出来，再相乘。

🧩 第一步：题目到底要什么

给一个数组 nums，要你返回一个新数组 answer， 其中 answer[i] 是 除了 nums[i] 自己， 其余所有元素乘起来的结果。并且不准用除法，还要 O(n) 时间内做完。

nums[1, 2, 3, 4]

answer[0] = 2×3×4 = 24（跳过自己 1）；answer[1] = 1×3×4 = 12； answer[2] = 1×2×4 = 8；answer[3] = 1×2×3 = 6 → [24, 12, 8, 6]。

🐢 第二步：最直接的笨办法

最容易想到的：先把所有数乘起来得到总乘积 total， 然后 answer[i] = total / nums[i] 就完了——可惜题目明确禁止用除法， 而且如果数组里有 0，这个除法直接就崩了（除以 0，或者总乘积本来就是 0， 没法反推出原来的某一项）。

那就老老实实来：对每个 i，把数组里除它以外的所有数再扫一遍乘起来。 两层循环，O(n²)，n 能到 10⁵ 就会超时。

关键观察：answer[i] 其实就是「i 左边的乘积」 乘上「i 右边的乘积」。如果能把「每个位置左边的乘积」和 「每个位置右边的乘积」都预先算好，那对每个 i 就只剩一次乘法了。

⚡ 第三步：前缀积 × 后缀积，三遍扫描搞定

定义两个辅助数组：

  • left[i] = nums[0..i-1] 的乘积（i 左边所有数乘起来； left[0] = 1，因为左边没有数，乘积约定为 1）。
  • right[i] = nums[i+1..n-1] 的乘积（i 右边所有数乘起来； 最后一位 right[n-1] = 1，同理）。

left 从左往右边走边乘一遍就能算完，right 从右往左边走边乘一遍也能算完， 各自都是 O(n)。最后 answer[i] = left[i] × right[i]， 再扫一遍 O(n)。全程没有除法，三遍线性扫描，总共还是 O(n)。

当前处理的下标 i 这一步用到的「来源」格子 本步刚算出的结果

先看第一遍怎么从左往右把 left 填满，再看第二遍从右往左把 right 填满，最后第三遍逐位相乘得到 answer。 试试把数组改成 -1,1,0,-3,3，看看出现 0 时答案怎么变化。

☕ 第四步：写成 Java 代码（顺手把空间也省掉）

动画里用了 left、right 两个数组方便讲清楚原理， 实际写代码时可以更省内存：先把「前缀积」直接写进 answer 数组里（相当于省掉了 left），再从右往左用一个变量滚动维护「后缀积」，边乘边写回 answer（相当于省掉了 right）。这样除了输出数组，只多用了一个变量， 额外空间降到 O(1)。 

class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] answer = new int[n];

        // 第一遍：从左往右，把「前缀积」直接写进 answer
        answer[0] = 1;
        for (int i = 1; i < n; i++) {
            answer[i] = answer[i - 1] * nums[i - 1];
        }

        // 第二遍：从右往左，用一个变量滚动维护「后缀积」，边乘边算最终结果
        int suffix = 1;
        for (int i = n - 1; i >= 0; i--) {
            answer[i] = answer[i] * suffix;
            suffix *= nums[i];
        }

        return answer;
    }
}
时间复杂度 O(n)

两遍线性扫描，每遍都是 O(n)。

空间复杂度 O(1)

除了输出数组 answer，只用了一个 suffix 变量（输出数组不计入额外空间）。

🚀 小结

套路：当某个位置的答案依赖「它左边一段」和「它右边一段」各自的某种累积值 （乘积、和、最值……）时，往往可以拆成从左往右和从右往左 两次线性扫描分别预处理，再 O(1) 合并出每个位置的答案——这正是「前缀和 / 前缀积」类技巧的核心， 能避免除法、避免重复计算，把 O(n²) 压成 O(n)。
