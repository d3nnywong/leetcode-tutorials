这题第一次把「图」和「拷贝」放在一起：你拿到的不是一个数组或字符串， 而是一张互相牵着手的节点网络。看完你会明白： 一张哈希表 + 一次 BFS，就能把整张图原样复制一份。

🧩 第一步：题目到底要什么

先搞清楚什么是「图」：图就是一堆点（术语叫节点 node）， 用一些线（术语叫边 edge）两两连起来的网络， 就像地图上的一个个城市，用公路两两连着。跟某个节点直接有一条线相连的 另一个节点，就叫它的邻居 neighbor。

题目给你图里一个节点的引用——「引用」可以理解成「一张写着某个节点地址的小纸条」， 拿着这张纸条你就能顺着它摸到那个节点，再顺着它的邻居、邻居的邻居……一路摸下去。 题目保证图是连通的：从这一个节点出发，顺着边走，能走到图里所有节点， 不会有摸不到的孤岛。

让你返回这张图的深拷贝。「深拷贝」是相对「只复制一张地址纸条」而言的： 如果只是把纸条抄一份给别人，两张纸条其实还是指向同一批节点—— 改动一个，另一边也会跟着变，这不算真的复制。深拷贝要求你真正造出一整套全新的节点， 新节点的值和原图一一对应，连边关系也和原图一模一样——但每个新节点都是 全新的对象，跟原图的节点不共用任何一个。

输入adjList = [[2,4],[1,3],[2,4],[1,3]]

4 个节点，1-2、1-4、2-3、3-4 四条边连成一个环。要复制出一张结构完全一样、 但节点全是新对象的新图，返回新图里那个值为 1 的节点。

关键陷阱：图和「一条路走到底」的结构不一样——图里可能有环 （比如这题的例子：1-2-3-4-1 正好绕成一圈）。假如你想到的克隆办法是 递归——也就是写一个函数，让它自己调用自己去处理更小的同类问题—— 「遇到一个节点，就递归调用自己去克隆它的每个邻居」：那么沿着这个环走， 1 会去克隆邻居 2，2 又去克隆邻居 3，3 又去克隆 4，4 又转回来克隆 1…… 如果不记住「谁已经克隆过」，就会绕着环无限转圈，永远停不下来。

🔑 第二步：关键洞察——一张「原节点 → 克隆节点」的哈希表

先说说哈希表是什么：你可以把它想成一本「秒查字典」——给它一个 key （比如某个原节点），它几乎不用挨个翻找，就能直接告诉你对应的 value （这个原节点克隆出来的那个新节点）是谁。查一次、存一次都近似「一步到位」 （术语叫 O(1)），不会因为表越来越大就变慢。

既然怕「重复克隆」和「绕圈」，那就用一张哈希表 visited 记住 每个原节点对应的克隆节点是谁。遍历时遇到任何一个原节点：

  • 已经在表里？说明它已经被克隆过了，直接拿现成的克隆节点用，不再新建。
  • 不在表里？新建一个值相同的克隆节点，存进表里，再接着处理它的邻居。

这样不管走多少条路径绕回同一个节点，表里查到的永远是同一个克隆节点—— 环不会让我们重复建节点，也不会死循环。

那具体怎么把图里的节点一个个「走」到呢？这里用队列 BFS（广度优先搜索）： 队列就像排队买票，先进去的人先出来（先进先出）；每次从队头 拿出一个节点当作「当前节点 cur」，检查它的每个邻居——没见过的邻居就新建克隆、 塞进队尾排队，等着以后轮到它被取出来处理。这样整张图会像水波纹一样一圈圈往外 扩散开，直到所有节点都处理完。（换成DFS（深度优先搜索）—— 也就是「沿一条路先走到黑，走不通再回头」——效果完全一样，只是走的顺序不同， 真正管用的还是那张哈希表。）

☕ 第四步：写成 Java 代码

把上面的思路翻译成代码：Map 就是那张哈希表， Queue（这里用 LinkedList 实现）就是那个「先进先出」的队列。 

class Solution {
    public Node cloneGraph(Node node) {
        if (node == null) return null;

        // visited: 原节点 -> 克隆节点，既防重复克隆，也防绕环死循环
        Map<Node, Node> visited = new HashMap<>();
        Queue<Node> queue = new LinkedList<>();

        // 先克隆起点自己，加入队列
        queue.offer(node);
        visited.put(node, new Node(node.val));

        while (!queue.isEmpty()) {
            Node cur = queue.poll();
            for (Node neighbor : cur.neighbors) {
                if (!visited.containsKey(neighbor)) {
                    // 第一次见到这个邻居：新建克隆节点，加入队列
                    visited.put(neighbor, new Node(neighbor.val));
                    queue.offer(neighbor);
                }
                // 不管新旧，都把克隆图里 cur 与 neighbor 对应的节点连起来
                visited.get(cur).neighbors.add(visited.get(neighbor));
            }
        }
        return visited.get(node);
    }
}
时间复杂度 O(V + E)

V 是节点总数、E 是边的总数：每个节点入队并处理一次，每条边也会被检查一次 （无向图的一条边会从两端各被看到一次）。

空间复杂度 O(V)

哈希表 visited 与队列最多各存下所有节点。

🚀 小结

套路：但凡涉及「克隆/遍历带环的结构」（图、带 random 指针的链表等）， 第一反应就是用一张哈希表记住「原对象 → 新对象」的映射， 它既能避免重复创建，又能天然防止绕环死循环。BFS／DFS 只是遍历顺序的选择， 真正的核心是这张映射表。
