两个字符串的字母「拆开打乱重排」会不会长得一样？ 看完你会明白：一张 26 格的计数表，就能一次扫描把答案算出来。

🧩 第一步：什么是「字母异位词」

字母异位词（anagram）就是把一个字符串里的字母打乱顺序重新排列， 能不能拼出另一个字符串。换句话说：两个字符串长度一样， 而且每种字母出现的次数也完全一样，只是顺序不同。

sanagram
tnagaram

两边都用了 a×3、n×1、g×1、r×1、m×1，只是排列顺序不同 → t 是 s 的字母异位词，返回 true。

🐢 第二步：最直接的笨办法

最容易想到的办法：把 s 和 t 各自排序， 排完序的两个字符串如果一模一样，那它俩用的字母种类和个数肯定也一样， 自然就是字母异位词。

慢在哪：排序一次是 O(n log n)，比线性扫描慢；而且这种做法 没有真正利用「字母只有 26 种」这个特点——其实我们根本不需要排序，只要 数一数每个字母出现了几次就够了。

⚡ 第三步：用计数数组「一边数一边抵消」

换个思路：先把 s 的每个字符都数一遍，记进一张「字母 → 次数」的计数表 count 里（每个字母 +1）。

这张计数表落到代码里，可以偷个懒：反正字母只有 26 种，那就干脆开一排 26 个格子（一个长度为 26 的数组），第 0 号格子专门记 'a' 出现了几次， 第 1 号格子记 'b'……第 25 号格子记 'z'。字符在计算机里其实也是数字，而且相邻字母正好 差 1（'b' 比 'a' 大 1，'c' 比 'a' 大 2……），所以用字符 c 减去 'a' 这个小算术，就能直接算出「这个字母该放第几号格子」——不用真的去查字典，一步减法搞定。

有了这排格子，接着扫一遍 t，每碰到一个字符就把它对应格子里的次数 −1 （相当于「拿 t 的字母去抵消 s 记下的字母」）。如果某个字符要抵消时，那个格子 里的次数已经是 0，说明 t 里这个字母比 s 多，肯定不是异位词，直接判 false；如果 t 顺利扫完、全部抵消干净， 就是异位词。整个过程只扫两遍字符串，是 O(n)。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isAnagram(String s, String t) {
        // 长度都不同，字母个数不可能凑成一样
        if (s.length() != t.length()) {
            return false;
        }
        // count[c - 'a']：字母 c 在 s 里还剩多少个「没被抵消」
        int[] count = new int[26];
        for (int i = 0; i < s.length(); i++) {
            count[s.charAt(i) - 'a']++;
        }
        for (int i = 0; i < t.length(); i++) {
            int idx = t.charAt(i) - 'a';
            if (count[idx] == 0) {
                // s 里没有多余的这个字母可以抵消了
                return false;
            }
            count[idx]--;
        }
        return true;
    }
}
时间复杂度 O(n)

s、t 各扫一遍，n 是字符串长度。

空间复杂度 O(1)

计数表固定开 26 格，不会因为字符串变长而变大。但如果字母不再局限于英文小写字母， 比如换成中文、emoji 这类更丰富的 unicode 字符（unicode 是计算机给全世界文字统一 编号的一套字符集，种类远不止 26 种），就没法再固定开 26 格了——这正是进阶问题追问的方向。

🚀 小结

套路：当题目的字符集合很小（比如只有 26 个小写字母）时， 排序往往不是最优解——开一张固定大小的计数表，边扫边加边减， 能把时间复杂度从 O(n log n) 降到 O(n)， 额外空间也从「跟输入相关」变成常数级的 O(1)。
