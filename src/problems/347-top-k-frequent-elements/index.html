给一堆数字，找出出现次数最多的 K 个。先学会用哈希表「边走边记」， 再看怎么用一个巧妙的「桶排序」把它从 O(n log n) 降到 O(n)。

🧩 第一步：题目到底要什么

给你一个数组 nums，里面的数字会重复出现。再给你一个整数 k，要你找出出现次数最多的 k 个不同数字， 顺序随意。

nums[1, 1, 1, 2, 2, 3]
k2

1 出现 3 次，2 出现 2 次，3 出现 1 次。出现次数前 2 高的是 1 和 2 → 返回 [1, 2]。

🐢 第二步：最直接的办法，以及它的瓶颈

第一反应通常分两步走：

  1. 先把每个数字出现的次数数一遍。这一步可以借助哈希表—— 你可以把它想象成一本会自动翻页的「查号簿」：拿一个数字当「名字」去查， 立刻就能翻到它对应的「次数」，不用把统计过的东西挨个比对一遍。 边扫数组边记次数，这一步是 O(n)。
  2. 再把这些「数字 → 次数」的对子，按次数从大到小排序， 取排在最前面的 k 个数字。
慢在哪：第 2 步的排序才是瓶颈。常见的排序办法都要靠「两两比较、谁大谁排前面」， 比较的次数大约是 O(u log u)（u 是不同数字的个数，最坏时 u = n，也就是所有数字都不重复）。题目「进阶」明确要求做到优于 O(n log n)——既然我们只要前 k 个，根本没必要把全部数字都排好序， 只要能把「次数最多的那一小撮」直接挑出来就够了。

有一个关键观察，能帮我们跳出「必须排序」的思维定式：一个数字在数组里最多能出现 n 次（不可能超过数组总长度），所以「出现次数」这个值永远落在 [1, n] 之间——是一个提前就知道大小、而且很小的区间。既然次数的取值这么有限， 我们完全可以不比较、直接按次数「对号入座」：给每一种可能出现的次数都准备一个格子， 一个数字出现了几次，就把它放进第几号格子。扫一遍数组，把所有数字都放进对应的格子， 再把格子从「次数最多」往「次数最少」的方向扫一遍，答案自然就按顺序冒出来了—— 全程不需要任何两两比较。这个「按值直接分格子」的思路，就叫桶排序 （格子在这里就叫「桶」）。

⚡ 第三步：用「桶排序」一遍搞定

把刚才那个「按次数分格子」的想法落地，做法分三步：

  1. 扫一遍 nums，用哈希表统计每个数字出现的次数。
  2. 开一个大小为 n + 1 的「桶」数组（就是刚才说的那一排格子）， 下标（第几号格子）就是出现次数。每个数字按它的出现次数， 被放进对应下标的桶里（同一个桶可能装下好几个数字，因为它们出现的次数正好一样多）。
  3. 从下标最大（出现次数最多）的桶开始，往下标小的桶扫，边扫边把桶里的数字收进答案， 凑够 k 个就停下——因为我们是从「次数最多」往「次数最少」的方向走， 收到的前 k 个自然就是出现次数最高的那批，完全不用先给所有数字排序。

因为桶的数量是 O(n)，每个数字只会进一个桶，整个过程没有任何比较排序， 时间复杂度是 O(n)——比 O(n log n) 更优，正好满足进阶要求。

☕ 第四步：写成 Java 代码

class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        // 1) 统计频率：值 -> 次数
        Map<Integer, Integer> freq = new HashMap<>();
        for (int x : nums) {
            freq.put(x, freq.getOrDefault(x, 0) + 1);
        }

        // 2) 按频率装桶：下标即出现次数，桶里放达到该频率的数字
        int n = nums.length;
        List<Integer>[] buckets = new List[n + 1];
        for (Map.Entry<Integer, Integer> e : freq.entrySet()) {
            int count = e.getValue();
            if (buckets[count] == null) {
                buckets[count] = new ArrayList<>();
            }
            buckets[count].add(e.getKey());
        }

        // 3) 从高频桶往低频桶收集，凑够 k 个就停
        int[] result = new int[k];
        int idx = 0;
        for (int count = n; count >= 1 && idx < k; count--) {
            if (buckets[count] == null) continue;
            for (int val : buckets[count]) {
                result[idx++] = val;
                if (idx == k) break;
            }
        }
        return result;
    }
}
时间复杂度 O(n)

统计频率一遍、装桶一遍、收集一遍，全程没有比较排序。

空间复杂度 O(n)

哈希表和桶数组最坏情况下都要存下接近 n 个元素。

🚀 小结

套路：当你要排序的「键」（这里是出现次数）的取值范围 有限且已知时，不一定要用通用的比较排序，可以考虑「桶排序」—— 把键直接当下标，开桶分组，一次扫描搞定。这道题还告诉我们： 题目只要前 k 个，往往意味着不需要把全部元素都排好序， 能省的功夫尽量省。
