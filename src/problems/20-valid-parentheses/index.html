括号匹配是「栈」这个数据结构最经典的应用场景。看完你会明白： 为什么「最近打开的括号要最先闭合」天生就该用一个先进后出的栈来记。

🧩 第一步：题目到底要什么

给你一个只包含 ()[]{} 六种字符的字符串，判断它的括号是不是「配对配得对」： 每个左括号都要能找到一个同类型、顺序也对的右括号跟它闭合。

s"([)]"

乍一看左右括号数量是对的（都各 1 个），但 ( 还没闭合时 ] 就先把 [ 闭合了，顺序乱了 → false。 可见光数「数量」不够，还得管「顺序」。

💡 第二步：关键洞察——后开的要先关

仔细观察「合法」的括号串，会发现一个规律：最后一个被打开、还没闭合的左括号， 必须是下一个被闭合的。比如 {[()]} 里，最内层的 ( 最后开、却要最先用 ) 闭合，然后才能轮到 [，最后才是 {。

「最后进来的，最先出去」——这正好是栈（Stack）这种数据结构的天性 （后进先出，LIFO）。所以思路很自然：

  • 从左到右扫描字符串，碰到左括号就直接压入栈。
  • 碰到右括号就看栈顶：如果栈是空的，或者栈顶括号类型对不上， 说明匹配失败，直接判 false；如果类型一致，就把栈顶弹出，配对成功。
  • 扫完整个字符串后，如果栈也正好空了，说明所有左括号都被配对了 → true； 否则还有左括号孤零零没人闭合 → false。
容易漏掉的两个边界：扫到右括号但栈是空的（比如 ")"）， 以及扫完了但栈还没空（比如 "(("）——这两种都是无效，别忘了判。

刚压入栈顶 左括号，入栈 右括号，配对成功并弹栈 右括号，配对失败

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isValid(String s) {
        Deque<Character> stack = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (c == '(' || c == '[' || c == '{') {
                stack.push(c); // 左括号，压入栈顶
            } else {
                if (stack.isEmpty()) {
                    return false; // 没有左括号能配它
                }
                char top = stack.pop();
                boolean matched = (c == ')' && top == '(')
                        || (c == ']' && top == '[')
                        || (c == '}' && top == '{');
                if (!matched) {
                    return false; // 类型不一致
                }
            }
        }
        return stack.isEmpty(); // 栈空 = 所有左括号都被配对了
    }
}
时间复杂度 O(n)

字符串只扫一遍，每个字符的入栈/出栈都是 O(1)。

空间复杂度 O(n)

最坏情况（全是左括号）栈要存下几乎所有字符。

🚀 小结

套路：只要问题里出现「最近发生的事要最先处理 / 撤销」这种「后进先出」 的结构关系——括号匹配、表达式求值、撤销操作、DFS 递归展开——第一反应就该想到栈。 有效的括号是这个套路最干净的入门例子。
