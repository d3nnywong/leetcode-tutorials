「最大子数组和」的孪生题，但乘法多了一个麻烦：负数会让最大变最小、最小变最大。 看完你会明白：为什么必须同时记住「最大」和「最小」两条线索，才能不漏掉翻身的机会。

🧩 第一步：题目到底要什么

给你一排数字（可能有正有负，也可能有 0），要你从中挑出连续的一段 （子数组，至少 1 个元素），让这一段所有数相乘的结果最大，返回这个最大乘积。

nums[2, 3, -2, 4]

挑出 [2, 3] 这一段，乘积是 2 × 3 = 6，比其他任何一段连续子数组的乘积都大 → 答案是 6。注意 [2,3,-2,4] 整段的乘积是 −48， 反而比只取 [2,3] 还小，这就是乘法的「坑」。

🐢 第二步：笨办法，以及乘法独有的麻烦

最容易想到的：枚举每一个起点 i 和终点 j，把 nums[i..j] 的乘积都求出来，记录最大值。两层循环再加求积，O(n²) 起步，n 能到 2×10⁴ 时已经很吃力。

这题还有一道「孪生题」——LeetCode 53「最大子数组和」（求连续一段加起来最大， 不是乘起来）。那道题有个很有名的省时技巧，叫 Kadane 算法：从左到右扫一遍， 每走一步只记住一个数字——「以当前这个位置结尾的连续一段，加起来最大是多少」， 记作 cur。如果 cur 已经变成负数（说明前面这段在拖后腿）， 就把前面全部丢掉，从当前这个数重新开始；否则就把当前数接到 cur 后面继续累加。这样只需要从左到右扫一遍，比两层循环快得多。

能不能把这一招原样搬到「乘积」上——也只维护一个「以当前元素结尾的最大乘积」 cur？不行。因为乘法和加法有一个本质区别：

负负得正：加法里，一个数越负，后面再加什么，也顶多是「没那么负」， 不会突然「翻身」变成很大的正数。但乘法不一样。举个例子：假设 cur 现在是 -6（一个很负的数，也就是「目前乘出来最小」的那条线索）。 下一个数又是一个负数，比如 -2：-6 × (-2) = 12， 瞬间从「最小」变成了一个不小的正数！如果我们只顾着记录「最大」、 把这种很负的 cur 早早扔掉不要，就会白白错过这次「负负得正」翻身的机会。

⚡ 第三步：同时维护 curMax 和 curMin

解法是：每走一步都同时维护两条线索——「以当前元素结尾」的 最大乘积 maxEnd[i] 和最小乘积 minEnd[i]。最小乘积存在的唯一意义，就是给「下一次乘上负数」当种子。

走到 nums[i] 时，有三个候选都可能成为新的 maxEnd[i] 或 minEnd[i]：

  • 自己重新开始： x = nums[i]（前面那段不值得要，或者前面就是 0）。
  • 接上前一步的最大乘积： maxEnd[i-1] × x。
  • 接上前一步的最小乘积： minEnd[i-1] × x（x 是负数时，这条往往才是真正的大赢家）。

把这三个候选写成公式，就是下面的「递推式」——「递推」的意思很朴素： 这一步的答案，是从上一步已经算好的答案推出来的，一步接一步往前推， 就能把整个数组算完，不用回头重算：

递推式 maxEnd[i] = max(x, maxEnd[i-1]×x, minEnd[i-1]×x)
minEnd[i] = min(x, maxEnd[i-1]×x, minEnd[i-1]×x)

每一步都用新的 maxEnd[i] 去刷新全局答案 best。 数组只扫一遍，O(n) 时间、O(1) 额外空间。

手算一遍nums = [-2, 3, -4]

i=0，x=-2：还没有「前一步」，直接自己开头。 maxEnd=-2，minEnd=-2，best=-2。
i=1，x=3：三个候选——自己重开是 3；接最大 maxEnd(-2)×3=-6； 接最小 minEnd(-2)×3=-6。三者取大的 3 当 maxEnd， 取小的 -6 当 minEnd。best 更新成 3。
i=2，x=-4：三个候选——自己重开是 -4；接最大 maxEnd(3)×(-4)=-12； 接最小 minEnd(-6)×(-4)=24。这一步「接最小」直接翻身成了全场最大！ maxEnd=24，minEnd=-12，best 更新成 24。

最终答案 24，正是 -2 × 3 × -4 整段乘起来。 如果只记录「最大乘积」这一条线索，i=1 时那条 minEnd=-6 的伏笔会被当成没用的东西扔掉， i=2 就再也翻不出 24 这个答案了——这正是本题非同时维护「最大 + 最小」不可的原因。

当前正在计算的下标 i 被「接上」的来源格（上一步的 maxEnd / minEnd） best 取得的位置 在此处「自己重新开始」

试试把数组改成 -2,0,-1（0 会把两条线索都「清零重启」）或 -2,3,-4（连续两个负数让 minEnd 翻身变成全局最大），观察 maxEnd / minEnd 的变化。

☕ 第四步：写成 Java 代码

class Solution {
    public int maxProduct(int[] nums) {
        // curMax / curMin：以当前元素结尾的最大、最小乘积
        int curMax = nums[0];
        int curMin = nums[0];
        int best = nums[0];

        for (int i = 1; i < nums.length; i++) {
            int x = nums[i];
            // 三个候选：自己重新开始 / 接上前一步的最大 / 接上前一步的最小
            int fromMax = curMax * x;
            int fromMin = curMin * x;

            int newMax = Math.max(x, Math.max(fromMax, fromMin));
            int newMin = Math.min(x, Math.min(fromMax, fromMin));

            curMax = newMax;
            curMin = newMin; // 留给下一步「乘上负数翻身」用
            best = Math.max(best, curMax); // 随时刷新全局最大
        }
        return best;
    }
}
时间复杂度 O(n)

数组只扫一遍，每步只做常数次乘法和比较。

空间复杂度 O(1)

只用了 curMax、curMin、best 三个变量，没有额外数组。

🚀 小结

套路：遇到「连续子数组的最优值」类问题，先想 Kadane 那一套——维护 「以当前元素结尾」的状态，边走边刷新全局答案。但只要运算不是单调的（比如这里的乘法， 负数会反转大小关系），就要多留一手：同时维护「最优」和「最劣」， 因为今天的最劣，可能就是明天翻身的最优。这种「一步接一步用上一步的结果往前推」的解题方式， 就是动态规划（简称 DP）；本题「同时维护最大 + 最小」的双线索版本， 在很多带符号反转的题里都会用到。
