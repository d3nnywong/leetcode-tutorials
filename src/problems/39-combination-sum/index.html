「回溯」家族的入门代表作：一边往路径里塞数字、一边检查还差多少， 凑够了就记一笔，凑不够或超了就把刚选的数字撤回去，换下一个再试。

🧩 第一步：题目到底要什么

给你一堆候选数字 candidates（没有重复）和一个目标值 target， 要你找出所有「挑几个数字加起来正好等于 target」的方案。 关键规则：同一个数字可以重复使用（比如 2 可以连选好几次）， 但「选了几个不同数字的组合」不能重复出现（顺序不同算同一组）。

candidates[2, 3, 6, 7]
target7

2 + 2 + 3 = 7，单独一个 7 也等于 7 → 答案是 [[2,2,3], [7]]。 注意 [2,2,3] 和 [3,2,2] 算同一组，只算一次。

🌲 第二步：把「试一试」画成一棵树

这题没法用公式直接算，但可以暴力地一个个试： 每一步面前摆着几个候选数字，选一个加进当前路径，剩下的目标就变小； 如果变成 0，说明这条路径凑出了一组解；如果还没到 0，就接着选； 选完这一条路之后，要把刚才选的数字撤回去（回溯）， 换下一个候选再试一遍。这样把所有「选哪个、选几次」的可能性都走一遍，就是 回溯法（Backtracking），本质是一棵决策树的深度优先遍历。

为了不把 [2,2,3] 和 [3,2,2] 当成两组不同答案， 每一层只允许从当前下标 start 往后选（不能往回选已经跳过的数字）。 又因为同一个数字能重复用，递归下一层时下标传的还是 i 本身，而不是 i + 1。

剪枝优化：把 candidates 先从小到大排序。 这样一旦发现当前候选数字已经比「还差多少（remain）」大， 后面排在它右边的数字只会更大，不可能再凑出 remain —— 直接跳出这一层循环，省掉一大片注定失败的分支，不用一个个再去试。

⚡ 第三步：看回溯怎么「选 → 试 → 撤」

下面动画把候选数组、当前路径、剩余目标 remain 都画出来。 每一步要么「选一个数字压入路径」，要么「发现选过头了/比剩余目标还大，剪枝跳过」， 要么「回溯，把刚选的数字弹出路径」。凑齐 remain = 0 的那一刻， 这条路径就会被记进右边的「已找到的组合」里。

☕ 第四步：写成 Java 代码

class Solution {
    private final List<List<Integer>> result = new ArrayList<>();
    private final List<Integer> path = new ArrayList<>();

    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        Arrays.sort(candidates); // 排序后才能用「比 remain 大就剪枝」的优化
        backtrack(candidates, target, 0);
        return result;
    }

    private void backtrack(int[] candidates, int remain, int start) {
        if (remain == 0) {
            result.add(new ArrayList<>(path)); // 找到一组合法组合，拷贝一份记下来
            return;
        }
        for (int i = start; i < candidates.length; i++) {
            if (candidates[i] > remain) break; // 已排序，后面的更大，直接剪枝
            path.add(candidates[i]);
            backtrack(candidates, remain - candidates[i], i); // 传 i 而不是 i+1：可以重复选自己
            path.remove(path.size() - 1); // 回溯：撤销这次选择，换下一个候选
        }
    }
}
时间复杂度 O(N^(T/M + 1))

N 是候选数个数，T 是 target，M 是候选中的最小值；本质是搜索树的节点数， 和具体输入强相关，剪枝能大幅减少实际访问的分支。

空间复杂度 O(T / M)

递归栈的深度最坏是 target 除以最小候选值（一直选最小的数字往下叠）。

🚀 小结

套路：遇到「找出所有满足条件的方案/组合」这类题， 先想「能不能一个个试」——选一个、递归往下走、不行就撤回来换下一个，这就是回溯。 排好序之后，「发现走不下去就提前跳出循环」是回溯题里最常见、最有效的剪枝手段， 后面的子集、排列、分割回文串等题都在用同一套模板。
