「回溯」家族的入门代表作：一边往路径里塞数字、一边检查还差多少， 凑够了就记一笔，凑不够或超了就把刚选的数字撤回去，换下一个再试。

🧩 第一步：题目到底要什么

给你一堆候选数字 candidates（没有重复）和一个目标值 target， 要你找出所有「挑几个数字加起来正好等于 target」的方案。 关键规则：同一个数字可以重复使用（比如 2 可以连选好几次）， 但「选了几个不同数字的组合」不能重复出现（顺序不同算同一组）。

candidates[2, 3, 6, 7]
target7

2 + 2 + 3 = 7，单独一个 7 也等于 7 → 答案是 [[2,2,3], [7]]。 注意 [2,2,3] 和 [3,2,2] 算同一组，只算一次。

🌲 第二步：把「试一试」画成一棵树

这题没有什么公式能一步算出答案，但可以暴力地一个个试—— 而且这种「试」是有章法的，不是瞎蒙。 把眼前已经选中的数字串成一条路径（比如选了两次 2，路径就是 [2, 2]）； 每往路径里加一个新数字，接下来还要凑的目标（我们把它叫 remain， 也就是「还差多少」）就跟着变小。 如果 remain 减到 0，说明这条路径刚好凑出了一组解，赶紧记下来； 如果 remain 还没到 0，就在这条路径的基础上，再选一个数字接着试。

把所有「选了什么、接下来还能选什么」的可能性画出来，会长成一棵倒着长的树： 最上面是「什么都还没选」，每往下一层，就是「又多选了一个数字」。 走到某一层，发现凑够了、或者发现无论怎么选都凑不出来了， 就沿着刚才走的这条树枝退回上一层，换一个没试过的数字，再往下走一遍。 这跟走迷宫很像：先顺着一条路一直走到底，走不通就退回上一个岔路口， 换另一条没走过的路再走到底，直到把所有岔路都试过一遍。 这种「一条路走到黑，走不通就退回去、换一条路再走到黑」的搜索方式，叫 深度优先遍历（DFS）；而「往下走一步、发现走不通就把刚才那一步选的数字撤销掉」 这个动作，就叫回溯（Backtracking）——回溯法，其实就是用 DFS 的方式， 把「所有可能选法」这棵树老老实实地走一遍。

代码里怎么实现「往下走一层」？靠的是递归：写一个函数， 每当它要「往下走一层」，就调用它自己去处理「选完当前数字之后，剩下还要凑多少」 这个规模更小、但形式完全一样的子问题——有点像俄罗斯套娃，一层套一层地往里钻， 直到 remain 变成 0（凑够了，可以打开最里面那层娃娃了） 或者已经没法再选（此路不通，往回退一层）。

为了不把 [2,2,3] 和 [3,2,2] 当成两组不同答案， 代码里用一个 start 下标（下标就是数组里「排第几个」的编号， 从 0 开始数）来规定：这一层只能从第 start 个候选数字开始往后挑， 不能往回选前面已经跳过的数字。又因为同一个数字可以重复使用， 递归下一层时传的下标还是 i 本身（表示「自己也还能再被选一次」）， 而不是 i + 1（那样就变成「只能选后面的、自己不能再选一次」了）。

剪枝优化：把 candidates 先从小到大排序。 这样一旦发现当前候选数字已经比「还差多少（remain）」大， 后面排在它右边的数字只会更大，不可能再凑出 remain —— 直接跳出这一层循环，相当于把这一整条注定凑不出答案的树枝提前砍掉 （这就是「剪枝」这个名字的由来），不用一个个再去傻试。

⚡ 第三步：看回溯怎么「选 → 试 → 撤」

下面动画把候选数组、当前路径、剩余目标 remain 都画出来。 每一步要么「选一个数字压入路径」，要么「发现选过头了/比剩余目标还大，剪枝跳过」， 要么「回溯，把刚选的数字弹出路径」。凑齐 remain = 0 的那一刻， 这条路径就会被记进右边的「已找到的组合」里。

☕ 第四步：写成 Java 代码

class Solution {
    private final List<List<Integer>> result = new ArrayList<>();
    private final List<Integer> path = new ArrayList<>();

    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        Arrays.sort(candidates); // 排序后才能用「比 remain 大就剪枝」的优化
        backtrack(candidates, target, 0);
        return result;
    }

    private void backtrack(int[] candidates, int remain, int start) {
        if (remain == 0) {
            result.add(new ArrayList<>(path)); // 找到一组合法组合，拷贝一份记下来
            return;
        }
        for (int i = start; i < candidates.length; i++) {
            if (candidates[i] > remain) break; // 已排序，后面的更大，直接剪枝
            path.add(candidates[i]);
            backtrack(candidates, remain - candidates[i], i); // 传 i 而不是 i+1：可以重复选自己
            path.remove(path.size() - 1); // 回溯：撤销这次选择，换下一个候选
        }
    }
}
时间复杂度 O(N^(T/M + 1))

N 是候选数个数，T 是 target，M 是候选中的最小值；本质是搜索树的节点数， 和具体输入强相关，剪枝能大幅减少实际访问的分支。

空间复杂度 O(T / M)

每往下递归一层，就像往一摞盘子上再摞一个盘子——这一摞「还没返回的调用」 就叫递归栈，只能从最上面一层层往下收； 最坏情况是一直选最小的候选值往下叠，摞的层数（深度）约等于 target 除以最小候选值。

🚀 小结

套路：遇到「找出所有满足条件的方案/组合」这类题， 先想「能不能一个个试」——选一个、递归往下走、不行就撤回来换下一个，这就是回溯。 排好序之后，「发现走不下去就提前跳出循环」是回溯题里最常见、最有效的剪枝手段， 后面的子集、排列、分割回文串等题都在用同一套模板。
