区间类问题的入门课。区间已经按起点排好序，这件事本身就给了我们一条免费的扫描路径—— 看完你会明白：为什么只需要从左到右扫一遍，而不需要重新排序或两两比较。

🧩 第一步：题目到底要什么

先说清楚一个词：区间。这里的区间就是一对数字 [start, end]， 表示「从 start 到 end 之间的一段范围」，这两个数字分别叫这个区间的起点和 结束点。你可以把它想成一段被占用的时间，比如 [9, 10] 就是「从第 9 分钟占到第 10 分钟」。

题目给你一串区间 intervals，它已经按起点从小到大排好序 （前一个区间的起点一定不大于后一个），而且互不重叠（谁也没跟谁挤在一起， 没有任何两个区间共享同一段范围）。再给你一个新区间 newInterval，要把它塞进这串区间里： 如果它跟原来的某些区间有重叠——哪怕只是边界刚好碰上，也算——就要把这些重叠的区间 合并成一个更大的区间；合并完之后，整个列表仍然要保持 「按起点排序、互不重叠」这两条性质不能被破坏。

intervals[[1,2],[3,5],[6,7],[8,10],[12,16]]
newInterval[4,8]

[4,8] 跟 [3,5]、[6,7]、[8,10] 都有重叠，合并成 [3,10]；[1,2] 和 [12,16] 没碰到，原样留着。 结果是 [[1,2],[3,10],[12,16]]。

🐢 第二步：为什么不用「插进去再排序再合并」

最直白的想法：把 newInterval 直接塞进数组，再整体排序、再跑一遍区间合并。 能做出来，但排序要 O(n log n)，而我们手里其实有一个被白白浪费的条件—— intervals 本来就是有序的。

关键洞察：既然原数组已经有序，新区间插入后，受影响的只会是 连续的一段（跟它重叠的那些区间），而不是整个数组散乱地变化。 于是整个问题可以拆成三段，从左到右扫一遍就搞定，完全不需要排序。

⚡ 第三步：三段式扫描合并

还是拿第一步的例子来走一遍：intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]]， newInterval = [4,8]。把 intervals 从左到右扫一遍， 拿每个区间跟新区间比较起点、结束点的大小关系，自然会分成三种情况：

  • 第一段（完全在左边，不重叠）：只要 intervals[i] 的结束点 还在新区间起点的左边，说明它俩隔得远远的，压根没碰到，原样收进结果。 比如 [1,2] 的结束点 2，小于新区间 [4,8] 的起点 4，直接收进去。
  • 第二段（重叠，吸收合并）：只要 intervals[i] 的起点 不超过新区间（目前已合并出的）结束点，就说明两个区间重叠了——哪怕只是边界刚好碰上 （比如 intervals[i] 的起点正好等于新区间当前的结束点）也算重叠，这一点很容易漏掉，一定要记住。 重叠的话，就把新区间的起点、结束点分别更新成两边的 min(起点) 和 max(结束点)，也就是把它「吸收」进新区间， 不单独放进结果。跟着例子走一遍：[3,5] 起点 3 ≤ 8，重叠， 新区间合并成 [3,8]；[6,7] 起点 6 ≤ 8，重叠，合并后仍是 [3,8]（因为 7 小于 8，结束点没变大）；[8,10] 起点 8 ≤ 8， 虽然只是刚好挨上，也算重叠，合并后新区间变成 [3,10]。
  • 第三段（完全在右边，不重叠）：合并阶段一旦结束，新区间就不会再变大了， 剩下的区间必然和它没有重叠，原样收进结果。比如 [12,16] 的起点 12，大于当前新区间 [3,10] 的结束点 10，直接收进去。

把合并完毕的新区间，插在第一段和第三段之间，三段拼起来正好是 [[1,2], [3,10], [12,16]]——跟第一步的答案完全一致。整个过程每个区间只看一次，O(n)。

当前正在比较的区间 不重叠，原样收进结果 与新区间重叠，被吸收合并 新区间（合并中 / 已插入结果）

试试把 newInterval 改成 2,5（对应示例 1 的 intervals），看它只合并一个区间的过程。

☕ 第四步：写成 Java 代码

class Solution {
    public int[][] insert(int[][] intervals, int[] newInterval) {
        List<int[]> result = new ArrayList<>();
        int i = 0, n = intervals.length;
        int start = newInterval[0], end = newInterval[1];

        // 1. 完全在新区间左边，不重叠，原样收进结果
        while (i < n && intervals[i][1] < start) {
            result.add(intervals[i]);
            i++;
        }

        // 2. 与新区间重叠，吸收合并（更新新区间的左右边界）
        while (i < n && intervals[i][0] <= end) {
            start = Math.min(start, intervals[i][0]);
            end = Math.max(end, intervals[i][1]);
            i++;
        }
        result.add(new int[]{start, end});

        // 3. 合并完毕后，剩下的区间必然不再重叠，原样收进结果
        while (i < n) {
            result.add(intervals[i]);
            i++;
        }

        return result.toArray(new int[result.size()][]);
    }
}
时间复杂度 O(n)

每个区间只看一次，不需要排序。

空间复杂度 O(n)

用来存放结果数组（不算返回值本身的话是 O(1) 额外空间）。

🚀 小结

套路：题目给的「已排序」条件往往不是摆设——它意味着受影响的区域一定是 连续的一段，可以用一次线性扫描、按「左边不重叠 → 中间合并 → 右边不重叠」 三段式处理，而不必重新排序再合并。这个思路在区间合并、会议室调度等题目里会反复出现。
