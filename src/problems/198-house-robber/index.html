一维动态规划的「选还是不选」经典题。看完你会发现：偷不偷某间房子， 其实就是在「保留上一步的最优解」和「加上这一步、放弃上一间」之间做一次选择。

🧩 第一步：题目到底要什么

一排房子，每间都有一笔钱 nums[i]。你想偷到尽量多的钱， 但有个规矩：相邻的两间房子不能在同一晚都偷（会触发警报）。 问：在这个限制下，最多能偷到多少钱？

nums[2, 7, 9, 3, 1]

偷 0 号（¥2）+ 2 号（¥9）+ 4 号（¥1），互不相邻 → 共 2 + 9 + 1 = 12。 注意 1 号的 ¥7 单看很诱人，但偷了它就必须放弃 0 号和 2 号，反而更少。

🐢 第二步：先想最后一间，而不是第一间

挨个房子去试「偷」或「不偷」会有指数级的组合，根本试不完。换个角度—— 盯着最后一间房子（第 i 间）往回看：不管前面怎么偷， 对第 i 间房子，永远只有两种选择：

  • 不偷第 i 间：那么前面 0..i-1 间的最优解是多少，答案就是多少——也就是 dp[i-1]。
  • 偷第 i 间：那么第 i-1 间绝对不能偷，能拿到的是「0..i-2 间的最优解」加上这一间的钱——也就是 dp[i-2] + nums[i]。

这两种选择互斥（第 i 间要么偷要么不偷，不会同时发生）， 所以「0..i 间能偷到的最高金额」就是这两者中较大的那个。

如果直接暴力枚举：每间房子两种选择（偷/不偷），n 间房子就是 2^n 种组合，逐一检查是否合法再求最大值——指数级，房子一多就算不动。 动态规划要做的，就是把每个「子问题」（dp[i-1]、dp[i-2]……）只算一次、记下来。

🧠 第三步：动态规划三件套

1

状态：dp[i] 表示什么？

dp[i] = 只考虑下标 0..i 这些房子时，能偷到的最高金额。

最终答案就是 dp[n-1]（n 是房子总数）。

2

转移：dp[i] 怎么由更小的问题推出来？

照上面的洞察，第 i 间要么不偷（沿用 dp[i-1]），要么偷（dp[i-2] 加上 nums[i]）：

dp[i] = max(dp[i-1], dp[i-2] + nums[i])（i ≥ 2）

3

边界：最小的情况是什么？

dp[0] = nums[0]（只有一间房，没得选，直接偷）； dp[1] = max(nums[0], nums[1])（两间相邻，二选一，取较大的）。 这两个地基定下来，后面全靠它们往后推。

下面这个动画一间一间往后算：当前在算的房子会高亮，它依赖的「不偷方案」dp[i-1] 和「偷的方案」dp[i-2] 会被标成来源色，算完把最高金额填到房子上方； 算到最后一间后，再回溯出真正被偷的那几间房子，用金色标出最终方案。

当前正在算的 dp[i] 依赖的 dp[i-1] 与 dp[i-2] 最终被偷的房子

试试把数组改成 1,2,3,1，看它怎么算出 4； 再改成 5,1,1,5 这种「两头大中间小」的数组，看它怎么避开中间偷两头。

☕ 第四步：写成 Java 代码

动画里我们把每一间的最高金额都存了下来，但仔细看转移方程会发现： 算 dp[i] 时只用到了最近的两个值 —— 没必要存下整张表，用两个变量滚动更新就够了，空间从 O(n) 降到 O(1)。 

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 1) return nums[0]; // 只有一间房，没得选

        int prev2 = nums[0];                         // 滚动变量：dp[i-2]，初始对应 dp[0]
        int prev1 = Math.max(nums[0], nums[1]); // 滚动变量：dp[i-1]，初始对应 dp[1]

        for (int i = 2; i < n; i++) {
            int cur = Math.max(prev1, prev2 + nums[i]); // 不偷 vs 偷
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
}
时间复杂度 O(n)

从第 2 间算到第 n-1 间，循环一遍。

空间复杂度 O(1)

只保留最近两个值，不需要整张 dp 表。

🚀 小结

套路：打家劫舍是「选还是不选」型一维 DP 的代表题—— 每个位置只有两种互斥选择，把它们的结果取较大值，就是这个位置的最优解。 它和爬楼梯一样，状态转移只依赖「最近两个值」，所以也能把空间从 O(n) 压到 O(1)。 这套「相邻不能同选」的思路后面还会反复出现（环形排列、二叉树上的打家劫舍等变体）。
