选课要先修课，先修课又要先修课……这是一道经典的「有向图找环」题。 看完你会明白：拓扑排序怎么用「入度 + 队列」一边剥洋葱一边判断能不能学完。

🧩 第一步：题目到底要什么

有 numCourses 门课，编号 0 到 numCourses - 1。 prerequisites[i] = [a, b] 表示「想学 a，必须先学完 b」，就跟大学选课一样：想上「数据结构」，得先修完「编程基础」。 题目问：给你这么一堆先修要求，能不能安排出一个学习顺序，把所有课都学完？

numCourses2
prerequisites[[1,0],[0,1]]

学 1 之前要先学 0，学 0 之前又要先学 1——两门课互相等对方，谁都先学不了， 这就是「环」，答案是 false。

🔑 第二步：把「先修关系」看成一张有向图

先把「课程」和「先修关系」翻译成一种更好处理的形式——图。 图就是「一堆点 + 连接它们的线」，你每天都在用它的例子：微信好友关系是一张图 （人是点，好友关系是线），地图上的城市和公路也是一张图（城市是点，公路是线）。

这道题里，每门课就是图上的一个节点；[a, b] 这条先修关系， 就画成一条从 b 指向 a 的箭头（写作 b → a），表示「b 要先学完，才能解锁 a」。因为这条线是有方向的 （b → a 和 a → b 含义完全不同），这种图就叫有向图。

翻译成图之后，问题就变成了一个很经典的图论问题—— 这张有向图里有没有「环」？「环」就是顺着箭头一路走，能绕回到出发点， 比如例子里 0 → 1 → 0。只要没有环，就一定能把所有课沿着箭头排出一个先后顺序， 让每门课都排在它的先修课后面（这个顺序在图论里有个专门的名字，叫拓扑排序）； 只要有环，环上的课就会互相等对方先学完，永远轮不到谁，谁也学不了。

怎么找环：给每个节点算一个入度——也就是有几条箭头指向它， 等价于「它还差几门先修课没学完」。入度为 0 的课不需要等任何人，可以马上学。 学完一门课后，把它指向的每个邻居（箭头对面的课）入度减一；如果某个课的入度被减到 0， 说明它的先修课全学完了，终于轮到它了。

⚡ 第三步：用「队列 + 入度」做拓扑排序（Kahn 算法）

这里要用到一个叫队列的小工具：它就像排队买票，谁先排进去就先被叫到 （先进先出）。我们把「暂时不用等任何先修课」的课程，依次放进这个队伍里， 谁排在前面就先处理谁。

具体做法：先把所有入度为 0 的课塞进队列；每次从队列最前面取出一门课， 标记为「已学完」，然后把它指向的每个邻居入度减一——谁的入度降到 0 就排进队尾， 继续重复。这其实就是一种叫广度优先搜索（BFS）的走法——不是顺着一条线 一路走到底，而是像水波纹一样，一层能学的课学完了，再解锁下一层能学的课， 一层一层往外扩散。整个过程也像剥洋葱：每一轮剥掉一层「当前不依赖任何人」的课程。

最后看一共「学完」了几门课：如果等于 numCourses，说明所有课都被剥光了， 能学完；如果队列提前空了，还剩一些课的入度始终降不到 0， 说明它们困在环里、互相等对方，永远轮不到——返回 false。

当前正在处理的课 在队列中（待处理） 已学完 正在检查的邻居 卡死在环里（学不完）

试试把先修关系改成 [[1,0],[0,1]]（numCourses 改成 2）， 看队列怎么提前清空、剩下的课怎么被标红成环。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean canFinish(int numCourses, int[][] prerequisites) {
        // adj[b]：列出「学完 b 之后，能解锁哪些课」
        List<List<Integer>> adj = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) adj.add(new ArrayList<>());

        int[] indeg = new int[numCourses]; // 每门课还差几门先修课
        for (int[] p : prerequisites) {
            int a = p[0], b = p[1]; // 学 a 之前要先学 b
            adj.get(b).add(a);
            indeg[a]++;
        }

        // 入度为 0 的课先入队：不需要等任何先修课
        Deque<Integer> queue = new ArrayDeque<>();
        for (int i = 0; i < numCourses; i++) {
            if (indeg[i] == 0) queue.offer(i);
        }

        int finished = 0; // 已经学完的课程数
        while (!queue.isEmpty()) {
            int cur = queue.poll();
            finished++;
            for (int next : adj.get(cur)) {
                if (--indeg[next] == 0) { // 先修课全学完了，该轮到它
                    queue.offer(next);
                }
            }
        }

        return finished == numCourses; // 全部学完才说明没有环
    }
}
时间复杂度 O(V + E)

V 是课程数，E 是先修关系数：建图、每条边、每个节点都只处理一次。

空间复杂度 O(V + E)

邻接表存所有边，外加入度数组和队列，最坏 O(V + E)。

🚀 小结

套路：遇到「A 依赖 B」「先做 B 才能做 A」这类先后关系， 先想到画一张有向图，再用拓扑排序（入度 + 队列） 一边剥洋葱一边判断有没有环。能拓扑排序完 ⇔ 无环 ⇔ 能按顺序完成； 排不完 ⇔ 有环 ⇔ 互相卡死。课程表、任务调度、编译依赖都是这个套路。
