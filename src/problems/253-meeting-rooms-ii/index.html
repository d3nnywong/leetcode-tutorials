上一题（252 会议室）只问「能不能」，这题更进一步：要凑齐最少几间会议室， 才能把所有会议都安排下去。答案藏在一个「最小堆」里。

🧩 第一步：题目到底要什么

给你一堆会议时间 [start, end)，这次不是判断「行不行」，而是要算出 同一时间最多有几场会议在重叠——因为重叠的会议必须分到不同房间， 重叠最多的那一刻，就决定了你至少要准备多少间房。

输入[[0,30],[5,10],[15,20]]

[0,30] 同时跟 [5,10]、[15,20] 重叠（但 [5,10] 和 [15,20] 互不重叠）， 最多只在某一刻同时撞上 2 场 → 答案是 2。

🐢 第二步：从「判断冲突」到「数房间」

252 题里我们学到：把会议按开始时间排序，重叠一定发生在「正在进行中的会议」和 「新来的会议」之间。这题的区别是，重叠可能不止两两一对，而是好几场会议同时占用着房间。

在往下讲之前，先认识一个新工具——堆（heap）。你可以把它想象成一个 「会自动把最小的顶上来」的箱子：不管你往里面扔进多少个数字、扔的顺序多乱， 它随时都能立刻告诉你「现在箱子里最小的是哪个」，这一步叫看堆顶， 几乎不花什么代价；如果你把这个最小的拿走，或者再往箱子里扔一个新数字， 它会自己悄悄调整一下顺序，保证你下次问「最小的是谁」还是能马上答上来， 而且这个调整也很轻松（不用把箱子里的东西全部重新排一遍）。这种「只关心最小值、 还能随时进出」的箱子，就叫最小堆。

关键洞察：按开始时间从早到晚处理每场会议时，我们只关心一件事—— 当前正在使用的房间里，最早会空出来的那一间，它的结束时间是多少。 如果它已经 ≤ 新会议的开始时间，这间房就能直接复用；否则所有房间都还占着， 只能新开一间。每次只需要看「最早空出来的房间」，而不用挨个比较所有房间—— 这正是最小堆最擅长的事：把每个正在使用中的房间的「结束时间」都扔进堆里， 堆顶自然就是最早会空出来的那一间，一眼就能看到，完全不用自己去找。

⚡ 第三步：维护一个「结束时间」最小堆

做法：先把会议按开始时间排序；用一个最小堆 heap 记录每个正在使用中的 房间「什么时候会空出来」（也就是结束时间）。依次处理每场会议 cur：

  • 看一眼堆顶 heap.peek()（这一步只是「看一眼」最早会空出来的房间， 并不会把它拿走）。如果 heap.peek() <= cur.start，说明那间房已经空出来了， 就弹出堆顶 heap.poll()（这次是真的把它从堆里取走），把它腾给 cur 用。
  • 不管复用没复用，都把 cur.end 压入堆 heap.offer()（塞一个新数字进去，堆会自动把最小的重新顶上来）—— 能复用就是「替换」堆顶（房间数不变），不能复用就是「新增」一个房间（房间数 +1）。

扫完所有会议后，堆里还剩多少个元素，就代表同一时刻最多同时开着多少间房， 也就是题目要的答案。

当前正在处理的会议 已分配房间的会议 复用了空出来的房间 新开的房间 / 最终房间数

试试改成 7,10;2,4，看两场会议完全不挨着，自始至终只用 1 间房； 再试试 1,5;2,6;3,7，看三场会议两两重叠，需要开到 3 间房。

☕ 第四步：写成 Java 代码

class Solution {
    public int minMeetingRooms(int[][] intervals) {
        // 按开始时间从小到大排序
        Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

        // 小顶堆：保存正在使用中的房间的「结束时间」，堆顶 = 最早空出来的房间
        PriorityQueue<Integer> heap = new PriorityQueue<>();

        for (int[] cur : intervals) {
            // 堆顶房间的结束时间 <= 当前会议的开始时间 → 已经空出来了，弹出复用
            if (!heap.isEmpty() && heap.peek() <= cur[0]) {
                heap.poll();
            }
            // 能复用就是替换堆顶（房间数不变）；不能复用就是新增一个房间
            heap.offer(cur[1]);
        }

        return heap.size(); // 堆的大小 = 同一时刻最多需要的房间数
    }
}
时间复杂度 O(n log n)

排序 O(n log n)；n 场会议每次堆操作 O(log n)。

空间复杂度 O(n)

最坏情况下所有会议互相重叠，堆里要存下 n 个结束时间。

🚀 小结

套路：遇到「区间，求同一时刻最多重叠多少个」的题，按开始时间排序后， 维护一个按结束时间排序的最小堆来代表「正在占用的资源」——堆顶就是 最快被释放的那一个，每来一个新区间只需要跟堆顶比一次。这比 252 题的「相邻比较」 更进一步：从只回答「有没有冲突」，升级成了「同时冲突的最大数量是多少」。
