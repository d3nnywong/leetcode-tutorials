数一数字符串里有多少个回文子串。「回文」就是正着读和倒着读完全一样的一段文字， 比如 "aba"、"abba"（有点像「上海自来水来自海上」这种回文句， 只不过我们这里只关心字符本身）。回文串有一个很重要的性质——每一个回文串都有唯一的「中心」， 从中心往两边看，字符总是一对一对相同的。这道题的关键洞察正来自这一点： 与其去检查每个子串是不是回文，不如反过来——从每个可能的中心向两边扩展，看能扩多远。

🧩 第一步：题目到底要什么

给你一个字符串 s，统计它一共包含多少个回文子串—— 也就是「这一段本身正读反读都一样」的子串。子串必须是连续的一段 （不能跳着挑字符：比如 "abc" 的子串只有 "a" "b" "c" "ab" "bc" "abc"， "ac" 中间跳过了 b，不算子串）； 同样的内容出现在不同位置要分别计数 （比如 "aaa" 里两个独立的 "a" 算两个，位置不同，虽然长得一样）。

s"aaa"

回文子串一共 6 个："a"、"a"、"a"、 "aa"、"aa"、"aaa"。

🐢 第二步：笨办法，以及更聪明的角度

最直接的想法：枚举所有 O(n²) 个子串（子串有「从哪开始」和「多长」两个维度， 组合起来数量级就是 n²），对每个子串再花 O(n) 去判断它是不是回文——具体做法是放两个「指针」（指针在这里就是两个记录位置的下标， 你可以把它们想象成两根手指，一根按在这段的最左边，一根按在最右边）， 两根手指同时一步步往中间靠拢，边靠边比较两边的字符是否相同，只要有一步不相同就不是回文。 三层加起来合计 O(n³)。n 能到 1000，十亿次运算起步，太慢。

换个角度想：与其问「这段子串是不是回文」，不如反过来问 「以这个位置为中心，能往两边扩出多长的回文」。 一个回文串永远关于它的正中心对称——把它对折起来，两边应该完全重合。 这个「中心」可能是一个字符（奇数长度，比如 "aba" 对折点正是中间的 b，两边各留下一个 a）， 也可能是两个字符之间的缝隙 （偶数长度，比如 "abba" 对折点在两个 b 之间的缝隙里， 两边刚好是镜像的 ab 和 ba）。 长度为 n 的字符串正好有 2n − 1 个这样的中心 （n 个字符本身 + n−1 个字符之间的缝隙）。

⚡ 第三步：从每个中心向外扩展

对每个中心摆好两个指针 L、R（单字符中心一开始 L 和 R 按在同一个字符上； 缝隙中心 L、R 分别按在缝隙左右相邻的两个字符上）， 只要 s[L] == s[R]（这两个位置上的字符相等），这一段就是回文， 计数 +1，然后两根手指同时往外挪一格（L−1、R+1），再比一次； 一旦某根手指要挪到字符串范围之外（也就是「越界」，比如 L 已经小于 0， 或者 R 已经不小于字符串长度），或者两边字符不相等， 就说明从这个中心出发不能再扩大了，停下来换下一个中心继续。把 2n − 1 个中心都这样扫一遍，每个中心最多扩 O(n) 步，合计 O(n²)。

当前中心（固定不动） 正在比较的左右指针 L / R 确认命中，计入回文子串 不相等，扩展到此为止

试试把字符串改成 aaa 或 abba，对比奇数中心和偶数中心分别怎么扩展。

☕ 第四步：写成 Java 代码

class Solution {
    public int countSubstrings(String s) {
        int n = s.length();
        int count = 0;
        // center 编号 0..2n-2：偶数是单字符中心，奇数是两字符之间的中心
        for (int center = 0; center <= 2 * n - 2; center++) {
            int l = center / 2;
            int r = l + center % 2;
            // 两边相等就计数，再继续往外扩；越界或不相等就停
            while (l >= 0 && r < n && s.charAt(l) == s.charAt(r)) {
                count++;
                l--;
                r++;
            }
        }
        return count;
    }
}
时间复杂度 O(n²)

2n − 1 个中心，每个中心最多扩展 O(n) 步。

空间复杂度 O(1)

只用了几个指针变量，没有额外数组。

🚀 小结

套路：回文问题经常用「枚举中心，向两边扩展」来代替「枚举区间，逐个验证」—— 因为回文天生关于中心对称，扩展过程本身就在验证。这个思路还能进一步和 动态规划（一种把大问题拆成一个个小问题、并把每个小问题算出的答案存进一张表里 以后直接查表、不用重算的方法）结合：把「从某个中心/某个区间出发是不是回文」这件事存进表里， 就能升级成「最长回文子串」（5 题）的解法。
