数一数字符串里有多少个回文子串。关键洞察是：每个回文串都有唯一的「中心」， 与其去检查每个子串是不是回文，不如反过来——从每个可能的中心向两边扩展，看能扩多远。

🧩 第一步：题目到底要什么

给你一个字符串 s，统计它一共包含多少个回文子串。 子串必须是连续的一段；同样的内容出现在不同位置要分别计数 （比如 "aaa" 里两个独立的 "a" 算两个）。

s"aaa"

回文子串一共 6 个："a"、"a"、"a"、 "aa"、"aa"、"aaa"。

🐢 第二步：笨办法，以及更聪明的角度

最直接的想法：枚举所有 O(n²) 个子串，对每个子串再花 O(n) 去判断它是不是回文（左右指针往中间夹），合计 O(n³)。n 能到 1000， 十亿次起步，太慢。

换个角度想：与其问「这段子串是不是回文」，不如反过来问 「以这个位置为中心，能往两边扩出多长的回文」。 一个回文串永远关于它的正中心对称——可能是一个字符（奇数长度，比如 "aba" 中心是 b），也可能是两个字符之间的缝隙 （偶数长度，比如 "abba" 中心在两个 b 之间）。 长度为 n 的字符串正好有 2n − 1 个这样的中心。

⚡ 第三步：从每个中心向外扩展

对每个中心摆好左右指针 L、R（单字符中心 L = R；缝隙中心 L、R 相邻）， 只要 s[L] == s[R]，这一段就是回文，计数 +1， 然后继续往外扩一格（L−1、R+1）；一旦越界或者两边字符不相等， 说明从这个中心出发不能再扩了，换下一个中心继续。把 2n − 1 个中心都扫一遍，每个中心最多扩 O(n) 步，合计 O(n²)。

当前中心（固定不动） 正在比较的左右指针 L / R 确认命中，计入回文子串 不相等，扩展到此为止

试试把字符串改成 aaa 或 abba，对比奇数中心和偶数中心分别怎么扩展。

☕ 第四步：写成 Java 代码

class Solution {
    public int countSubstrings(String s) {
        int n = s.length();
        int count = 0;
        // center 编号 0..2n-2：偶数是单字符中心，奇数是两字符之间的中心
        for (int center = 0; center <= 2 * n - 2; center++) {
            int l = center / 2;
            int r = l + center % 2;
            // 两边相等就计数，再继续往外扩；越界或不相等就停
            while (l >= 0 && r < n && s.charAt(l) == s.charAt(r)) {
                count++;
                l--;
                r++;
            }
        }
        return count;
    }
}
时间复杂度 O(n²)

2n − 1 个中心，每个中心最多扩展 O(n) 步。

空间复杂度 O(1)

只用了几个指针变量，没有额外数组。

🚀 小结

套路：回文问题经常用「枚举中心，向两边扩展」来代替「枚举区间，逐个验证」—— 因为回文天生关于中心对称，扩展过程本身就在验证。同样的思路稍加一个数组记忆化， 就能升级成「最长回文子串」（5 题）的二维动态规划解法。
