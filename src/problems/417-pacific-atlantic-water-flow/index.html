一座岛，左边和上边挨着太平洋，右边和下边挨着大西洋。雨水往低处（或一样高）流， 哪些格子的水两个海洋都能流到？正着模拟很慢， 反过来从海岸线往高处逆流，一次扫描就能求出答案。

🧩 第一步：题目到底要什么

给你一个 m × n 的高度矩阵 heights，代表一座岛。 「矩阵」听着吓人，其实就是一张网格表：一共 m 行、 n 列，每个小格子里填一个数字，代表这块地方的海拔高度。 我们用 (r, c) 表示「第 r 行、第 c 列」的那个格子（行号列号都从 0 开始数）。

岛的左边界和上边界挨着太平洋， 右边界和下边界挨着大西洋。 雨水可以从一个格子流向上下左右相邻、且高度 小于或等于自己的格子（水往低处流，一样高也能流；不能斜着流，也不能往高处流）。 紧挨着海洋的格子，水可以直接流进海洋。

问：哪些格子的雨水，顺着流，既能流到太平洋，又能流到大西洋？ 把这些格子的坐标 [r, c] 都找出来。

heights[[1,2,2,3,5],[3,2,3,4,4],[2,4,5,3,1],[6,7,1,4,5],[5,1,1,2,4]]

比如格子 (0,4)（第 0 行、第 4 列）高度是 5，本身就贴着太平洋（第 0 行） 和大西洋（最后一列），两边都能直接流到 → 它是答案之一。

🐢 第二步：直接模拟为什么慢

最直接的想法：对每一个格子，都从它自己出发，往外「扩散」一遍， 顺着「能流向更低或相等的邻居」不断往外摸，看看摸到的范围能不能同时碰到 第 0 行/第 0 列（太平洋）和最后一行/最后一列（大西洋）。

这种「从一个点出发，把能到达的格子都摸一遍」的动作，专业上叫搜索， 常见有两种摸法：DFS（深度优先搜索）——像走迷宫一样， 咬定一个方向一条路走到黑，走不通了再退回来换方向； BFS（广度优先搜索）——像水波纹一样，从起点开始 一圈一圈地往外扩散，先摸完最近的一圈，再摸下一圈。 两种摸法最终能摸到的格子范围是一样的，只是走的顺序不同。

慢在哪：网格有 m × n 个格子，每个格子都要做一次最坏 O(m × n) 的搜索，总复杂度 O((m × n)²)。 200 × 200 的网格能到 16 亿次级别——本质问题是：我们对「水流方向」 正着想，每个格子都要重新走一遍完整的路，没有任何复用。

⚡ 第三步：反过来想——从海岸线倒着「逆流」

关键洞察：与其对每个格子分别判断「它能不能流到海」，不如反过来—— 直接从海岸线出发，找「谁能流到我这里」。水从高处流到低（或相等）处， 那反过来从低处「爬」到高（或相等）处，正好就是「正着能流过来」的所有格子。

于是：把第 0 行和第 0 列（挨着太平洋的格子）一起当成 起点——这就是所谓的「多源」，意思是同时有好多个出发点，而不是只有一个。 做一次 BFS：先把所有起点放进一个待处理名单（专业叫「队列」， 规则跟排队买票一样，先放进去的先被拿出来处理），每次从名单里取一个格子出来， 看它上下左右的邻居——只要邻居高度 ≥ 当前格子（意味着水能从邻居那 「倒流」到这个更低或相等的格子），就把邻居标记为「能到达」并塞进名单等着处理。 名单空了，说明能摸到的格子都摸完了——这些就是「能流到太平洋」的格子。 再把最后一行和最后一列（挨着大西洋的格子）作为起点， 照样做一次，得到「能流到大西洋」的格子。 最后两边都摸到的格子，就是答案。

当前出队处理的格子 能流到太平洋（蓝色，左/上有蓝色描边的是海岸） 能流到大西洋（绿色，右/下有绿色描边的是海岸） 两边都能流到 = 答案

试试把矩阵改成官方示例 2 的 1（单格小岛，两边都能直接流到）， 或者改小一点的 1,2;1,2，更容易跟着每一步看清楚「逆流」的方向。

☕ 第四步：写成 Java 代码

上面讲思路时说的是「BFS + 队列」，但代码里为了简洁，换成了效果完全等价的 递归写法：一个方法在执行到一半时，又调用了自己 去处理下一个格子，像俄罗斯套娃一样一层套一层——处理到「摸不到新格子」自然就停下， 再一层层把结果传回去。下面代码里的 dfs 方法就是这样： 它标记完当前格子，就对每个符合条件的邻居调用一次自己，直到把这一片能摸到的格子都标完为止。 最终标出来的范围，和「一圈圈往外摸」的 BFS 是完全一样的，只是走的顺序不同。 

class Solution {
    private int[][] heights;
    private int rows, cols;
    private static final int[][] DIRS = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

    public List<List<Integer>> pacificAtlantic(int[][] heights) {
        this.heights = heights;
        rows = heights.length;
        cols = heights[0].length;
        boolean[][] pacific = new boolean[rows][cols];
        boolean[][] atlantic = new boolean[rows][cols];

        // 第 0 行/最后一行：分别贴着太平洋 / 大西洋
        for (int c = 0; c < cols; c++) {
            dfs(0, c, pacific);
            dfs(rows - 1, c, atlantic);
        }
        // 第 0 列/最后一列：分别贴着太平洋 / 大西洋
        for (int r = 0; r < rows; r++) {
            dfs(r, 0, pacific);
            dfs(r, cols - 1, atlantic);
        }

        List<List<Integer>> result = new ArrayList<>();
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (pacific[r][c] && atlantic[r][c]) {
                    result.add(Arrays.asList(r, c));
                }
            }
        }
        return result;
    }

    // 从海岸线格子 (r, c) 出发「逆流」：邻居高度 >= 当前格子就说明水能从那倒流过来
    private void dfs(int r, int c, boolean[][] visited) {
        visited[r][c] = true;
        for (int[] d : DIRS) {
            int nr = r + d[0], nc = c + d[1];
            if (nr < 0 || nr >= rows || nc < 0 || nc >= cols || visited[nr][nc]) continue;
            if (heights[nr][nc] < heights[r][c]) continue; // 比当前还低，水流不过来，跳过
            dfs(nr, nc, visited);
        }
    }
}
时间复杂度 O(m × n)

太平洋和大西洋各做一次搜索，每个格子在每次搜索里最多被访问一次。

空间复杂度 O(m × n)

两个 visited 布尔矩阵，加上递归（或队列）最坏 O(m × n) 的深度。

🚀 小结

套路：当「从每个点出发判断能否到达终点」要做 n 次重复搜索时， 想想能不能反过来，从终点（或多个终点）出发，一次多源 BFS/DFS 把所有「能到达」的点标出来。这道题的两次反向 BFS，加上动画里看到的「双层标记取交集」， 是网格连通性问题里非常通用的一个技巧。
