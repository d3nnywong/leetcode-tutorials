链表只能从头往后走，没法像数组那样直接「首尾互换」。 看完你会明白：用 prev、cur 两个指针一趟扫过去，把每个结点的 next 箭头逐个掉头，链表就反过来了。

🧩 第一步：题目到底要什么

给你一个单链表的头结点 head，要你把整条链表的方向反过来， 并返回反转后链表的新头结点。

head[1, 2, 3, 4, 5]

原来是 1 → 2 → 3 → 4 → 5 → null，反转后变成 5 → 4 → 3 → 2 → 1 → null。新的头结点是原来的最后一个结点。

如果是数组，反转很简单：开两个下标从两头往中间交换值就行。但链表不一样—— 它没有「下标」，每个结点只知道自己的下一个是谁（next 指针）， 不知道自己的上一个是谁。要反转，本质上就是要把每个结点的 next 指针，从「指向下一个」改成「指向上一个」。

🐢 第二步：直接改指针会遇到的坑

想法很直接：把 cur.next 改成指向前一个结点 prev 不就行了？ 但试着写一下就会发现一个坑——

坑在哪：一旦执行 cur.next = prev， cur 原来的下一个结点的地址就丢了—— 因为整条链表只能靠 next 指针才能往后走，指针一改， 后面的结点就「断联」了，没法继续往下处理。

关键洞察：在改指针之前，先用一个临时变量 next 把 cur.next 偷偷存起来，改完指针之后再用这个 next 继续往后走，就不会丢链了。

⚡ 第三步：prev、cur 双指针一趟扫描

用两个指针 prev（一开始是 null）和 cur （一开始是 head）从头到尾扫一遍，每到一个结点重复同样的三个动作：

  • ① 暂存：next = cur.next，先记下原来的下一个结点。
  • ② 掉头：cur.next = prev，让当前结点指向它原来的前一个。
  • ③ 前移：prev = cur，cur = next，两个指针一起往后挪一位。

重复这三步，直到 cur 变成 null（说明已经走完整条链表）。 这时 prev 正好停在原来的最后一个结点上 —— 它就是反转后的新头结点，直接返回它。

prev 指针 / 已反转的箭头 cur 指针（当前处理的结点） 已经掉头完成的结点

试试把链表改成只有 1 一个结点，或者留空表示空链表 []， 看看边界情况下动画怎么处理。

☕ 第四步：写成 Java 代码

下面给出最常用的迭代写法，以及思路一致的递归写法。

// Definition for singly-linked list.
class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

class Solution {
    // 写法一：迭代，O(n) 时间、O(1) 额外空间
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode cur = head;

        while (cur != null) {
            ListNode next = cur.next; // ① 先暂存下一个结点
            cur.next = prev;             // ② 把当前结点的指针掉头
            prev = cur;                  // ③ prev、cur 一起前移
            cur = next;
        }
        return prev; // cur 为 null 时，prev 正好停在新头结点上
    }

    // 写法二：递归，O(n) 时间、O(n) 递归栈空间
    public ListNode reverseListRecursive(ListNode head) {
        if (head == null || head.next == null) {
            return head; // 空链表或只剩一个结点，它自己就是新头结点
        }
        ListNode newHead = reverseListRecursive(head.next); // 先反转后面的部分
        head.next.next = head; // 让后面那个结点反过来指向自己
        head.next = null;     // 自己变成新的尾部，断开原来的 next
        return newHead;
    }
}
时间复杂度 O(n)

不管迭代还是递归，链表都只需要遍历一遍。

空间复杂度 迭代 O(1) / 递归 O(n)

迭代只用了 prev、cur、next 三个指针；递归会用掉跟链表长度等长的调用栈。

🚀 小结

套路：链表上要「反转方向」，记住三步口诀—— 先存下一个、再掉头当前、最后一起前移。 用 prev 记住走过的路，用临时变量 next 保住没走的路， 就不会出现「指针一改、后面全丢」的坑。这套 prev/cur 双指针思路， 后面反转链表的一部分、按组反转、回文链表等题都会反复用到。
