这是大多数人接触「二维动态规划」的第一题。别怕，我们一步步来， 看完你不仅会做这一题，还会真正理解「动态规划」到底在干什么。

🧩 第一步：先搞懂「子序列」

很多人卡在第一关——把「子序列」和「子串」搞混了。它们差别很大：

子串（连续）

必须是连续的一段，中间不能断。

在 abcde 里：
bcd ✅ 是子串
ace ❌ 不是子串（不连续）

子序列（可跳过）

按原来的先后顺序挑字符， 中间可以跳过一些，但不能打乱顺序。

在 abcde 里：
ace ✅ 是子序列（跳过 b、d）
aec ❌ 不是（顺序乱了）

公共子序列：一个序列，它同时是 text1 和 text2 的子序列。我们要找的就是其中最长的那个的长度。

🐢 第二步：先想最笨的办法（暴力法）

最直接的想法：把 text1 的所有子序列都列出来， 再一个个检查是不是 text2 的子序列，取最长的。

问题在哪？一个长度为 n 的字符串，每个字符都有「选 / 不选」两种可能， 一共有 2ⁿ 个子序列。n 才 30，就超过 10 亿了—— 电脑会算到天荒地老。

关键转折：暴力法慢，是因为它把同一个子问题 算了无数遍。动态规划的核心思想就是—— 把每个子问题只算一次，把答案记下来重复利用。

🧠 第三步：动态规划三件套

所有二维 DP 题，都可以套这三个问题来思考。我们一个个填空：

1

状态：dp[i][j] 表示什么？

dp[i][j] = text1 的前 i 个字符， 和 text2 的前 j 个字符，这两段的最长公共子序列长度。

想求的最终答案，就是 dp[m][n]（m、n 是两个字符串的总长度）。

2

转移：dp[i][j] 怎么由更小的问题推出来？

只看正在比较的两个字符 text1[i-1] 和 text2[j-1]：

  • 如果相等：这个字符可以接在「双方都去掉它」的那段 公共子序列后面，长度 +1：
    dp[i][j] = dp[i-1][j-1] + 1
  • 如果不等：这个字符至少有一个用不上， 那就在「丢掉 text1 这个字符」和「丢掉 text2 这个字符」里取更大的：
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])
3

边界：最小的情况是什么？

当 i = 0 或 j = 0，意味着其中一个字符串是空的。 空串和任何字符串都没有公共子序列，所以：
dp[0][j] = 0，dp[i][0] = 0 （表格的第一行、第一列全是 0）。

🎬 第四步：亲手看它一格一格算出来

光看公式还是抽象。下面这张表你可以亲自点—— 看每一格是怎么根据「左上 / 上方 / 左方」的格子算出来的。 建议先用默认的 abcde 和 ace 走一遍。

当前正在算 它依赖的来源格 最终 LCS 路径

☕ 第五步：写成 Java 代码

把上面的三件套翻译成代码，几乎就是一一对应：

class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length(), n = text2.length();

        // dp[i][j]: text1 前 i 个字符 与 text2 前 j 个字符 的 LCS 长度
        // 多开一行一列表示「空串」，它们默认就是 0（边界）
        int[][] dp = new int[m + 1][n + 1];

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                // 注意：第 i 个字符的下标是 i-1
                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    // 字符相等：左上角 + 1
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    // 字符不等：上方、左方取最大
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }
        return dp[m][n]; // 右下角就是答案
    }
}
时间复杂度 O(m × n)

每个格子只算一次，一共 m×n 个格子。

空间复杂度 O(m × n)

用了一张 m×n 的二维表。下面可以优化掉。

🚀 进阶：把空间压到 O(n)

观察上面的表格动画你会发现：算第 i 行时，只用到了第 i-1 行。 既然更早的行用不上，那就没必要存整张表，用一维数组滚动即可。

唯一的坑：相等时要用「左上角」的旧值，所以更新前先用一个变量 prev 把它存住。 

public int longestCommonSubsequence(String text1, String text2) {
    int m = text1.length(), n = text2.length();
    int[] dp = new int[n + 1];

    for (int i = 1; i <= m; i++) {
        int prev = 0; // prev 相当于二维里的 dp[i-1][j-1]（左上角）
        for (int j = 1; j <= n; j++) {
            int temp = dp[j]; // 更新前先存住，给下一列当左上角
            if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                dp[j] = prev + 1;
            } else {
                dp[j] = Math.max(dp[j], dp[j - 1]);
            }
            prev = temp;
        }
    }
    return dp[n];
}
小结：遇到二维 DP，先按「状态 → 转移 → 边界」想清楚， 老老实实写出二维版本；能 AC 之后，再观察「每行只依赖上一行」来做空间优化。 顺序别反，先正确，再优化。
