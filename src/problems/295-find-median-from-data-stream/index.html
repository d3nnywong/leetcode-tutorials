数据像流水一样一个个进来，还要随时能报出「目前为止的中位数」。 看完你会明白：两个堆顶顶在一起，怎么把插入做到 O(log n)、取中位数做到 O(1)。

🧩 第一步：题目到底要什么

要你设计一个结构，支持源源不断地 addNum(num) 往里加数， 并且随时可以调用 findMedian() 拿到目前为止所有数的中位数。 中位数就是把所有数从小到大排好后，正中间的那个数；个数是偶数时，是中间两个数的平均值。

操作addNum(1) → addNum(2) → findMedian()

目前的数是 [1, 2]，排好序中间两个数是 1 和 2，中位数 = (1 + 2) / 2 = 1.5。再 addNum(3) 后变成 [1, 2, 3]，中位数就是正中间的 2.0。

🐢 第二步：先想想笨办法，再看为什么要用堆

最直接的想法：每来一个数就插入一个有序数组（或者全存下来， 每次 findMedian 现场排序），中位数直接看中间位置的值。

慢在哪：往有序数组中间插入一个数，后面的元素都要整体挪动，是 O(n)；现场排序更是每次 O(n log n)。题目最多调用 5 × 10⁴ 次，数一多就很容易超时。

关键洞察：我们并不需要整个数组都有序，只需要随时知道「比中间小的那一半」 里的最大值、和「比中间大的那一半」里的最小值——这正好是两个堆的堆顶能做的事： 一个大顶堆存较小的一半（堆顶是这一半的最大值），一个小顶堆 存较大的一半（堆顶是这一半的最小值）。这种「两堆顶在中间」的结构，俗称对顶堆。

⚡ 第三步：对顶双堆，怎么插入、怎么找中位数

维护两个堆，并始终让两边数量最多相差 1：

  • 大顶堆（max-heap）：存「较小的一半」，堆顶是这一半里最大的数。
  • 小顶堆（min-heap）：存「较大的一半」，堆顶是这一半里最小的数。

每来一个新数 num：先和大顶堆堆顶比较，更小（或大顶堆为空） 就放进大顶堆，否则放进小顶堆。放完后如果两堆数量差超过 1，就把数量较多那堆的堆顶 挪给较少的那堆，重新拉平到相差不超过 1。

这样大顶堆堆顶永远是「下半区」最大值、小顶堆堆顶永远是「上半区」最小值——两堆数量相等时， 中位数就是两个堆顶的平均值；数量相差 1 时，中位数就是数量较多那堆的堆顶。插入是 O(log n)（堆的上浮/下沉），取中位数是 O(1)（直接看堆顶）。

刚插入的新数 当前堆顶 再平衡时被挪动的数 决定中位数的堆顶

默认演示依次对 5, 2, 8, 1, 9, 3 调用 addNum，每次之后都求一次中位数。 试试把序列改成 1,2,3,4,5，看两堆怎么始终保持「左堆最多比右堆多 1 个」。

☕ 第四步：写成 Java 代码

class MedianFinder {
    // 大顶堆：存较小的一半，堆顶是这一半里的最大值
    private PriorityQueue<Integer> small =
        new PriorityQueue<>(Collections.reverseOrder());
    // 小顶堆：存较大的一半，堆顶是这一半里的最小值
    private PriorityQueue<Integer> large = new PriorityQueue<>();

    public MedianFinder() {
    }

    public void addNum(int num) {
        // 先决定新数进哪个堆：不大于大顶堆堆顶（或大顶堆为空）就进大顶堆
        if (small.isEmpty() || num <= small.peek()) {
            small.offer(num);
        } else {
            large.offer(num);
        }
        // 再平衡：让两堆数量差不超过 1
        if (small.size() - large.size() > 1) {
            large.offer(small.poll());
        } else if (large.size() - small.size() > 1) {
            small.offer(large.poll());
        }
    }

    public double findMedian() {
        if (small.size() == large.size()) {
            // 数量相等：中位数是两堆堆顶的平均值
            return (small.peek() + large.peek()) / 2.0;
        }
        // 数量较多的那堆堆顶就是中位数
        return small.size() > large.size() ? small.peek() : large.peek();
    }
}

/**
 * Your MedianFinder object will be instantiated and called as such:
 * MedianFinder obj = new MedianFinder();
 * obj.addNum(num);
 * double param_2 = obj.findMedian();
 */
时间复杂度 addNum O(log n)，findMedian O(1)

插入堆要做上浮/下沉，最坏 O(log n)；找中位数只看堆顶，O(1)。

空间复杂度 O(n)

两个堆加起来要存下所有插入过的数。

🚀 小结

套路：当你只关心一组数据里「中间附近」的值，而不需要整体有序时， 用对顶堆把数据一分为二，靠两个堆顶随时反映「中间状态」—— 这比维护整体有序便宜得多。这个套路还能推广到滑动窗口中位数、Top-K 这类只关心局部极值的问题。
