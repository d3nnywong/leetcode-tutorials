一道经典的「滑动窗口」题：窗口要扩张到多大，才能用最多 k 次替换把窗口里的字符都变成同一个？ 看完你会明白，答案藏在一个不起眼的小技巧里——maxCount 只增不减。

🧩 第一步：题目到底要什么

给你一个只含大写字母的字符串 s，你有 k 次「魔法」机会， 每次可以把字符串里任意一个字符改成任意其他大写字母。 用最多 k 次魔法之后，整个字符串里能拼出的「同一个字母连续重复」的最长一段有多长？

s"AABABBA"
k1

把中间那个 'A' 改成 'B'，字符串变成 "AABBBBA"， 其中 "BBBB" 是连续 4 个相同字母 → 答案 4。

注意：题目要的是「替换之后」的最长子串（连续的一段），不是子序列； 而且我们不需要真的把字符串改出来，只要算出这个最长长度即可。

🐢 第二步：从暴力到「窗口要满足的条件」

最直接的想法：枚举所有子串 s[l..r]，数一数里面出现次数最多的字母有几个 （记作 maxCount），剩下 (r − l + 1) − maxCount 个字符就是 需要替换成那个最多字母的数量。如果这个数量 ≤ k，这个子串就「合法」 （能在 k 次内变成全同一字母）。

慢在哪：枚举所有子串是 O(n²)，每个子串再数一遍字母出现次数又是 O(n)，总共 O(n³)（哪怕优化到边扩展边统计也是 O(n²)）。 数组长度能到 10⁵，这样肯定超时。

关键洞察：判断一个窗口合不合法只需要一个公式—— 窗口长度 − maxCount ≤ k。 于是想法变成：用一个滑动窗口 [left, right]， 一边向右扩张一边维护这个公式，不合法就收缩左边界，全程只扫一遍。

⚡ 第三步：滑动窗口 + 「maxCount 只增不减」

维护一个窗口 [left, right] 和一个 26 个字母的计数数组 count。 right 每右移一格，就把新字符计入 count，并顺手更新 maxCount = max(maxCount, count[新字符])。

只要 (right − left + 1) − maxCount > k，说明这个窗口光靠 k 次替换凑不出 「全同一字母」，就把 left 右移一格（把移出窗口的字符计数减一）。 每一步结束，right − left + 1 就是当前合法窗口的长度，取这些长度里的最大值就是答案。

为什么 maxCount 可以「只增不减」、不用每次重新扫一遍找最大值？ 因为我们的目标不是「让窗口始终合法」，而是「找到出现过的最长合法窗口长度」。 就算 maxCount 用的是过期的（其实当前窗口里最多字母数已经变少）也没关系—— 它最多让窗口少收缩一次，窗口长度也就原地不变，不会变长， 所以不会把错误答案当成更优答案；而它确实曾经达到过那个 maxCount， 说明「答案至少有这么长」是成立的。这样窗口长度只增不减（最多回退 1），扫一遍就够。

当前 right 指向的字符 当前窗口 [left, right] 收缩时移出窗口的字符 目前为止最优子串

试试把字符串改成 ABAB、k 改成 2，看窗口怎么一路扩张到整个字符串都不用收缩。

☕ 第四步：写成 Java 代码

class Solution {
    public int characterReplacement(String s, int k) {
        int[] count = new int[26]; // 窗口内每个大写字母出现的次数
        int left = 0, maxCount = 0, ans = 0;

        for (int right = 0; right < s.length(); right++) {
            int idx = s.charAt(right) - 'A';
            count[idx]++;
            maxCount = Math.max(maxCount, count[idx]); // 只增不减，无需重新扫描

            // 窗口长度 - maxCount 就是需要替换的字符数，超过 k 就收缩左边界
            while (right - left + 1 - maxCount > k) {
                count[s.charAt(left) - 'A']--;
                left++;
            }

            ans = Math.max(ans, right - left + 1);
        }
        return ans;
    }
}
时间复杂度 O(n)

left 和 right 各自只会向右移动，加起来最多移动 2n 次。

空间复杂度 O(1)

计数数组固定 26 个大写字母，大小不随输入变化。

🚀 小结

套路：当题目里出现「最多操作 k 次」「连续子串」这类条件时， 先想想能不能把它转成一个滑动窗口的合法性判断公式—— 本题就是 窗口长度 − maxCount ≤ k。一旦能写出这样的公式， 就可以一边扩张窗口一边维护统计量，不合法就收缩，整个过程线性扫描搞定， 这是「最长连续子串」类问题的通用解法。
