整个 LeetCode 的第一题，也是「用空间换时间」最经典的入门例子。 看完你会明白：一个哈希表，怎么把笨办法的两层循环压成轻轻一遍。

🧩 第一步：题目到底要什么

给你一排数字和一个目标值 target，要你找出两个数， 它们加起来正好等于 target，返回这两个数的下标（不是数本身）。

nums[2, 7, 11, 15]
target9

2 + 7 = 9，它们在下标 0 和 1 → 返回 [0, 1]。

🐢 第二步：最直接的笨办法

最容易想到的：拿每个数去和它后面的每个数两两相加，看有没有等于 target 的。两层循环，能做出来，但每个数都要把后面扫一遍。

慢在哪：两层循环是 O(n²)。数组一长（题目里能到 10⁴）， 就要算上亿次。问题的本质是——我们对「另一半在哪」这件事，每次都重新找。

⚡ 第三步：用哈希表「边走边记」

换个思路：扫到 nums[i] 时，我们其实只关心一件事—— 前面有没有出现过 target − nums[i]（也就是「还差的那一半」need）。

于是边走边把见过的「值 → 下标」记进哈希表。到一个新数就先查表： need 在不在？在就立刻配对成功；不在就把自己记进去。查表是 O(1)， 整个数组只扫一遍。

当前 nums[i] 已记入哈希表 配对成功的另一半

试试把数组改成 3,2,4、target 改成 6，看它怎么跳过 3、用 2 配对 4。

☕ 第四步：写成 Java 代码

class Solution {
    public int[] twoSum(int[] nums, int target) {
        // seen: 值 -> 下标，记录「见过的数」
        Map<Integer, Integer> seen = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int need = target - nums[i]; // 还差的那一半
            if (seen.containsKey(need)) {
                return new int[]{seen.get(need), i};
            }
            seen.put(nums[i], i); // 没配上，把自己记下来
        }
        return new int[]{}; // 题目保证有解，不会到这
    }
}
时间复杂度 O(n)

数组只扫一遍，每次查表 / 存表都是 O(1)。

空间复杂度 O(n)

最坏情况哈希表要存下几乎所有元素。

🚀 小结

套路：当你发现「在数组里找另一个满足某关系的元素」时， 先想想能不能把见过的东西记进哈希表，把「查找」从 O(n) 变成 O(1)。 两数之和是这个套路的起点，后面很多题（如分组、去重、配对）都用它。
