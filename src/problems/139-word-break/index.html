这是「字符串 + 动态规划」里最经典的入门题之一。我们先把题目翻译成大白话， 再用最笨的办法亲手试一遍，最后看它怎么被动态规划「驯服」。

🧩 第一步：这题到底在问什么

抛开术语，它问的是一件很具体的事：能不能把字符串 s 切成若干段，让每一段都正好是词典里的一个词？

sleetcode
词典["leet", "code"]

能切成 leet + code，两段都在词典里 → 返回 true。

两个容易忽略的规则：词典里的词不必全用，而且可以重复用。 比如 applepenapple 用 apple、pen、apple 就能拼出来。

🐢 第二步：先用最笨的办法——从左往右试着切

最自然的想法：从开头出发，试着切下一个「在词典里的词」，切成功就从断点往后接着切； 要是走到某处发现怎么都切不动，就退回去换一种切法。这其实就是一次 回溯搜索。

下面这个动画你可以亲手点，看指针怎么一段段往前「啃」，遇到死路又怎么退回来：

命中词典的尝试 不在词典的尝试 已切好的段
关键转折：暴力法会把同一个位置反复试很多遍—— 从前面不同的切法走过来，都会再问一次「从这里能不能走通」。 动态规划的核心就是：每个位置只问一次，把答案记下来反复用。

🧠 第三步：动态规划三件套

把「从某个位置能不能走通」这件事，正过来定义成一张一维的表：

1

状态：dp[i] 表示什么？

dp[i] = 字符串 s 的前 i 个字符 能不能被切成若干个词典里的词（true / false）。

最终答案就是 dp[n]（n 是 s 的长度）——整串能不能拆。

2

转移：dp[i] 怎么由更小的问题推出来？

要判断「前 i 个字符」能不能拆，就枚举最后一个词的起点 j（0 ≤ j < i）：

  • 如果 dp[j] 为真（前 j 个字符已经能拆）， 并且子串 s[j..i) 正好是词典里的词， 那么前 i 个字符也能拆：
    dp[i] = true（找到一个这样的 j 就够了）。
3

边界：最小的情况是什么？

dp[0] 代表「空串」。空串不需要任何词就「拆」好了， 所以 dp[0] = true。它是整张表的地基—— 第一个真正的词正是接在它后面的。

🎬 第四步：看 dp 数组一格一格点亮

下面这张一维表你可以亲自点。dp 的格子落在字符之间的「缝」上： 算 dp[i] 时，会回头看每个切点 j 的 dp[j]， 再检查中间那段子串 s[j..i) 在不在词典里。

正在算的 dp[i] 回看的 dp[j] 已确定可拆 (T)

试试把词典改成 leet,code,le,etcode，或者把 s 改成 catsandog、词典改成 cats,dog,sand,and,cat， 看它怎么判定「拼不出来」。

☕ 第五步：写成 Java 代码

把三件套翻译成代码，几乎就是一一对应：

class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        // 放进 HashSet，查一个词在不在词典里就是 O(1)
        Set<String> dict = new HashSet<>(wordDict);
        int n = s.length();

        // dp[i]: s 的前 i 个字符能否被拆分
        boolean[] dp = new boolean[n + 1];
        dp[0] = true; // 边界：空串天然「拆」好了

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j < i; j++) {
                // 前 j 个字符能拆，且 s[j..i) 是词典里的词
                if (dp[j] && dict.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break; // 找到一种切法就够了
                }
            }
        }
        return dp[n];
    }
}
时间复杂度 O(n²)

两层循环枚举 i、j；每次还要取子串并查哈希，约 O(n² · L)。

空间复杂度 O(n)

一个长度 n+1 的 dp 数组，外加把词典装进哈希集合。

🚀 小结：从「试切」到「记账」

单词拆分的精髓，是把「能不能切」这个会反复发生的问题，用一张 dp 表 记账：每个前缀只判一次，后面的判断直接查表。

套路迁移：很多「能否 / 有几种方法把序列切成合法段」的题 （比如解码方法、分割回文串的判定）都能套这个一维 DP 的框架： dp[i] 由「枚举最后一段的起点 j」+「dp[j] 且 s[j..i) 合法」推出。 先想清楚状态 → 转移 → 边界，再写代码。
