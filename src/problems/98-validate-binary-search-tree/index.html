一棵树「看起来」左小右大，不代表它真的是二叉搜索树。 这道题教你一个所有树形递归都通用的武器：给每个节点带上一对「允许区间」。

🧩 第一步：题目到底要什么

给你一棵二叉树的根节点 root，判断它是不是一棵合法的二叉搜索树（BST）。 合法的意思是：对树里每一个节点，它的左子树里所有节点都要严格小于它， 右子树里所有节点都要严格大于它——而且这个要求对左右子树自己也要成立（递归地）。

root[5,1,4,null,null,3,6]

根节点是 5，它的右子树里有个节点 4。可右子树的所有节点都该比根节点 大，4 却比 5 小——直接违反了 BST 的定义，所以这不是合法 BST，答案是 false。

🐢 第二步：一个很容易踩的坑

最容易想错的写法是：只比较每个节点和它的直接孩子—— 检查 node.left.val < node.val 且 node.val < node.right.val， 递归下去。这样写在示例 2 上会出错：节点 4 和它的左孩子 3、右孩子 6 单独看完全没问题 （3 < 4 < 6），但 4 其实出现在根节点 5 的右子树里， 整个右子树都必须比 5 大，4 却比 5 小——只看父子关系，发现不了这种「跨层级」的违规。

关键洞察：一个节点合不合法，不能只看它的父节点， 要看它所有祖先一路收紧下来的合法区间。根节点的区间是 (−∞, +∞)；往左走一步，区间的上界收紧成父节点的值； 往右走一步，区间的下界收紧成父节点的值。每个节点的值必须落在 它继承到的这个区间里。

⚡ 第三步：用一个显式栈做「区间收缩」

用一个栈模拟「先序遍历」：栈里每一项是 (节点, low, high)。 弹出一个节点先检查 low < node.val < high；不满足直接判 false。 满足的话，把它的右孩子（区间下界收紧成 node.val）和左孩子（区间上界收紧成 node.val）都压栈继续检查——先压右孩子、再压左孩子，这样左子树会先被弹出处理， 顺序和递归「先左后右」一致。

当前正在检查 已验证合法 区间外，判定失败 全部通过

试试把树改成合法的 2,1,3，对比它和默认的 5,1,4,null,null,3,6 在哪一步分出胜负。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isValidBST(TreeNode root) {
        return valid(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }

    // low、high 用 long，避免节点值正好等于 int 边界时区间判断出错
    private boolean valid(TreeNode node, long low, long high) {
        if (node == null) return true; // 空树天然合法
        if (node.val <= low || node.val >= high) {
            return false; // 跳出了祖先们划定的区间
        }
        // 往左走：上界收紧成当前值；往右走：下界收紧成当前值
        return valid(node.left, low, node.val) && valid(node.right, node.val, high);
    }
}

动画里展示的「显式栈」版本和这段递归本质上是同一个算法——栈里压的 (节点, low, high) 就是一次函数调用的参数，只是把递归用循环摊开，方便逐步演示。

时间复杂度 O(n)

每个节点恰好被检查一次。

空间复杂度 O(h)

h 是树高，来自递归栈（或显式栈）；平衡树 O(log n)，最坏（链状树）退化到 O(n)。

🚀 小结

套路：判断一棵树「是否满足某种全局性质」时， 光比较父子节点常常不够——要想清楚从根到当前节点这一路上积累下来的约束， 把它打包成参数（这里是 (low, high) 区间）一路传下去。 这种「自顶向下传约束」的递归模式，在树形问题里会反复出现。
