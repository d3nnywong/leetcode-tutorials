二叉树里的「最长一段路」是多少？关键是想明白：这条路径可以在任意节点拐弯， 也不一定经过根。看完你会明白怎么用一次后序遍历把它算出来。

🧩 第一步：题目到底要什么

二叉树里的路径，就是沿着边走出来的一串节点，每个节点最多走一次。 关键的两个「反直觉」点：

  • 路径不一定经过根节点——可以完全发生在某个子树内部。
  • 路径可以在某个节点拐弯：从左子树绕上来，再拐进右子树， 像一个倒过来的「人」字形。
root[-10, 9, 20, null, null, 15, 7]

根节点 −10 不在最优路径里！真正的答案是 15 → 20 → 7，在节点 20 处「拐弯」， 路径和 = 15 + 20 + 7 = 42。

🔑 第二步：关键洞察——「拐弯」和「单边贡献」要分开看

暴力枚举「所有节点对之间的路径」要 O(n²) 起步，太慢。换个角度想： 把每个节点轮流当作路径的「最高点」，算一遍它能撑起的最大路径和， 再从所有候选里取最大值——这样每个节点只需要被处理一次。

对节点 x，它的左子树最多能贡献 leftGain，右子树最多能贡献 rightGain（如果某一边算出来是负数，那还不如不要这条边，所以和 0 取较大值——这是这道题最容易忽略的一步）。那么：

  • 以 x 为拐弯点的候选答案 = x.val + leftGain + rightGain， 随时更新全局最大值；
  • 但 x 往上汇报给父节点时，路径只能从一个方向「穿过」它， 所以只能带 x.val + max(leftGain, rightGain) 这一边上去。
一句话：「能更新答案的拐弯值」和「能往上传的单边值」是两个不同的东西， 全程后序遍历（先左右子树，再自己）一遍就能同时算出来，O(n)。

⚡ 第三步：后序遍历动画

动画按后序遍历的顺序逐节点展开：递归先扎到底，子树都算完后再回头处理当前节点—— 算出左右贡献、刷新全局最大值、再把单边贡献汇报给父节点。

当前节点 调用栈上的祖先 / 正在被读取的子节点 最终的最优路径

节点下方的 gain=… 是该节点算完后往上汇报的值；红色表示是负数 （父节点会当它是 0）。试试把数组改成 1,2,3，看根节点自己就是最高点的情况。

☕ 第四步：写成 Java 代码

/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val; this.left = left; this.right = right;
 *     }
 * }
 */
class Solution {
    private int maxSum = Integer.MIN_VALUE;

    public int maxPathSum(TreeNode root) {
        maxGain(root);
        return maxSum;
    }

    // 返回：从 node 出发、只能往一个方向延伸、能贡献给「父节点」的最大路径和
    private int maxGain(TreeNode node) {
        if (node == null) return 0;

        // 子树贡献为负就不要（相当于不走这条边），所以和 0 取较大值
        int leftGain = Math.max(maxGain(node.left), 0);
        int rightGain = Math.max(maxGain(node.right), 0);

        // 以 node 为「最高点」、左右都拐弯的路径和，是一个候选答案
        int priceNewPath = node.val + leftGain + rightGain;
        maxSum = Math.max(maxSum, priceNewPath);

        // 往上汇报给父节点时，只能带一条边走，所以左右只能选一边
        return node.val + Math.max(leftGain, rightGain);
    }
}
时间复杂度 O(n)

每个节点只在后序遍历中被访问一次。

空间复杂度 O(h)

h 是树高，来自递归调用栈；最坏（链状树）退化到 O(n)。

🚀 小结

套路：遇到「树上任意两点间路径」的最值问题，常见思路是 把每个节点轮流当成路径的最高点，用后序遍历分别算出 「能更新答案的拐弯值」和「能往上传的单边值」。这个套路同样适用于树的直径、 相机覆盖等一大类「以某节点为支点」的二叉树 DP 问题。
