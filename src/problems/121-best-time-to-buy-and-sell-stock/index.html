只能买一次、卖一次的股票题。看完你会发现答案藏在一句话里： 每天问自己「如果今天卖，赚得最多是多少」，一遍扫描就够了。

🧩 第一步：题目到底要什么

给你一串股价 prices，prices[i] 表示第 i 天这只股票的价格。 规则很简单：你手里最多只能买一次、卖一次，而且必须先买后卖 ——这好理解，你得先把股票买到手，才有得卖，所以卖出那天肯定在买入那天之后。 任务是求出你能赚到的最大利润；如果不管怎么选，买卖都是亏本的， 就返回 0（相当于「干脆不做这笔交易」，不买不卖也不算错）。

prices[7, 1, 5, 3, 6, 4]

第 1 天（价格 1）买入，第 4 天（价格 6）卖出，利润 6 − 1 = 5，是所有买卖组合里最大的。

🐢 第二步：最直接的笨办法

最容易想到的办法：把每一对「买入日 i、卖出日 j」（j > i，卖必须在买之后） 都拿出来试一遍——这叫枚举，说白了就是「不放过任何一种可能，一个个都试」。 对每一对算出 prices[j] − prices[i]，从所有结果里挑出最大的那个。 买入日、卖出日各转一圈，写成代码就是两层嵌套的循环，两边一乘，组合数就是 O(n²)。

慢在哪：题目里 n（股价的天数）能到 10⁵，平方就是 10¹⁰， 这是电脑在合理时间内根本算不完的规模。真正浪费在哪呢——枚举每个卖出日 j 时， 我们其实没必要把它前面所有买入日的价格都重新比一遍， 只要记得「j 之前出现过的最低价格是多少」就够了：因为不管买在哪一天， 只要不是那个最低价的那天买入，算出来的利润都不会更高。

⚡ 第三步：一遍扫描，边走边记「历史最低价」

换个思路：只从左往右扫一遍，边走边记住目前为止见过的最低价 minPrice ——它就是「如果今天要卖出，这些天里最划算的买入点」。这样走到新的一天时， 我们都不用回头再看一遍前面所有的价格，因为最有用的那个信息（历史最低价）已经随身带着了。

走到第 i 天时，只有两种可能：

  • 如果今天的价格 比 minPrice 还低，说明今天才是更好的买入点，于是更新 minPrice。（这一天不用算利润——如果今天的价格就是目前最低的， 拿它当买入价再减它自己当卖出价，利润是 0，不可能比之前算过的记录更好。）
  • 否则，说明今天的价格不低于历史最低价，那今天就可以当一个候选的卖出日： 算一下「今天价格 − minPrice」这笔利润， 如果比历史最大利润 maxProfit 还大，就刷新答案。

拿题目给的例子 [7, 1, 5, 3, 6, 4] 亲手走一遍：第 0 天价格 7，minPrice 先设成 7； 第 1 天价格 1 比 7 还低，minPrice 更新成 1；第 2 天价格 5，5 − 1 = 4，把 maxProfit 刷新成 4； 第 3 天价格 3，3 − 1 = 2，不够大，maxProfit 保持 4；第 4 天价格 6，6 − 1 = 5，刷新 maxProfit 为 5； 第 5 天价格 4，4 − 1 = 3，不够大。扫完一遍，答案就是 5——正好对应「第 1 天买、第 4 天卖」。

全程只扫一遍、不回头，相当于把「枚举所有买卖组合」压缩成「每天只跟历史最低价比一次」， 复杂度从 O(n²) 降到 O(n)。

历史最低价（买入候选） 当前扫描的日子 当前可获得的利润 最终最大利润的买卖点

虚线是「历史最低价」这条参考线；绿色色带的高度就是当天能赚的利润，刷新纪录时会变成金色。 试试把数组改成 7,6,4,3,1，看它怎么一路找不到能获利的卖出点，最终返回 0。

☕ 第四步：写成 Java 代码

class Solution {
    public int maxProfit(int[] prices) {
        int minPrice = prices[0]; // 目前见过的最低价
        int maxProfit = 0;       // 目前能获得的最大利润
        for (int i = 1; i < prices.length; i++) {
            if (prices[i] < minPrice) {
                // 今天比历史最低还低，它才是更好的买入点
                minPrice = prices[i];
            } else {
                // 假设今天卖出，看看能不能刷新最大利润
                maxProfit = Math.max(maxProfit, prices[i] - minPrice);
            }
        }
        return maxProfit;
    }
}
时间复杂度 O(n)

价格数组只扫一遍，每天只做常数次比较。

空间复杂度 O(1)

只用了 minPrice 和 maxProfit 两个变量。

🚀 小结

套路：当答案依赖「某个位置之前的最优值」时（这里是「之前出现过的最低价」， 这类值也叫前缀最值——从开头到当前位置为止，见过的最大或最小的那个数）， 不需要每次都重新枚举，边扫边维护这个最优值，就能把 O(n²) 的枚举压成 O(n) 的一遍扫描。 后面遇到「求区间/序列内某种最大差值」的题，都可以往这个方向想一想。
