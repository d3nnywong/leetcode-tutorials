链表只能往前走，不知道总长度就不知道「倒数第几个」是谁。 看完你会明白：加一个虚拟头结点、用快慢指针拉开 n+1 步的间距，一趟扫描就能搞定。

🧩 第一步：题目到底要什么

先花 10 秒弄清楚链表长什么样：把它想象成一串珠子， 每颗珠子（在代码里叫一个结点 node）除了自己的数值， 手里还攥着一张纸条，写着「下一颗珠子在哪」（这张纸条就是它的 next）。 跟数组不一样，链表的珠子不是整整齐齐挨在一起摆放的，你手上能直接拿到的只有第一颗—— 也就是头结点 head——想摸到后面第几颗，只能顺着纸条 一颗接一颗地往后走，没法像数组那样凭下标「唰」地一下跳过去。

题目给你这样一条链表的头结点 head，再给一个数字 n， 要你删掉倒数第 n 个结点（倒数第几颗珠子），返回删除后链表的头结点。

head[1, 2, 3, 4, 5]
n2

倒数第 2 个是值为 4 的结点（从右往左数：5 是倒数第 1，4 是倒数第 2）。 删掉它之后链表变成 [1, 2, 3, 5]。

难点在于：链表没法直接用「长度 − n」算出下标去访问——因为你压根不知道这条链表有多长， 只能老老实实从头开始，一颗珠子一颗珠子地往后摸。

🐢 第二步：最直接的笨办法

最容易想到的：先从头到尾走一遍，数出链表总长度 sz； 倒数第 n 个就是正数第 sz − n + 1 个，再从头走一遍找到它的前一个结点，把它摘掉。 这样能做对，但要走两趟链表。

能不能只走一趟？ 关键洞察：这里说的「指针」，你可以理解成两根手指—— 分别按在链表的某颗珠子上，「指针往前走一步」就是把手指顺着 next 纸条挪到下一颗珠子上。如果有两根这样的手指（指针），一个比另一个 先出发 n+1 步，那么当先出发的那根走到链表末尾 （null，表示「已经没有下一颗了」）时，后出发的那根正好停在 「倒数第 n 个结点的前一个」——因为它们之间的距离全程都是 n+1，不需要事先知道链表总长度。

⚡ 第三步：虚拟头结点 + 快慢指针

这里的 fast、slow 也常被叫做「快慢指针」，但和判断链表有没有环时那种 「快指针每次走 2 步、慢指针每次走 1 步」不太一样——本题里 fast、slow 每一步都走得一样快，只是 fast 提前出发了 n+1 步， 之后两根手指同时往前挪，中间的间距全程都保持 n+1， fast 走到头时，slow 就正好停在我们要的位置上。

为什么还要加一个虚拟头结点 dummy？因为如果倒数第 n 个恰好就是 head 本身（比如整条链表只有 1 个结点，删它自己）， 就没有「前一个结点」可以操作了。让 dummy.next = head， 一切删除操作都变成「修改某个结点的 next」，不用再特殊处理头结点。

具体三步：

  • ① 拉开间距：fast、slow 都从 dummy 出发，先让 fast 单独走 n+1 步。
  • ② 一起走：fast、slow 同时往前走，直到 fast 走到链表末尾 null。
  • ③ 摘掉结点：此时 slow 正好停在待删除结点的前一个，执行 slow.next = slow.next.next，返回 dummy.next。
fast 指针 slow 指针 待删除 / 已删除的结点 删除后新的 next 连接

☕ 第四步：写成 Java 代码

// Definition for singly-linked list.
class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        // 虚拟头结点：避免「删的正好是 head」这种特殊情况
        ListNode dummy = new ListNode(0);
        dummy.next = head;

        ListNode fast = dummy;
        ListNode slow = dummy;

        // 1. fast 先单独走 n + 1 步，拉开 fast 与 slow 的间距
        for (int i = 0; i <= n; i++) {
            fast = fast.next;
        }

        // 2. fast、slow 一起走，直到 fast 越过链表末尾（变成 null）
        while (fast != null) {
            fast = fast.next;
            slow = slow.next;
        }

        // 3. 此时 slow 正好停在待删除结点的前一个，摘掉它
        slow.next = slow.next.next;

        return dummy.next;
    }
}
时间复杂度 O(sz)

链表只扫一遍（fast 走到底，slow 跟着走），sz 是结点数。

空间复杂度 O(1)

只用了 dummy、fast、slow 三个指针，不依赖结点数。

🚀 小结

套路：链表上要找「倒数第 n 个」，别想着先数长度再算下标—— 用快慢指针拉开固定间距，让快指针探路、慢指针跟着停在正确位置，一趟扫描搞定。 再配上虚拟头结点统一处理「删的正好是 head」这种边界情况， 这是链表题里最常见、最值得记住的组合技巧之一。
