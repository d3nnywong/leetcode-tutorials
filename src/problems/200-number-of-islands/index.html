一张由陆地和水组成的地图，怎么数出有几座「岛」？答案是 网格上的 DFS/BFS——把每一片连在一起的陆地「淹没」一遍， 淹过的不再重复计数，淹了几次就是几座岛。

🧩 第一步：题目到底要什么

给你一个二维网格，每个格子要么是 '1'（陆地）要么是 '0'（水）。上下左右相邻的陆地会连成同一座岛， 斜着挨着不算。问：这张地图上一共有几座岛？

grid11000 / 11000 / 00100 / 00011

左上角 4 个 1 连成一片是第 1 座岛；中间单独的 1 是第 2 座岛； 右下角两个 1 挨在一起是第 3 座岛 → 答案 3。

🐢 第二步：暴力会怎么数错

如果只是「数一数有多少个 1」，那是错的——一片连起来的陆地不管有多少个 1，都只能算一座岛。真正要数的是 「连通块」的个数。

关键洞察：从任意一个陆地格子出发，往上下左右四个方向「扩散」， 能摸到的陆地都属于同一座岛。只要扩散时把摸过的格子标记为已访问， 扫描时跳过已访问的格子，每发现一块「全新」的陆地就让岛屿数 +1—— 网格只需要整体扫一遍，每个格子最多被访问一次。

⚡ 第三步：DFS 灌水法

按行扫描网格，遇到一个未访问的陆地，就说明发现了新岛屿（岛屿数 +1），然后从这个格子开始做一次 DFS：用一个栈，不断弹出栈顶格子、 标记为已访问、把它四周还没访问过的陆地压进栈，直到栈空——这一整片陆地就被 「淹没」完了，继续扫描下一个格子。

当前格子（扫描指针 / 正在 DFS 访问） 本步新发现、压入栈的陆地 已访问、归入某座岛屿（不同颜色 = 不同岛屿）

试试把网格改成 11110,11010,11000,00000（这是官方示例 1，只有 1 座岛）， 看看一整片陆地是怎么被一次 DFS 全部淹没的。

☕ 第四步：写成 Java 代码

class Solution {
    public int numIslands(char[][] grid) {
        int rows = grid.length, cols = grid[0].length;
        int count = 0;
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (grid[r][c] == '1') {
                    count++; // 发现一块未访问的陆地，新岛屿
                    dfs(grid, r, c); // 把这一整片陆地「淹没」
                }
            }
        }
        return count;
    }

    // 把 (r, c) 这片陆地连通的所有格子都标记为已访问（淹没成 '0'）
    private void dfs(char[][] grid, int r, int c) {
        int rows = grid.length, cols = grid[0].length;
        if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] != '1') {
            return; // 越界，或者是水/已访问过，停止扩散
        }
        grid[r][c] = '0'; // 标记为已访问
        dfs(grid, r - 1, c);
        dfs(grid, r + 1, c);
        dfs(grid, r, c - 1);
        dfs(grid, r, c + 1);
    }
}
时间复杂度 O(m × n)

每个格子最多被访问一次（成为陆地后立刻被标记，不会重复扩散）。

空间复杂度 O(m × n)

最坏情况整张网格都是陆地，DFS 递归栈（或显式栈）深度可达 m × n。

🚀 小结

套路：网格上「数连通块」的题，核心就是 扫描 + 灌水（flood fill）：扫到一个未访问的目标格子就计数 +1， 再用 DFS 或 BFS 把它所在的整个连通块标记为已访问，避免重复计数。 这个套路同样适用于「最大岛屿面积」「被围绕的区域」等一大类网格连通性问题。
