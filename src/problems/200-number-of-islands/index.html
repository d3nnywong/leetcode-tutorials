一张由陆地和水组成的地图，怎么数出有几座「岛」？答案是 网格上的 DFS（深度优先搜索）——一条路摸到黑， 把每一片连在一起的陆地都「淹没」一遍，淹过的不再重复计数，淹了几次就是几座岛。

🧩 第一步：题目到底要什么

给你一个二维网格，每个格子要么是 '1'（陆地）要么是 '0'（水）。上下左右相邻的陆地会连成同一座岛， 斜着挨着不算。问：这张地图上一共有几座岛？

grid11000 / 11000 / 00100 / 00011

左上角 4 个 1 连成一片是第 1 座岛；中间单独的 1 是第 2 座岛； 右下角两个 1 挨在一起是第 3 座岛 → 答案 3。

🐢 第二步：暴力会怎么数错

最容易想歪的办法：数一数网格里一共有多少个 '1'。这是错的—— 一片连成一整块的陆地，不管里面挤了多少个 '1'， 从头到尾都只能算一座岛。

真正要数的是「连成一片的陆地」有几块——这种「一堆格子，靠上下左右能互相走到， 就算同一伙」的一片区域，专业说法叫连通块。就像操场上几群手拉手的学生， 不管一群里有几个人、站得多分散，只要手是拉着的（能通过上下左右走到），就还是同一群。 我们要数的不是「人头」，而是「有几群」。

关键洞察：从任意一个陆地格子出发，往上下左右四个方向「扩散」， 能摸到的陆地都属于同一座岛（同一个连通块）。只要扩散时 把摸过的格子标记为已访问，扫描时跳过已访问的格子， 每发现一块「全新」的陆地就让岛屿数 +1——网格只需要整体扫一遍，每个格子最多被访问一次。

⚡ 第三步：DFS 灌水法

按行扫描网格，遇到一个未访问的陆地，就说明摸到了一整块全新的岛 （岛屿数 +1）。接下来要做的事，专业说法叫 DFS（深度优先搜索）—— 说白了就是「认准一个方向一条路走到黑，走不通了再退回来试别的方向」， 直到这一整片陆地都被摸遍为止。

下面的动画用了一个栈来实现 DFS：栈就像一摞叠起来的盘子， 只能从最上面拿、也只能往最上面放（最后放进去的会最先被拿出来）。做法是： 先把起点格子放进栈里；然后反复「从栈顶拿出一个格子 → 标记为已访问 → 把它四周还没访问过的陆地也放进栈顶」，直到栈被拿空——这说明这一片陆地已经 全部摸完、被「淹没」完了，再去扫描下一个格子。

Java 代码里换了一种效果等价、但写法更简洁的方式——递归： 让一个函数自己调用自己去解决一个更小的同类问题，有点像照镜子时镜子里还有一面镜子， 一层套一层。我们写一个「淹没 (r, c) 这个格子」的函数：它先把当前格子标记为已访问， 再把上、下、左、右四个方向的格子，各自交给自己去继续淹没。 只要给它一个「该停下来」的条件——出界了，或者这个格子本来就是水、已经访问过—— 递归就不会没完没了地套下去。跟用栈的思路是一回事：一路调用下去， 会像野火一样把整片连通的陆地都标记成已访问。

当前格子（扫描指针 / 正在 DFS 访问） 本步新发现、压入栈的陆地 已访问、归入某座岛屿（不同颜色 = 不同岛屿）

试试把网格改成 11110,11010,11000,00000（这是官方示例 1，只有 1 座岛）， 看看一整片陆地是怎么被一次 DFS 全部淹没的。

☕ 第四步：写成 Java 代码

class Solution {
    public int numIslands(char[][] grid) {
        int rows = grid.length, cols = grid[0].length;
        int count = 0;
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (grid[r][c] == '1') {
                    count++; // 发现一块未访问的陆地，新岛屿
                    dfs(grid, r, c); // 把这一整片陆地「淹没」
                }
            }
        }
        return count;
    }

    // 把 (r, c) 这片陆地连通的所有格子都标记为已访问（淹没成 '0'）
    private void dfs(char[][] grid, int r, int c) {
        int rows = grid.length, cols = grid[0].length;
        if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] != '1') {
            return; // 越界，或者是水/已访问过，停止扩散
        }
        grid[r][c] = '0'; // 标记为已访问
        dfs(grid, r - 1, c);
        dfs(grid, r + 1, c);
        dfs(grid, r, c - 1);
        dfs(grid, r, c + 1);
    }
}
时间复杂度 O(m × n)

每个格子最多被访问一次（成为陆地后立刻被标记，不会重复扩散）。

空间复杂度 O(m × n)

最坏情况整张网格都是陆地：递归会一层套一层地往下叫，每一层没返回之前都占着一份内存 （这些「还没返回」的调用叠在一起，就是所谓的递归栈）， 最深能套到 m × n 层。

🚀 小结

套路：网格上「数连通块」的题，核心就是 扫描 + 灌水（flood fill）：扫到一个未访问的目标格子就计数 +1， 再用 DFS（一条路走到黑）或 BFS（一圈圈地往外扩，广度优先） 把它所在的整个连通块标记为已访问，避免重复计数。 这个套路同样适用于「最大岛屿面积」「被围绕的区域」等一大类网格连通性问题。
