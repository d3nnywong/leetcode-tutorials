这题是 100「相同的树」的延伸：不再只比较两个固定的根，而是要在一棵大树里 挨个挑候选根，每挑一个就复用一次「判断两棵树是否相同」。

🧩 第一步：题目到底要什么

给你两棵二叉树 root 和 subRoot，问 root 里有没有 某个节点，从它往下长出来的那一整棵子树，跟 subRoot 长得一模一样（结构相同 + 每个对应位置的值也相同）。

root[3,4,5,1,2]
subRoot[4,1,2]

root 里节点 4（带着它的孩子 1、2）单独拎出来看，正好是「4 → 左 1、右 2」， 和 subRoot 完全一样 → 答案 true。

注意「子树」的定义：必须是某个节点和它的全部后代，不能是树中间随便截一段。 另外 tree 也算它自己的子树——也就是说 root 整棵和 subRoot 整棵相同，也算数。

💡 第二步：把它拆成「逐个候选根」+「判断相同」

直接想「root 里有没有和 subRoot 一样的子树」会觉得无从下手，但只要换个角度： root 的每一个节点，都可能是那个「子树的根」。我们只要把 root 的节点 一个一个挑出来当候选根，每挑一个，就整体比较一次——这正是 100「相同的树」里 写过的 isSameTree。

  • 挑到候选根 node，调用 isSameTree(node, subRoot)。
  • 比上了 → 直接返回 true，不用再找别的候选根。
  • 没比上 → 这个候选根本身不行，但它的左右孩子还各自可能是更小一棵 「以自己为根」的子树，继续往它们那里找下一个候选。
  • root 所有节点都试过还是没找到，才返回 false。
容易搞混的地方：候选根比对失败 ≠ 整道题失败，它只代表「以这个节点为根」 这一种可能性不行；root 里还有别的节点可以再当候选根。这也是为什么算法要对 root 做一次遍历（外层 DFS），每到一个节点就调用一次完整的 isSameTree（内层比较）——是两层递归叠在一起。

⚡ 第三步：动画看「挑候选根 → 整体比较 → 换下一个」

下面用先序遍历 root：每走到一个新节点，就把它当成候选根（高亮），和右边的 subRoot 整体比较一次。比对失败的候选根会变暗、标出第一次出现不同的位置， 然后动画继续走向它的左右孩子，换下一个候选；一旦比上，整段匹配的子树会一起变绿。

当前候选根 已淘汰的候选根 这一步第一次出现不同 完全匹配的子树

默认例子会在节点 4 处匹配成功（结果 true）；试试把 root 改成 3,4,5,1,2,null,null,null,null,0（subRoot 不变），看候选根 4 为什么这次会 因为多出来的节点 0 而比对失败、改成 false。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isSubtree(TreeNode root, TreeNode subRoot) {
        if (root == null) {
            // root 都走完了，没找到；题目保证 subRoot 非空，所以这里只能是 false
            return false;
        }
        // 把当前节点当候选根，整体比较一次
        if (isSameTree(root, subRoot)) {
            return true;
        }
        // 这个候选根不行，去它的左右子树里继续找下一个候选根
        return isSubtree(root.left, subRoot) || isSubtree(root.right, subRoot);
    }

    // 100. 相同的树：判断两个节点为根的两棵树是否完全相同
    private boolean isSameTree(TreeNode p, TreeNode q) {
        if (p == null && q == null) {
            return true;
        }
        if (p == null || q == null) {
            return false;
        }
        return p.val == q.val
            && isSameTree(p.left, q.left)
            && isSameTree(p.right, q.right);
    }
}
时间复杂度 O(m · n)

m、n 分别是 root、subRoot 的节点数：最坏情况 root 的每个节点都要当一次候选根， 每次 isSameTree 比较最多要看完 subRoot 那么多节点。

空间复杂度 O(h₁ + h₂)

h₁、h₂ 分别是 root、subRoot 的高度：外层 isSubtree 的递归栈深度跟 root 的高度有关， 它调用到的 isSameTree 又会再压一层最深到 subRoot 高度的栈。

🚀 小结

套路：遇到「在一棵大结构里找有没有某个小结构」，可以想想能不能拆成 「枚举每一个可能的起点（候选根）」+「复用一个已经会写的『判断是否相同』函数」。 这道题就是 100「相同的树」套上一层遍历——把熟悉的递归当作积木，往往比从零想新算法更快。
