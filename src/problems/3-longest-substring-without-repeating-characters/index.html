一条字符串里「滑」出一个不含重复字符的窗口。看完你会明白： 为什么不用每次都从头数，一个哈希表 + 一个左边界指针就能一遍扫完。

🧩 第一步：题目到底要什么

给你一个字符串 s，要你找出其中最长的一段 连续子串（子串，不是子序列），这段里面不能有重复字符。 只要返回这段的长度就行。

s"abcabcbb"

不含重复字符的最长子串是 "abc"，长度 3。 注意从第 4 个字符起又出现了 a，所以不能继续往后数。

🐢 第二步：最直接的笨办法

最容易想到的：枚举每一个起点 i，再从 i 往后一个字符一个字符地扩， 每加一个字符就检查「这一段里有没有重复」，直到出现重复或到字符串末尾为止，记录这段的最大长度。

慢在哪：枚举起点是 O(n)，每个起点往后扩 + 查重复又要 O(n)，整体 O(n²)（甚至更高，如果查重复本身也要扫一遍）。 问题的本质是——起点往右挪一格之后，大部分判断其实是重复劳动： 上一段已经确认无重复的那一大截，没必要从头再查一遍。

⚡ 第三步：用滑动窗口「只进不退」地扫一遍

换个角度看：与其每次都重新选起点，不如维护一个窗口 [left, right]，永远保持窗口里没有重复字符。 right 一格一格往右走，把新字符纳入窗口；

同时用一个哈希表记录每个字符最后一次出现的下标。 扫到 s[right] 时，如果这个字符在哈希表里，并且 那次出现还落在当前窗口 [left, right] 内，说明窗口里有重复了—— 直接把 left 跳到那次出现的下一格，窗口立刻恢复无重复； 否则窗口照常扩大。每一步 right − left + 1 就是「以 right 结尾的最长无重复子串长度」， 扫完一遍取这些长度的最大值就是答案。left 全程只往右移、不会后退， 所以整个数组只需要扫一遍。

当前窗口 [left, right] 当前 right 指针 触发跳跃的重复字符 最终最长窗口

试试把字符串改成 pwwkew，看 left 怎么从 0 一路跳到 2，最终找出 "wke"。

☕ 第四步：写成 Java 代码

class Solution {
    public int lengthOfLongestSubstring(String s) {
        // lastSeen：字符 -> 它最后一次出现的下标
        Map<Character, Integer> lastSeen = new HashMap<>();
        int left = 0, maxLen = 0;
        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
                // 重复字符还在窗口内，左边界跳到它的下一格
                left = lastSeen.get(c) + 1;
            }
            lastSeen.put(c, right);
            maxLen = Math.max(maxLen, right - left + 1);
        }
        return maxLen;
    }
}
时间复杂度 O(n)

right 只往右走一遍，left 也全程单调右移，每次查表/存表都是 O(1)。

空间复杂度 O(min(n, Σ))

哈希表最多存下字符集大小 Σ（如 ASCII 是 128）个字符，或字符串长度 n，取较小者。

🚀 小结

套路：当你需要在一维序列里找「满足某种连续性质的最长/最短区间」时， 先想想能不能维护一个左右边界单调移动的窗口，配合哈希表/计数器 快速判断「窗口是否还合法」——这就是滑动窗口，能把许多 看似需要 O(n²) 枚举的问题压到 O(n)。
