一道关于「原地操作」的经典矩阵题：不许新开矩阵，怎么把图像顺时针转 90 度？ 答案是两步小动作——先沿对角线「翻个身」，再把每一行「左右对调」。

🧩 第一步：题目到底要什么

给你一个 n × n 的方阵 matrix，把它当成一张图片，要求把这张图片 顺时针旋转 90 度。最关键的限制是：不能新建一个矩阵， 必须在原地直接改 matrix 本身。

matrix[[1,2,3],[4,5,6],[7,8,9]]

顺时针转 90° 后，原来第一列 [1,4,7] 倒过来变成新的第一行 [7,4,1]， 结果是 [[7,4,1],[8,5,2],[9,6,3]]。

🐢 第二步：先想最直白的办法，再想为什么不能这么用

最直观的做法：新开一个同样大小的矩阵 res，按公式 res[j][n-1-i] = matrix[i][j] 把每个格子填到旋转后该去的位置， 填完把 res 拷回 matrix。这个公式是对的，但它用了 O(n²) 的额外空间——题目明确不让用。

那怎么「原地」做？顺时针旋转 90° 其实可以拆成两个更简单、 互不冲突的小操作的组合：转置（沿主对角线把 matrix[i][j] 和 matrix[j][i] 互换）之后，把每一行左右翻转。 这两步都只是「交换两个格子的值」，不需要额外的整块内存，空间 O(1)。

⚡ 第三步：转置 + 逐行翻转，动画演示

为什么「转置 + 逐行翻转」等于「顺时针转 90°」？转置之后，第 i 行的内容， 正好变成了旋转前第 i 列从上到下的内容；而顺时针旋转其实就是要让原来的列 倒过来变成新的行——所以只要把转置后的每一行再左右翻转一次， 顺序就对上了。

试试把矩阵改成 5,1,9,11;2,4,8,10;13,3,6,7;15,14,12,16（4×4）， 看同样的两步是怎么把它转过来的。

☕ 第四步：写成 Java 代码

class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;

        // 第一步：转置（沿主对角线对称交换）
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                int tmp = matrix[i][j];
                matrix[i][j] = matrix[j][i];
                matrix[j][i] = tmp;
            }
        }

        // 第二步：每一行左右翻转
        for (int i = 0; i < n; i++) {
            int l = 0, r = n - 1;
            while (l < r) {
                int tmp = matrix[i][l];
                matrix[i][l] = matrix[i][r];
                matrix[i][r] = tmp;
                l++;
                r--;
            }
        }
    }
}
时间复杂度 O(n²)

转置要碰约一半格子，逐行翻转再碰一遍全部格子，总量仍是 n² 级别。

空间复杂度 O(1)

只用了几个临时变量做交换，没有新开任何矩阵或数组。

🚀 小结

套路：「原地」类矩阵变换题，常常可以拆成几个简单、互不依赖的 对称交换步骤的组合——本题是「转置 + 逐行翻转」， 逆时针旋转 90° 则是「转置 + 逐行翻转」换成「转置 + 逐列翻转（或先翻转再转置）」。 记住这种拆解思路，比死记一个旋转公式更管用。
