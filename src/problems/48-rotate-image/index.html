一道关于「原地操作」的经典矩阵题：不许新开一个矩阵当草稿纸，怎么把图像顺时针转 90 度？ 答案是两步小动作——先沿对角线「翻个身」，再把每一行「左右对调」。

🧩 第一步：题目到底要什么

给你一个 n × n 的方阵 matrix（也就是行数和列数相等的表格， 比如 3×3、4×4），你可以把它想成一张正方形的照片，表格里每个格子就是一个像素点。 题目要求把这张「照片」顺时针旋转 90 度——想象你拿着一张正方形照片， 顺时针转个直角，原来朝上的那条边现在转到右边去了。

最关键的限制是：不能新建一个矩阵，必须在原地 直接改 matrix 本身。「原地」是什么意思？就是不许再找一个同样大小的新 表格来当草稿纸、把答案写好了再搬过去；你手里只有这一张表格，只能在它自己身上 把格子里的数字挪来挪去、直接调换位置，最多允许借用几个变量来临时存一下数字 （就像倒水时临时借用一个小杯子，而不是再买一整个新水桶）。

matrix[[1,2,3],[4,5,6],[7,8,9]]

顺时针转 90° 后，原来第一列 [1,4,7] 倒过来变成新的第一行 [7,4,1]， 结果是 [[7,4,1],[8,5,2],[9,6,3]]。不信的话可以拿手边一张写了数字的纸， 实际转一转，看看每条边是怎么跑位的。

🐢 第二步：先想最直白的办法，再想为什么不能这么用

最直观的做法：新开一个同样大小的矩阵 res 当草稿纸，按公式 res[j][n-1-i] = matrix[i][j] 把每个格子填到旋转后该去的位置， 填完把 res 拷回 matrix。这个公式是对的——但它需要额外 整整开一个 n × n 的新表格来周转，也就是用了 O(n²) 的额外空间，而题目明确不让用。

那怎么「原地」做？关键洞察是：顺时针旋转 90° 其实可以拆成两个 更简单、互不冲突的小动作，一步步直接在原来那张表格上完成，完全不用借新表格。

第一个动作叫「转置」：把矩阵沿着主对角线翻个身。 主对角线就是从左上角一路连到右下角的那条斜线，也就是 matrix[0][0]、 matrix[1][1]、matrix[2][2]……这些「行号等于列号」的格子 连起来的线。转置的做法就是：把这条线两边、镜像对称的两个格子互相交换位置—— matrix[i][j] 换到 matrix[j][i] 的位置， matrix[j][i] 换到 matrix[i][j] 的位置。 拿例子里的 [[1,2,3],[4,5,6],[7,8,9]] 来说：2 和 4 换位置， 3 和 7 换位置，6 和 8 换位置，恰好落在对角线上的 1、5、9 不用动， 结果变成 [[1,4,7],[2,5,8],[3,6,9]]——你会发现原来的每一列， 现在原样变成了每一行。

第二个动作是「把每一行左右翻转」：把转置后的每一行首尾对调 （第一个和最后一个换、第二个和倒数第二个换，一路往中间夹）。接着上面的例子， 转置后的第一行 [1,4,7] 左右翻转后变成 [7,4,1]， 正好就是我们最终想要的旋转结果的第一行。

这两步全程都只是「交换两个格子里的值」，用不上任何一整块的新内存， 空间只要 O(1)（几个临时变量而已）。

⚡ 第三步：转置 + 逐行翻转，动画演示

再串一遍为什么「转置 + 逐行翻转」正好等于「顺时针转 90°」：转置之后， 第 i 行的内容，正好变成了旋转前第 i 列从上到下的内容； 而顺时针旋转其实就是要让原来的列倒过来变成新的行—— 所以只要把转置后的每一行再左右翻转一次，顺序就对上了。下面这个动画你可以亲手拖着看， 每一次高亮的两个格子就是当前正在互换的一对。

☕ 第四步：写成 Java 代码

class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;

        // 第一步：转置（沿主对角线对称交换）
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                int tmp = matrix[i][j];
                matrix[i][j] = matrix[j][i];
                matrix[j][i] = tmp;
            }
        }

        // 第二步：每一行左右翻转
        for (int i = 0; i < n; i++) {
            int l = 0, r = n - 1;
            while (l < r) {
                int tmp = matrix[i][l];
                matrix[i][l] = matrix[i][r];
                matrix[i][r] = tmp;
                l++;
                r--;
            }
        }
    }
}
时间复杂度 O(n²)

转置要碰约一半格子，逐行翻转再碰一遍全部格子，总量仍是 n² 级别。

空间复杂度 O(1)

只用了几个临时变量做交换，没有新开任何矩阵或数组。

🚀 小结

套路：「原地」类矩阵变换题，常常可以拆成几个简单、互不依赖的 对称交换步骤的组合——本题「顺时针转 90°」拆成的是 「转置 + 逐行翻转」；如果要求的是「逆时针转 90°」，把第二步换成 「逐列翻转」（或者反过来，先翻转、再转置）就行了。 记住这种拆解思路，比死记一个旋转公式更管用。
