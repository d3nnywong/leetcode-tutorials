一堆区间互相打架，最少删掉几个才能让剩下的相安无事？这题是「贪心」算法 最经典的入门例子之一：按结束时间排序，谁结束得早就优先留下谁。

🧩 第一步：题目到底要什么

给你一堆区间 [start, end]，有些会互相重叠。你要移除最少数量 的区间，让剩下的区间两两之间都不重叠（只在一个点上挨着，比如 [1,2] 和 [2,3]，算不重叠）。

输入[[1,2],[2,3],[3,4],[1,3]]

[1,3] 同时和 [1,2]、[2,3]、[3,4] 都有重叠，只要把它移除，剩下 [1,2]、[2,3]、[3,4] 就严丝合缝、互不重叠了。所以只需要移除 1 个。

🐢 第二步：换个角度想——「移除最少」等于「保留最多」

直接想「该删哪些区间」很容易绕进死胡同：删了一个，剩下的重叠关系还会变。 换个角度：移除的区间数量 = 总数 − 保留的区间数量，所以「移除最少」 完全等价于「保留最多互不重叠的区间」——这就是经典的活动选择问题。

关键洞察：如果只能保留一个，应该留结束得最早的那个—— 它占用的「地盘」最小，给后面留出的空间最多，越不容易和别人冲突。所以先把所有区间按 结束时间从小到大排序，然后贪心地、能留就留：只要当前区间和已保留的 最后一个不冲突，就留下它。

⚡ 第三步：排序 + 贪心扫描，一遍搞定

排好序后，维护一个变量 lastEnd，表示「最后一个被保留区间的结束点」 （一开始还没保留任何区间，可以看成 −∞）。从左到右依次看当前区间 cur：

  • 如果 cur 的起点 ≥ lastEnd，说明和已保留的 区间不重叠 → 保留它，把 lastEnd 更新成它的结束点。
  • 如果 cur 的起点 < lastEnd，说明重叠了， 两者只能留一个 → 当然留结束更早的那个（已经保留的那个一定结束更早，因为按结束时间 排过序），所以移除当前这个 cur，移除计数 +1， lastEnd 不变。

扫完一遍，移除计数就是答案。

当前正在判断的区间 已保留 已移除 最终保留下来的区间 / 边界线

试试改成 1,2;1,2;1,2，看三个完全重复的区间怎么留一个、移除两个； 再试试 1,2;2,3，看首尾刚好相接（不算重叠）时一个都不用移除。

☕ 第四步：写成 Java 代码

class Solution {
    public int eraseOverlapIntervals(int[][] intervals) {
        if (intervals.length == 0) return 0;

        // 按结束时间从小到大排序：结束得越早，留给后面的空间越多
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[1], b[1]));

        int removed = 0;
        int lastEnd = Integer.MIN_VALUE; // 最后一个保留区间的结束点

        for (int[] cur : intervals) {
            if (cur[0] >= lastEnd) {
                // 不重叠，保留它，更新边界
                lastEnd = cur[1];
            } else {
                // 重叠了，留结束更早的那个，移除当前这个
                removed++;
            }
        }
        return removed;
    }
}
时间复杂度 O(n log n)

瓶颈在按结束时间排序；之后从左到右扫一遍只要 O(n)。

空间复杂度 O(log n) ~ O(n)

排序所需的额外空间，没有用到别的数据结构。

🚀 小结

套路：「移除最少的区间使剩余互不重叠」本质就是「保留最多互不重叠的区间」， 也就是活动选择问题——按结束时间排序后贪心保留，是这一类问题的标准解法。 它和合并区间（56 题）、会议室（252/253 题）同属「排序 + 贪心/扫描」家族，但排序的 关键字不同：合并区间按起点排序是为了让重叠的区间挨在一起；这里按 结束点排序，是为了优先保留「占地最小」的区间，从而保留得最多。
