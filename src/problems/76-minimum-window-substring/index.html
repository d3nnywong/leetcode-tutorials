滑动窗口里最考验功力的一道困难题。难点不在「扩窗口」，而在想清楚—— 什么时候才算「覆盖了」，以及覆盖之后为什么要反过来去「收缩」。

🧩 第一步：题目到底要什么

给你两个字符串 s 和 t，要你在 s 里面找一段 最短的连续子串，这段子串里要把 t 的每一个字符都「凑齐」—— t 里出现几次，子串里就至少要有几次（多了没关系，顺序也不限）。

s"ADOBECODEBANC"
t"ABC"

子串 "BANC" 里有 A、B、C 各一个，凑齐了 t 的要求， 而且是所有满足条件的子串里最短的 → 答案 "BANC"。

🐢 第二步：最直接的笨办法

最容易想到的：枚举 s 里所有的子串（起点、终点各一层循环）， 每个子串都拿去检查一遍「是否覆盖了 t」，记录最短的那个。

慢在哪：子串数量是 O(m²)，每次检查覆盖还要再扫一遍， 整体 O(m² · n)。m 能到 10⁵，这样肯定超时。 问题在于——我们对「窗口换大换小」这件事，每次都从头检查。

⚡ 第三步：先扩窗口找到「能覆盖」，再缩窗口找「最短」

换个思路：维护一个窗口 [left, right]，并且用两张「计数表」记账——

  • need：t 里每个字符要出现几次（固定不变）。
  • window：当前窗口里每个字符出现了几次（随窗口变化）。

再用一个 valid 记录「窗口里已经凑够数量的字符种类数」。 只要 right 往右扩，窗口只会变得更「富裕」，valid 只增不减； 一旦 valid 等于 t 的字符种类数，说明当前窗口已经覆盖了 t。

这时候窗口里很可能有「多余」的部分（比如两端是不需要的字符），于是反过来 收缩 left：每往右移一格 left，就检查一次「现在还覆盖吗」， 覆盖就继续收缩并刷新最短答案，一旦收缩到不再覆盖就停下，让 right 继续扩。 left 和 right 各自只会从头走到尾一次，整体 O(m + n)。

当前窗口 [left, right] L / R 指针所在格 刚移出窗口的字符 已凑够数量的字符 最终答案窗口

试试把 s 改成 "a"、t 改成 "aa"，看它怎么扩到底也凑不齐、最终返回空字符串。

☕ 第四步：写成 Java 代码

class Solution {
    public String minWindow(String s, String t) {
        int[] need = new int[128];
        for (char c : t.toCharArray()) need[c]++; // t 里每个字符要出现几次
        int required = 0; // t 里有多少种不同字符
        for (int cnt : need) if (cnt > 0) required++;

        int[] window = new int[128]; // 当前窗口里每个字符出现了几次
        int valid = 0; // 窗口里「已凑够数量」的字符种类数
        int left = 0, bestLen = Integer.MAX_VALUE, bestStart = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            if (need[c] > 0) {
                window[c]++;
                if (window[c] == need[c]) valid++; // 这种字符刚好凑够了
            }
            // 窗口已覆盖 t：尝试收缩 left，找更短的可行窗口
            while (valid == required) {
                if (right - left + 1 < bestLen) {
                    bestLen = right - left + 1;
                    bestStart = left;
                }
                char d = s.charAt(left);
                if (need[d] > 0) {
                    if (window[d] == need[d]) valid--; // 移出后这种字符不够了
                    window[d]--;
                }
                left++;
            }
        }
        return bestLen == Integer.MAX_VALUE ? "" : s.substring(bestStart, bestStart + bestLen);
    }
}
时间复杂度 O(m + n)

right 从头扫到尾一次，left 总共也最多移动 m 步；统计 t 是 O(n)。

空间复杂度 O(1)

计数数组大小固定为 128（ASCII 英文字母范围），与输入长度无关。

🚀 小结

套路：当题目要求「找一个满足某种覆盖/计数条件的最短（或最长）连续区间」时， 先想滑动窗口——用 right 扩张去「凑条件」，条件一旦满足就反过来用 left 收缩去「榨干多余」，全程只用一个计数表判断条件成立与否， 避免每次都重新扫一遍区间。这正是双指针「单调移动」思想在字符串题上的延伸。
