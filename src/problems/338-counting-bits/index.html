这题把「二进制」和「动态规划」放在一起考。别慌——二进制只是计算机数数的方式， 动态规划则是一种「用算过的结果直接推出新答案、绝不重复劳动」的思路，下面都会从头讲。 看完你会发现：算一个数二进制里有几个 1，根本不用从头数——它的答案就藏在「比它小一半」的那个数里。

🧩 第一步：题目到底要什么

先说清楚什么是「二进制」：我们平时数数用的是十进制，逢十进一， 用 0～9 这十个数字；计算机里的数字则记成二进制，只有 0 和 1 两个数字，逢二进一。比如十进制的 5，写成二进制就是 101（也就是 4 + 0 + 1）；十进制的 3 写成二进制是 11（也就是 2 + 1）。任何一个整数，都能唯一地写成这样一串 0 和 1。

这道题要你做的是：给你一个整数 n，把 0 到 n 这每一个数都 写成二进制，数一数它里面有几个 1，把这些「1 的个数」按顺序放进一个数组 ans 里返回。

n5

0 = 0（0 个 1）；1 = 1（1 个）；2 = 10（1 个）； 3 = 11（2 个）；4 = 100（1 个）；5 = 101（2 个） → 返回 [0,1,1,2,1,2]。

🐢 第二步：先想最笨的办法，再找规律

最直接的想法：对 0 到 n 里的每一个数，都老老实实地把它变成二进制，然后一位一位地去数有几个 1 （比如反复看这个数除以 2 的余数）。这样能做出来，但每个数都要单独「数一遍」， n 个数、每个数最多有 log n 位（也就是二进制的位数，n 越大位数越多，但涨得很慢）， 总共要花 O(n log n) 的力气 —— 这里的 O(...) 只是「大致要算多少步」的一种写法， 括号里的式子越大代表越慢。

问题出在哪：我们把每个数都当成全新的、互不相干的问题来算， 完全没有利用「前面已经算过的数」。其实数字之间是有联系的， 这就要请出两个「位运算」——也就是直接对二进制的每一位动手的运算：

  • i >> 1（读作「右移一位」）：把 i 的二进制去掉最后一位， 相当于把整个数「除以 2 再抹掉小数」。比如 6 是二进制 110， 右移一位变成 11，也就是 3。
  • i & 1（读作「按位与」，这里用来取最后一位）：只看 i 的二进制 最后一位是 0 还是 1。比如 6（110）的最后一位是 0， 7（111）的最后一位是 1。

关键洞察来了：i >> 1 这个数一定比 i 小， 也就是说它的答案早就算好了——我们完全不用把 i 当成一个全新的数从头数！

于是：把 i 的二进制「砍掉最后一位」，剩下的部分就是 i >> 1； 被砍掉的那一位，要么是 0 要么是 1，正好等于 i & 1。 所以 i 里 1 的个数 = i >> 1 里 1 的个数，再加上被砍掉那一位是不是 1。

🧠 第三步：动态规划三件套

这里要用到「动态规划」（英文常缩写成 DP）：说白了就是把一个大问题拆成一串从小到大的小问题， 把每个小问题的答案存进一张表里；后面要用到某个小问题的答案时，直接查表拿现成的， 绝不重新计算一遍。我们按「状态 → 转移 → 边界」这三步，把这张表搭出来：

1

状态：dp[i] 表示什么？

dp[i] = 整数 i 的二进制表示里，1 的个数。

最终答案就是整个 dp[0..n] 数组，直接作为 ans 返回。

2

转移：dp[i] 怎么由更小的问题推出来？

照上面的洞察，把 i 的最低位砍掉得到 i >> 1（一定比 i 小，已经算好）， 再看被砍掉的那一位 i & 1 是不是 1：

dp[i] = dp[i >> 1] + (i & 1)（i ≥ 1）

3

边界：最小的情况是什么？

dp[0] = 0：0 的二进制全是 0，里面没有任何 1，这是唯一不用推导、直接给定的地基。

下面这个动画从 i = 0 开始一格一格往后算：当前在算的 i 会高亮， 它依赖的来源格 i >> 1 会标成来源色，算完把结果填进格子里， 整条数组只需要从左到右扫一遍。

当前正在算的 dp[i] 依赖的来源 dp[i >> 1]

试试把 n 改成 8，留意 i = 4（100）和 i = 8（1000） 这种「2 的幂」——它们都只来自 dp[0]，所以 dp 值永远是 1。

☕ 第四步：写成 Java 代码

转移方程只依赖 dp[i >> 1]，而 i >> 1 < i 永远成立， 所以只要从小到大顺序填一遍数组，每个位置都能直接查到它需要的来源，一趟扫描即可。 

class Solution {
    public int[] countBits(int n) {
        int[] dp = new int[n + 1]; // dp[0] = 0，数组默认初始化为 0，正好是边界
        for (int i = 1; i <= n; i++) {
            // i >> 1 比 i 小，已经算好了；i & 1 是被砍掉的最低位
            dp[i] = dp[i >> 1] + (i & 1);
        }
        return dp;
    }
}
时间复杂度 O(n)

从 1 到 n 顺序扫一遍，每个位置 O(1) 计算。

空间复杂度 O(n)

结果数组本身就要占 O(n)，不算它的话额外空间是 O(1)。

🚀 小结

套路：比特位计数是「用已经算好的更小问题拼出当前答案」这类一维动态规划的代表题—— 一个数的二进制信息，可以拆成「去掉最后一位剩下的更小的数」加「这一位是不是 1」两部分。 以后遇到跟二进制、位运算相关的动态规划题，都可以想想：能不能通过右移一位， 把当前问题变成一个更小、已经算过的同类问题？记住 i >> 1 永远比 i 小，这条思路就有了落脚点。
