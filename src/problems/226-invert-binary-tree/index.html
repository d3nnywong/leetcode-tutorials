一道经典的「递归入门题」。难点不是代码量，而是要相信递归—— 只要管好「当前这一个节点」该干什么，剩下的全部交给递归自己去完成。

🧩 第一步：「翻转」到底是个什么操作

翻转二叉树，就是把它变成镜像对称的样子——每一个节点的 左孩子和右孩子互换位置，而且不只是根节点，每一层、每一个节点 都要这么换。

root[4,2,7,1,3,6,9]

根节点 4 的左孩子 2、右孩子 7 互换；2 的左孩子 1、右孩子 3 互换；7 的左孩子 6、右孩子 9 互换……整棵树左右镜像之后变成 [4,7,2,9,6,3,1]。

🔑 第二步：把大问题拆成「一样的小问题」

直接去想「整棵树怎么镜像」会很绕。换个角度——翻转一棵树，其实可以拆成三句话：

  • 树是空的？什么都不用做，直接返回（递归的终止条件）。
  • 不是空的？先把当前节点的左右孩子换一下位置。
  • 换完之后，新的左子树和新的右子树各自也要被翻转—— 而「翻转一棵子树」和「翻转整棵树」是同一件事，所以直接对它们调用 同一个函数就行，这就是递归。
不用一次想清楚全部层级：你只需要相信——只要每个节点都老老实实 「交换左右孩子、再把同样的事丢给两个孩子去做」，递归会自动把这件事铺满整棵树， 一路传到最深的叶子节点为止。

⚡ 第三步：看动画——前序递归，一边交换一边往下走

动画按前序遍历的顺序展开递归：进入一个节点 → 交换它的左右孩子 → 再分别递归进入「新的左子树」「新的右子树」。 注意连线两端的节点其实从未改变，变的只是某个节点把谁当成左孩子、谁当成右孩子—— 所以你会看到两棵子树在交换的瞬间，左右换了个位置。

正在访问的节点 正在交换左右孩子 叶子节点（无需交换） 全部翻转完成

试试把输入改成 2,1,3，或者输入 [] 看看空树的特殊情况。

☕ 第四步：写成 Java 代码

class Solution {
    public TreeNode invertTree(TreeNode root) {
        // 终止条件：空节点直接返回（递归到叶子的下一层时触发）
        if (root == null) {
            return null;
        }

        // 单层逻辑：交换当前节点的左右孩子
        TreeNode tmp = root.left;
        root.left = root.right;
        root.right = tmp;

        // 子问题：对换完之后的新左子树、新右子树做同样的事
        invertTree(root.left);
        invertTree(root.right);

        return root;
    }
}
时间复杂度 O(n)

每个节点恰好被访问一次，做一次常数时间的交换。

空间复杂度 O(h)

h 是树的高度，来自递归调用栈；最坏情况（树退化成链）是 O(n)，平衡树是 O(log n)。

🚀 小结

套路：看到「对树的每个节点都做同一件事」，先想递归三件套—— 终止条件、当前节点要做的事、把剩下的丢给子问题。这题的子问题恰好就是「翻转一棵（更小的）树」， 和原问题一模一样，递归用起来特别顺。如果不想用递归，也可以用 BFS（队列）或显式栈 把所有节点取出来逐个交换左右孩子，思路完全一样，只是把「调用栈」换成了自己维护的队列/栈。
