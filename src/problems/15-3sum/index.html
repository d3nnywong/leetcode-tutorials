两数之和的「升级版」：从找两个数变成找三个数。核心套路是先把数组排序， 再把其中一个数当「锚点」，剩下的就退化成你已经会的双指针找两数之和。

🧩 第一步：题目到底要什么

给你一个整数数组 nums，要你找出所有「三个数加起来等于 0」 的组合（不是下标，是数值组成的三元组），而且不能有重复的组合。

nums[-1, 0, 1, 2, -1, -4]

-1 + 0 + 1 = 0，-1 + (-1) + 2 = 0 → 答案是 [[-1,-1,2],[-1,0,1]]， 顺序不重要，但每一组三元组内部和组与组之间都不能重复。

🐢 第二步：最直接的笨办法

最容易想到的：三层循环枚举所有 i, j, k，看哪些组合的和为 0， 再用一个集合去掉重复的三元组。这是 O(n³)，数组长度能到 3000， 三次方就是 270 亿次，跑不动。

慢在哪：每确定一个 i，后面两个数还要再来一次「暴力配对」。 但「两个数加起来等于某个目标值」——这不就是两数之和吗？我们已经有更快的办法了。

⚡ 第三步：排序后，退化成「双指针版两数之和」

先把数组从小到大排序。排序带来两个好处：相同的数会挨在一起（方便去重）， 而且排好序后可以用双指针从两端向中间夹逼。

固定第一个数 nums[i] 当「锚点」，问题就变成：在 i 右边的区间里， 找两个数 nums[l] + nums[r] = -nums[i]。这正是双指针解两数之和的套路：

  • 三数之和 等于 0 → 记录这组答案，左右指针各跳过和刚才相同的值，再继续夹逼（去重）。
  • 三数之和 小于 0 → 太小，说明左边的数偏小，左指针右移找更大的数。
  • 三数之和 大于 0 → 太大，右指针左移找更小的数。

另外两个剪枝：排序后如果 nums[i] > 0，后面的数只会更大，可以直接结束； 如果 nums[i] 和上一个锚点相同，跳过它，否则会算出重复的三元组。

锚点 nums[i] 左右指针 l / r 命中的三元组 跳过重复 / 提前结束

试试把数组改成 0,0,0 或 0,1,1，看它怎么处理「正好一组」和「无解」。

☕ 第四步：写成 Java 代码

import java.util.*;

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums); // 排序后才能用双指针、方便跳过重复值
        List<List<Integer>> ans = new ArrayList<>();
        int n = nums.length;
        for (int i = 0; i < n - 2; i++) {
            if (nums[i] > 0) break; // 最小的数都大于 0，后面只会更大
            if (i > 0 && nums[i] == nums[i - 1]) continue; // 跳过重复锚点

            int l = i + 1, r = n - 1;
            while (l < r) {
                int sum = nums[i] + nums[l] + nums[r];
                if (sum == 0) {
                    ans.add(Arrays.asList(nums[i], nums[l], nums[r]));
                    l++; r--;
                    while (l < r && nums[l] == nums[l - 1]) l++; // 跳过重复左值
                    while (l < r && nums[r] == nums[r + 1]) r--; // 跳过重复右值
                } else if (sum < 0) {
                    l++; // 太小，左指针右移找更大的数
                } else {
                    r--; // 太大，右指针左移找更小的数
                }
            }
        }
        return ans;
    }
}
时间复杂度 O(n²)

排序 O(n log n)；外层枚举锚点 O(n)，内层双指针再扫一遍 O(n)，合计 O(n²)。

空间复杂度 O(log n) ~ O(n)

不计返回结果的话，主要是排序所用的栈空间；返回的三元组列表本身大小不计入复杂度。

🚀 小结

套路：遇到「找 k 个数满足某种和」的题，常见思路是先排序， 再固定前 k-2 个数，把问题降维成「双指针找两数之和」。 三数之和把两数之和从 O(n) 升级成了「固定一维 + 双指针扫一维」的 O(n²)， 四数之和、最接近的三数之和都是同一个套路的变体。
