这题是「单词搜索 I」的加强版：网格不变，单词从一个变成成千上万个。 看完你会明白：把所有单词揉进一棵 Trie（字典树）， 再配合回溯 DFS 一起剪枝，怎么把本来要跑无数遍的搜索压缩成一次。

🧩 第一步：题目到底要什么

给你一个 m × n 的字母网格，和一份单词列表 words。 要从这堆单词里挑出所有能在网格里拼出来的——从某个格子出发， 每步只能走上下左右相邻的格子，同一个格子在同一个单词里 不能用两次。

board[["o","a","a","n"],["e","t","a","e"],["i","h","k","r"],["i","f","l","v"]]
words["oath","pea","eat","rain"]

从 (0,0)=o 出发，向右走到 a、向下走到 t、再向下走到 h，拼出 oath； "pea" 和 "rain" 在这个网格里怎么走都拼不出来 → 答案是 ["eat","oath"]。

🐢 第二步：先想最直接的办法，再看它浪费在哪

这题本质上是「单词搜索 I」（判断一个单词能不能在网格里拼出来）的 DFS：从每个格子出发，沿着单词的字母一步步走，走不通就回溯换方向。 最直接的做法就是把 words 里的每个单词都单独跑一次这个 DFS。

慢在哪：题目里 words.length 能到 3×10⁴， 每个单词都要在网格上独立搜一遍，等于把同一个网格扫了几万遍。 更可惜的是，很多单词共享同一个前缀（比如都以 "th" 开头），这些重复的前缀 在每次独立搜索里都被从头走一遍，白白浪费。

关键洞察：能不能把所有单词「该往哪走」这件事合并起来， 只对网格做一次 DFS，搜的过程中同时检查它是不是凑成了某个单词？

⚡ 第三步：把单词列表压成一棵 Trie，再配合回溯剪枝

先把 words 里的所有单词一起插进一棵 Trie（字典树）： 公共前缀只存一份，每个单词的最后一个字母所在节点上记一个「这里是某个单词的结尾」标记。

然后对网格做一次回溯 DFS：从每个格子出发，每走一步都问 Trie—— “当前节点有没有这个字母的分支？”

  • 没有 → 哪怕只差一个字母，也说明这条路所有单词都凑不出来，直接剪掉，不用再往下试。
  • 有 → 往下走，如果正好落在某个单词的结尾标记上，就说明找到了它；继续探四个方向，看 Trie 这个节点还能不能往下接。

走过的格子临时标记为「已访问」（防止同一个单词重复用同一个格子）， 一条路走到头（或被剪枝）之后回溯还原，换下一个方向继续试。 因为剪枝依据是共享的 Trie，所有单词的搜索被天然地揉进了同一次 DFS。

当前递归路径上的格子 正在访问的格子 恰好拼出一个单词的路径 已经找到的单词

虚线格子表示「Trie 根节点没有这个字母的分支」，DFS 看都不看一眼就跳过—— 这正是 Trie 剪枝省下来的功夫。试试把 words 改成 oath,oat,rain，看共享前缀 "oa" 怎么只被走一次就同时帮两个单词探路。

☕ 第四步：写成 Java 代码

class Solution {
    public List<String> findWords(char[][] board, String[] words) {
        TrieNode root = buildTrie(words);
        List<String> result = new ArrayList<>();
        int m = board.length, n = board[0].length;
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                dfs(board, i, j, root, result);
            }
        }
        return result;
    }

    // 从 (i, j) 出发，沿着 Trie 的 node 节点往下走
    private void dfs(char[][] board, int i, int j, TrieNode node, List<String> result) {
        char ch = board[i][j];
        // 已访问过（标记成 '#'）或 Trie 没有这个分支 → 剪枝，直接返回
        if (ch == '#' || node.children[ch - 'a'] == null) return;

        TrieNode next = node.children[ch - 'a'];
        if (next.word != null) {
            result.add(next.word);
            next.word = null; // 避免同一个单词被不同路径重复加入
        }

        board[i][j] = '#'; // 标记已访问
        int[] dx = {-1, 1, 0, 0};
        int[] dy = {0, 0, -1, 1};
        for (int d = 0; d < 4; d++) {
            int ni = i + dx[d], nj = j + dy[d];
            if (ni >= 0 && ni < board.length && nj >= 0 && nj < board[0].length) {
                dfs(board, ni, nj, next, result);
            }
        }
        board[i][j] = ch; // 回溯：把字母还原
    }

    private TrieNode buildTrie(String[] words) {
        TrieNode root = new TrieNode();
        for (String w : words) {
            TrieNode node = root;
            for (char c : w.toCharArray()) {
                int idx = c - 'a';
                if (node.children[idx] == null) node.children[idx] = new TrieNode();
                node = node.children[idx];
            }
            node.word = w; // 单词结尾节点记下完整单词
        }
        return root;
    }

    class TrieNode {
        TrieNode[] children = new TrieNode[26];
        String word = null;
    }
}
时间复杂度 O(W + m·n·4^L)

建 Trie 是 W（所有单词总长度）；DFS 最坏情况下从每个格子出发都要探索 4^L 条路径（L 是最长单词长度），但因为剪枝， 远比「逐个单词独立搜索」的 O(words.length · m·n·4^L) 快得多。

空间复杂度 O(W + L)

Trie 节点总数最多是所有单词的总长度 W；递归栈深度最多是单词最大长度 L。

🚀 小结

套路：当你要在同一份数据上对很多个模式串分别做搜索/匹配时， 先想想能不能把这些模式串合并成一棵 Trie——公共前缀只需要走一次， 剪枝判断也变成 O(1) 的「查孩子节点存不存在」。 Trie 与 DFS/回溯 的组合是「网格 + 多模式串」这类题的标准搭档， 除了单词搜索，还能用在自动补全、敏感词过滤等场景。
