一道「按方向走格子」的模拟题：把矩阵想象成一圈圈的「画框」， 顺时针一圈圈往里画，画完最外圈就缩小范围继续画里面一圈。

🧩 第一步：题目到底要什么

给你一个 m 行 n 列的矩阵，要你按 顺时针螺旋的顺序把所有元素「读出来」，拼成一维数组返回。

matrix[[1,2,3],[4,5,6],[7,8,9]]

从左上角 1 开始：先往右走到 3，再往下走到 9，再往左走到 7，再往上走到 4， 最后只剩中间的 5 → [1,2,3,6,9,8,7,4,5]。

🔑 第二步：怎么不写歪、不重复、不漏格

直接「模拟走路」容易写错：什么时候该转弯？转到第几圈算遍历完？ 更稳的办法是用四个边界把「还没访问的区域」框起来： top（上边界行）、bottom（下边界行）、 left（左边界列）、right（右边界列）。

每一圈固定走 4 步：

  • 沿 top 行从 left 走到 right → 这一行用完了，top++
  • 沿 right 列从 top 走到 bottom → 这一列用完了，right--
  • 如果 top <= bottom，沿 bottom 行从 right 走到 left → bottom--
  • 如果 left <= right，沿 left 列从 bottom 走到 top → left++
为什么要判断 top<=bottom / left<=right： 矩阵不是正方形、或者走到最后只剩一行/一列时，前两步可能已经把这一圈走完了， 不加这两个判断会把同一行/同一列重复走一遍。四个边界一旦「框」收缩到不存在 （top>bottom 或 left>right），就说明全部访问完了。

☕ 第四步：写成 Java 代码

class Solution {
    public List<Integer> spiralOrder(int[][] matrix) {
        List<Integer> res = new ArrayList<>();
        if (matrix == null || matrix.length == 0) return res;

        // 四个边界，框住「还没访问的区域」
        int top = 0, bottom = matrix.length - 1;
        int left = 0, right = matrix[0].length - 1;

        while (top <= bottom && left <= right) {
            // 1) 沿上边界从左走到右
            for (int j = left; j <= right; j++) res.add(matrix[top][j]);
            top++;

            // 2) 沿右边界从上走到下
            for (int i = top; i <= bottom; i++) res.add(matrix[i][right]);
            right--;

            // 3) 沿下边界从右走到左（这一行还在，没被走过才走）
            if (top <= bottom) {
                for (int j = right; j >= left; j--) res.add(matrix[bottom][j]);
                bottom--;
            }

            // 4) 沿左边界从下走到上（这一列还在，没被走过才走）
            if (left <= right) {
                for (int i = bottom; i >= top; i--) res.add(matrix[i][left]);
                left++;
            }
        }
        return res;
    }
}
时间复杂度 O(m·n)

每个格子恰好被访问一次。

空间复杂度 O(1)

不算返回结果本身，只用了 4 个边界变量。

🚀 小结

套路：遇到「按某种顺序走遍矩阵/网格」的题， 先想清楚用什么状态描述「还没走完的区域」（这里是四个边界）， 再把一圈拆成固定的几步直线移动，每走完一条边就收缩对应的边界。 这个「边界收缩」的思路在螺旋矩阵 II（59 题，反过来填数字）里也完全适用。
