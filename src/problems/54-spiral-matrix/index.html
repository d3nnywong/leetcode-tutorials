一道「按方向走格子」的模拟题：把矩阵想象成一圈圈的「画框」， 顺时针一圈圈往里画，画完最外圈就缩小范围继续画里面一圈。

🧩 第一步：题目到底要什么

先说清楚「矩阵」是什么：它就是一张有 m 行、n 列的表格 （跟 Excel 表格差不多），每个格子里放一个数字。题目要你按 顺时针螺旋的顺序——也就是先往右走、再往下走、再往左走、再往上走， 跟时钟指针转动的方向一样——把矩阵里所有数字依次「读出来」， 再按读到的先后顺序拼成一条一维数组（说白了就是一串排成一行的数字， 不再分行分列）返回。

matrix[[1,2,3],[4,5,6],[7,8,9]]

这是一个 3 行 3 列的矩阵。从左上角的 1 出发：先往右走到 3（第一行读完）， 再往下走到 9（最右一列读完），再往左走到 7（最下一行读完）， 再往上走到 4（这时最左列的 7 已经读过了，只需要再走到 4）， 最后只剩中间的 5 还没被读过 → 结果是 [1,2,3,6,9,8,7,4,5]。

🔑 第二步：怎么不写歪、不重复、不漏格

最直接的想法是「模拟走路」：设一个当前方向（右/下/左/上），沿着这个方向一直走， 撞到矩阵边缘、或者撞到已经走过的格子，就右转换下一个方向接着走。 这个思路没错，但真动手写就容易卡壳：怎么判断「已经走过」？是不是要 额外开一个数组专门记录「哪些格子走过了」？循环到底什么时候该停？

更省心的办法是换个角度：不追踪「哪些格子走过了」，而是追踪 还没读过的区域还剩多大。想象矩阵外面围着一个矩形框， 框里就是「还没读过」的部分；每读完一条边，就把框往里收一格，框越缩越小， 直到缩没了，矩阵就读完了。我们用四个数字来描述这个框： top（框的上边在第几行）、bottom（框的下边在第几行）、 left（框的左边在第几列）、right（框的右边在第几列）。

走一圈固定做 4 件事，正好对应「右 → 下 → 左 → 上」：

  • 沿 top 行从 left 走到 right（往右走）→ 这一行读完了，框的上边往下收一格：top++
  • 沿 right 列从 top 走到 bottom（往下走）→ 这一列读完了，框的右边往左收一格：right--
  • 如果 top <= bottom（框里还有行剩下），沿 bottom 行从 right 走到 left（往左走）→ bottom--
  • 如果 left <= right（框里还有列剩下），沿 left 列从 bottom 走到 top（往上走）→ left++
为什么第 3、4 步要多判断一次： 拿只有 1 行的矩阵 [[1,2,3]] 举例——走完第 1 步（往右读完 1、2、3）后， top 从 0 变成 1，可 bottom 还是 0，这说明「框」已经缩没了 （top > bottom，框里已经没有行了），压根没有下面的行可走。 如果不加判断，第 3 步会把这一行倒着再读一遍，1、2、3 就重复出现了。 所以每走完「右、下」两步，都要先确认「框里是不是还剩东西」，才能继续走 「左、上」两步；等四个边界收缩到 top>bottom 或 left>right，就说明所有格子都读完了，循环结束。

☕ 第四步：写成 Java 代码

下面这份代码，就是把第二步的思路照着写下来：四个边界变量 + 一个 while 循环，循环体里固定做四件事，正好对应四条边。 

class Solution {
    public List<Integer> spiralOrder(int[][] matrix) {
        List<Integer> res = new ArrayList<>();
        if (matrix == null || matrix.length == 0) return res;

        // 四个边界，框住「还没访问的区域」
        int top = 0, bottom = matrix.length - 1;
        int left = 0, right = matrix[0].length - 1;

        while (top <= bottom && left <= right) {
            // 1) 沿上边界从左走到右
            for (int j = left; j <= right; j++) res.add(matrix[top][j]);
            top++;

            // 2) 沿右边界从上走到下
            for (int i = top; i <= bottom; i++) res.add(matrix[i][right]);
            right--;

            // 3) 沿下边界从右走到左（这一行还在，没被走过才走）
            if (top <= bottom) {
                for (int j = right; j >= left; j--) res.add(matrix[bottom][j]);
                bottom--;
            }

            // 4) 沿左边界从下走到上（这一列还在，没被走过才走）
            if (left <= right) {
                for (int i = bottom; i >= top; i--) res.add(matrix[i][left]);
                left++;
            }
        }
        return res;
    }
}
时间复杂度 O(m·n)

每个格子恰好被访问一次。

空间复杂度 O(1)

不算返回结果本身，只用了 4 个边界变量。

🚀 小结

套路：遇到「按某种顺序走遍矩阵/网格」的题， 先想清楚用什么状态描述「还没走完的区域」（这里是四个边界）， 再把一圈拆成固定的几步直线移动，每走完一条边就收缩对应的边界。 这个「边界收缩」的思路在螺旋矩阵 II（59 题，反过来填数字）里也完全适用。
