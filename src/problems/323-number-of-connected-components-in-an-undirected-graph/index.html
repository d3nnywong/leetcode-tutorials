一张图被边连成了几块「孤岛」？这道题就是在问这件事。看完你会明白： 只要拿一个「并查集」边扫描边「认祖宗」，就能一遍把分量数算出来。

🧩 第一步：「连通分量」到底是什么

给你 n 个节点（编号 0 ~ n-1），和一些 edges ——每条边把两个节点连起来。把这些节点和边画成一张图，顺着边能互相走到的节点 就算一伙的，这一伙叫一个连通分量。题目要你数一数，图被分成了几伙（几座孤岛）。

n5
edges[[0,1],[1,2],[3,4]]

0—1—2 顺着边能互相到达，是一伙；3—4 是另一伙；没有边能把这两伙连起来 → 答案 2。

🐢 第二步：最直观的办法——DFS/BFS 染色

最容易想到的：先把 edges 建成「邻接表」（每个节点连着哪些节点）。 然后从节点 0 开始，挨个找还没访问过的节点，每碰到一个就做一次 DFS/BFS， 把这次能走到的所有节点都「染上同一种颜色」标记为已访问。染色的次数就是连通分量数。

能做，但要先建图：这个办法是对的，时间复杂度也是 O(n + e)， 但要先花空间建出邻接表，再维护一个「访问标记」数组和递归/队列。 有没有更「轻」的办法，不用真的建图，边读边知道分量数？

⚡ 第三步：并查集——边扫描边「认祖宗」

换个角度：一开始让每个节点自己当自己的根（parent[i] = i）， 这时候有 n 个分量。然后依次看每条边 (a, b)： 分别顺着 parent 往上找到 a、b 各自所在分量的「根」——这叫 find。

如果两个根不一样，说明这条边把两个原本不相通的分量焊接到了一起： 把其中一个根接到另一个根下面，分量数 −1。 如果两个根已经一样，说明 a、b 早就在同一伙了， 这条边是多余的，直接跳过。边扫完，剩下的分量数就是答案——全程不用建邻接表。

☕ 第四步：写成 Java 代码

class Solution {
    public int countComponents(int n, int[][] edges) {
        int[] parent = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i; // 初始时每个节点是自己的根

        int count = n; // 一开始有 n 个连通分量
        for (int[] e : edges) {
            int ra = find(parent, e[0]);
            int rb = find(parent, e[1]);
            if (ra != rb) {
                parent[ra] = rb; // 把两个分量焊接成一个
                count--;         // 分量数减一
            }
        }
        return count;
    }

    private int find(int[] parent, int x) {
        while (parent[x] != x) {
            parent[x] = parent[parent[x]]; // 路径压缩（隔代压缩），把树压扁
            x = parent[x];
        }
        return x;
    }
}
时间复杂度 O(n + e·α(n))

初始化 n 个节点 O(n)，扫描 e 条边，每条边的 find/union 在路径压缩下接近 O(1)（α 是反阿克曼函数，可视为常数）。

空间复杂度 O(n)

只需要一个长度为 n 的 parent 数组，不用建邻接表。

🚀 小结

套路：遇到「图里有多少个互相连通的小团体」这类问题， 并查集是比 DFS/BFS 染色更轻量的工具——不用建图，边读边「认祖宗」、顺手合并， 一遍扫描就知道答案。这个套路在判环、最小生成树（Kruskal）、账户合并等题里都会反复出现。
