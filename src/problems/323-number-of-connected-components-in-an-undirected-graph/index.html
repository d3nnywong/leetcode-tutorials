一张图被边连成了几块「孤岛」？这道题就是在问这件事。看完你会明白： 只要拿一个「并查集」边扫描边「认祖宗」，就能一遍把分量数算出来。

🧩 第一步：「连通分量」到底是什么

先说清楚什么是「图」：把一堆「点」用一些「线」连起来，就是一张图—— 比如地图上的城市是点，连接城市的公路是线，这样点加线的结构，在编程里就叫图。 这道题里，点就是编号 0 ~ n-1 的 n 个节点，线就是 edges 里给出的每一条边（每条边连接两个节点）。

拿到这张图，顺着边能互相走到的节点就算一伙的，这一伙叫一个 连通分量。你可以把它想象成被海隔开的一座座孤岛： 同一座岛上的地方走陆路就能互相到达，不同岛之间没有路，走不过去。 题目要你数一数，这张图一共被分成了几座「孤岛」。

n5
edges[[0,1],[1,2],[3,4]]

0—1—2 顺着边能互相到达，是一伙；3—4 是另一伙；没有边能把这两伙连起来 → 答案 2。

🐢 第二步：最直观的办法——DFS/BFS 染色

最容易想到的：先把 edges 转成「邻接表」——说白了就是给每个节点建一张 小本子，记下它直接连着哪些节点，这样以后想知道某节点的邻居，翻它自己的小本子就行， 不用把所有边再扫一遍。

然后从节点 0 开始，挨个找还没访问过的节点，每碰到一个新的就从它出发「扩散」一次： 可以用DFS（深度优先搜索）——沿着一条路一直往前走，走到头再退回来试别的方向， 像走迷宫时一条道走到黑；也可以用BFS（广度优先搜索）——先看当前节点的所有邻居， 再看邻居的邻居，像水面上的波纹一圈圈往外扩散。不管用哪种，这一次扩散能走到的所有节点， 都属于同一伙，把它们「染上同一种颜色」标记为已访问。一共染色的次数就是连通分量数。

能做，但要先建图：这个办法是对的，时间复杂度也是 O(n + e)， 但要先花空间建出邻接表，再维护一个「访问标记」数组；实现 DFS 通常要用到 递归（一个函数自己调用自己，去解决更小的同类问题，像照镜子时镜子里还有一面镜子）， 实现 BFS 则要用到队列（像排队买票，先来的先被处理）。 有没有更「轻」的办法，不用真的建图，也不用挨个访问，边读边就知道分量数？

⚡ 第三步：并查集——边扫描边「认祖宗」

先说说并查集（Union-Find）是什么：它是一个专门用来快速回答 「这两个东西是不是一伙的」、并且能把两伙合并成一伙的小工具。它的核心想法是： 给每一伙人选一个「代表」，这道题里就叫它「根」。想知道两个节点在不在同一伙， 只要看它们各自找到的「根」是不是同一个人就行，不用真的把这一伙的所有成员都走一遍。

回到这道题：一开始让每个节点自己当自己的根（parent[i] = i）， 这时候谁也不认识谁，一共有 n 个分量。然后依次看每条边 (a, b)： 分别顺着 parent 往上找到 a、b 各自所在分量的「根」——这一步叫 find，你可以理解成「一路问上去，问到底是谁当家」。

如果两个根不一样，说明这条边把两个原本不相通的分量焊接到了一起： 把其中一个根接到另一个根下面（这一步叫 union，合并），分量数 −1。 如果两个根已经一样，说明 a、b 早就在同一伙了， 这条边是多余的，直接跳过。边扫完，剩下的分量数就是答案——全程不用建邻接表， 也不用像 DFS/BFS 那样把每个节点都真的走一遍。

☕ 第四步：写成 Java 代码

class Solution {
    public int countComponents(int n, int[][] edges) {
        int[] parent = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i; // 初始时每个节点是自己的根

        int count = n; // 一开始有 n 个连通分量
        for (int[] e : edges) {
            int ra = find(parent, e[0]);
            int rb = find(parent, e[1]);
            if (ra != rb) {
                parent[ra] = rb; // 把两个分量焊接成一个
                count--;         // 分量数减一
            }
        }
        return count;
    }

    private int find(int[] parent, int x) {
        while (parent[x] != x) {
            parent[x] = parent[parent[x]]; // 路径压缩（隔代压缩），把树压扁
            x = parent[x];
        }
        return x;
    }
}
时间复杂度 O(n + e·α(n))

初始化 n 个节点 O(n)，扫描 e 条边，每条边的 find/union 在路径压缩下接近 O(1)（α 是反阿克曼函数，可视为常数）。

空间复杂度 O(n)

只需要一个长度为 n 的 parent 数组，不用建邻接表。

🚀 小结

套路：遇到「图里有多少个互相连通的小团体」这类问题， 并查集是比 DFS/BFS 染色更轻量的工具——不用建图，边读边「认祖宗」、顺手合并， 一遍扫描就知道答案。这个套路在判环、最小生成树（Kruskal）、账户合并等题里都会反复出现。
