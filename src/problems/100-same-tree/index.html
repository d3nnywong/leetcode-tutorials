判断两棵树「长得一不一样」，听起来要对比很多东西， 其实只要把这个大问题不断拆成一个个「这一个位置和那一个位置是否相同」的小问题就够了。 这也是学「递归」这个技巧的绝佳入门题——读完你会发现，所谓递归， 其实就是让同一套思路在越来越小的问题上反复使用。

🧩 第一步：题目到底要什么

先说说「二叉树」是什么：把它想象成一棵倒过来长的家谱图——最上面有一个起点， 叫根节点；每个节点下面最多能分出两个「孩子」，习惯上叫左孩子、右孩子； 每个孩子下面又可以再分出两个孩子，一层层往下长，直到某个节点没有孩子为止。

题目给你两棵这样的树，分别用它们的根节点 p 和 q 来代表整棵树， 问它们是不是「一模一样」—— 不仅每个位置上的数字要相同，连形状（谁在左边、谁在右边、谁有孩子谁没有）也要完全一致。

p[1,2]（1 只有左孩子 2）
q[1,null,2]（1 只有右孩子 2）

两棵树的节点值都是 1 和 2，但 2 一个在左边、一个在右边——形状不同，所以不相同。

💡 第二步：把大问题拆成「一对节点」的小问题

直接对比「两棵树」很难下手，但对比两个节点很简单——只有 4 种情况：

  • 都是空节点 → 这一支什么都没有，当然相同。
  • 只有一个是空节点 → 一边有东西一边没有，形状已经不同，直接不同。
  • 两个都存在，但值不一样 → 直接不同。
  • 两个都存在，值也一样 → 这一个节点本身没问题，但还不能下结论， 因为下面可能藏着不一样的地方。这时要接着比较它们各自的左子树和右子树—— 所谓左子树，就是把某个节点的左孩子当成一个新的「根」，它和它下面挂着的所有节点自成一棵、规模更小的树； 右子树同理。于是「一整棵树是否相同」这个大问题，就变成了两个一模一样、只是规模更小的小问题。
什么是递归：像上面这样，「判断是否相同」这个方法在检查过程中， 又调用了一次它自己（只不过这次传进去的换成了左孩子或右孩子，问题比上一次更小）， 这种「用同一个方法解决更小的同类问题」的技巧就叫递归—— 有点像照镜子时镜子里还套着一面镜子，一层套一层，直到套到最小的那一层（这里就是遇到空节点）才会停下来往回走。

整棵树是否相同 就可以写成：isSameTree(p, q) 这一支是否相同 = 「这一对节点相不相同」且「左子树相不相同」且「右子树相不相同」。 一旦某一处出现不同，后面就没必要再比了——就像谈合同一旦有一条不同意，后面条款也不用再逐条看了。 Java 里的 && 天生有这种「一假就停」的特性，专业上叫「短路」： 左边一旦算出 false，右边就直接跳过、不会再执行。这正是动画里 一出现红色就停止扩展的原因。

⚡ 第三步：同步往下走，逐对比较

想象两个人分别站在 p 和 q 的根节点上，手拉手同步往下走： 先比较当前这一对节点；如果一样，就一起走向左孩子接着比较，等左边这一支全部确认完， 再一起走向右孩子比较。这种「一条路先走到底、再回头走另一条路」的走法， 就是常说的深度优先搜索（DFS）。

那「递归调用栈」是什么呢？每往下调用一次 isSameTree（比如去比较左孩子）， 程序都会先把手头这一层还没做完的事「压」到一摞暂存记录上——这一摞记录就叫调用栈， 有点像一摞盘子：只能往最上面放，也只能从最上面拿，最后放上去的最先被拿走。 等左子树、右子树都确认相同了，就把这一层的记录从最上面「弹」出来，退回上一层接着往下比较。 下面用一个调用栈面板，把这一层层「压进去（进栈）」「弹出来（出栈）」的过程画出来。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isSameTree(TreeNode p, TreeNode q) {
        // 两边都是空节点：这一支相同
        if (p == null && q == null) {
            return true;
        }
        // 只有一边是空节点：结构已经不同
        if (p == null || q == null) {
            return false;
        }
        // 值相同，且左子树相同，且右子树相同，才算这一支相同
        // && 会短路：左子树一旦不同，右子树就不会再比较
        return p.val == q.val
            && isSameTree(p.left, q.left)
            && isSameTree(p.right, q.right);
    }
}
时间复杂度 O(min(m, n))

m、n 分别是两棵树的节点数；最坏情况要把较小的那棵树逐个节点走一遍才能确认相同或第一次发现不同。

空间复杂度 O(min(h₁, h₂))

递归调用栈（就是上面动画里那摞暂存记录）最多能叠多高，取决于树有多高：h₁、h₂ 分别是两棵树的高度。

🚀 小结

套路：遇到「比较两棵树」「比较两个结构」这类问题，先想清楚 「一对最小单位（这里是一对节点）」有哪几种可能的关系，再用递归把 「整体是否相同」表达成「这一对是否相同 且 子结构是否相同」。 这是后面对称二叉树、合并二叉树、子树等一大类树形递归题的共同起点。
