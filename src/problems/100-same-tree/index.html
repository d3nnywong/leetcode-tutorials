判断两棵树「长得一不一样」，听起来要对比很多东西， 其实只要把问题不断拆成「这一个节点和那一个节点是否相同」就够了——递归的入门好题。

🧩 第一步：题目到底要什么

给你两棵二叉树的根节点 p 和 q，问它们是不是「一模一样」—— 不仅每个位置的值要相同，连形状（结构）也要完全一致。

p[1,2]（1 只有左孩子 2）
q[1,null,2]（1 只有右孩子 2）

两棵树的节点值都是 1 和 2，但 2 一个在左边、一个在右边——形状不同，所以不相同。

💡 第二步：把大问题拆成「一对节点」的小问题

直接对比「两棵树」很难下手，但对比两个节点很简单——只有 4 种情况：

  • 都是空节点 → 这一支什么都没有，当然相同。
  • 只有一个是空节点 → 一边有东西一边没有，形状已经不同，直接不同。
  • 两个都存在，但值不一样 → 直接不同。
  • 两个都存在，值也一样 → 这一个节点本身没问题，但还不能下结论， 得继续递归检查它们的左子树和右子树是否也分别相同。
递归的整棵树思路： isSameTree(p, q) 这一支是否相同 = 「这一对节点相不相同」且「左子树相不相同」且「右子树相不相同」。 一旦某一处出现不同，&& 会立刻短路，不用再往下比对——这正是动画里 一出现红色就停止扩展的原因。

⚡ 第三步：同步往下走，逐对比较

想象两个人分别站在 p 和 q 的根节点上，手拉手同步往下走： 先比较当前这一对节点，再一起走向左孩子比较，回来后再一起走向右孩子比较。 下面用「递归调用栈」面板把这个同步 DFS 的进栈、出栈过程画出来。

当前正在比较的一对节点 已确认这一支完全相同 第一次出现不同的位置

默认两棵树完全一样（结果 true），可以试试把 q 改成 1,2,3,4,5,6,8，看动画在哪一对节点第一次变红、之后又怎么立刻停止往下走。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isSameTree(TreeNode p, TreeNode q) {
        // 两边都是空节点：这一支相同
        if (p == null && q == null) {
            return true;
        }
        // 只有一边是空节点：结构已经不同
        if (p == null || q == null) {
            return false;
        }
        // 值相同，且左子树相同，且右子树相同，才算这一支相同
        // && 会短路：左子树一旦不同，右子树就不会再比较
        return p.val == q.val
            && isSameTree(p.left, q.left)
            && isSameTree(p.right, q.right);
    }
}
时间复杂度 O(min(m, n))

m、n 分别是两棵树的节点数；最坏情况要把较小的那棵树逐个节点走一遍才能确认相同或第一次发现不同。

空间复杂度 O(min(h₁, h₂))

递归调用栈的深度等于树高，h₁、h₂ 分别是两棵树的高度。

🚀 小结

套路：遇到「比较两棵树」「比较两个结构」这类问题，先想清楚 「一对最小单位（这里是一对节点）」有哪几种可能的关系，再用递归把 「整体是否相同」表达成「这一对是否相同 且 子结构是否相同」。 这是后面对称二叉树、合并二叉树、子树等一大类树形递归题的共同起点。
