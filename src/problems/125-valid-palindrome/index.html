判断一个字符串是不是回文串，关键不在「怎么比较」，而在「哪些字符算数」。 想清楚这一点，双指针从两端往中间走，一遍就能扫完。

🧩 第一步：题目到底要什么

给你一个字符串 s，把它里面大写字母变小写、 再去掉所有不是字母和数字的字符（空格、逗号、冒号……统统去掉）， 剩下的字符串如果正着读和倒着读一样，就是回文串，返回 true，否则返回 false。

s"A man, a plan, a canal: Panama"
处理后"amanaplanacanalpanama"

把大写变小写、去掉逗号/空格/冒号之后剩下这串字母，从左往右读和从右往左读完全一样 → 是回文串。

🐢 第二步：最直接的笨办法

最容易想到的办法：先老老实实走一遍字符串，把字母数字都转小写、拼成一个 新字符串，再用「头尾向中间比较」或者直接和它的反转比一比，看是否相等。

能用，但多花了空间：这样需要额外建一个新字符串（甚至再建一个反转串）， 多用了 O(n) 的空间。其实我们完全不需要真的「造出」这个干净的字符串—— 原字符串上直接用两个指针扫一遍就够了。

⚡ 第三步：双指针从两端向中间靠拢

在原字符串上放两个指针：l 从最左边开始， r 从最右边开始。每一步：

如果 s[l] 不是字母或数字，它不算数，左指针右移，跳过它； 如果 s[r] 不是字母或数字，同理右指针左移。 直到两边都停在「真正算数」的字符上，再忽略大小写比较它们：

不同 → 立刻可以断定不是回文串，直接返回 false； 相同 → 这一对字符没问题，两个指针同时向中间靠拢一格，继续下一轮。 一直比到两个指针相遇或交错，说明每一对都比对成功，是回文串。

左指针 L 右指针 R 比对相同 比对不同 相遇的中心字符

淡色虚线框的格子是标点/空格，不参与比较。试试把字符串改成 race a car，看它在哪一步发现不匹配。

☕ 第四步：写成 Java 代码

class Solution {
    public boolean isPalindrome(String s) {
        int l = 0, r = s.length() - 1;
        while (l < r) {
            char cl = s.charAt(l);
            char cr = s.charAt(r);
            // 不是字母/数字就跳过，指针往中间挪一格
            if (!Character.isLetterOrDigit(cl)) {
                l++;
                continue;
            }
            if (!Character.isLetterOrDigit(cr)) {
                r--;
                continue;
            }
            // 忽略大小写比较这一对字符
            if (Character.toLowerCase(cl) != Character.toLowerCase(cr)) {
                return false;
            }
            l++;
            r--;
        }
        return true;
    }
}
时间复杂度 O(n)

两个指针总共最多移动 n 步，每步只做常数次比较。

空间复杂度 O(1)

直接在原字符串上用两个下标扫描，没有额外建新字符串。

🚀 小结

套路：遇到「先清洗字符串再判断回文/对称」的题，先别急着真的造一个新字符串—— 想想能不能在原字符串上用双指针、靠「跳过不算数的字符」来省掉这一步。 这是双指针处理字符串清洗类问题最常用的省空间技巧。
