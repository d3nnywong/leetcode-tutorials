链表不能像数组那样直接「倒着数」。看完你会明白： 快慢指针找中点、原地反转后半段、再用两个指针交替拼接——三步组合拳， O(n) 时间、O(1) 额外空间就能把链表掰成头尾交替的样子。

🧩 第一步：题目到底要什么

给你一个单链表 head，把它「对折」起来：第 1 个结点后面接最后 1 个， 再接第 2 个，再接倒数第 2 个……像这样从两头往中间交替地排列， 而且不能只改值，必须真的把结点接到新的位置上。

head[1, 2, 3, 4, 5]

head 是 1，tail 是 5 → 先接 1 → 5；接下来 2，对应的尾是 4 → 接 2 → 4； 中间剩下 3，自己结尾。最终变成 [1, 5, 2, 4, 3]。

🐢 第二步：最直接的笨办法

最容易想到的：先把所有结点的值依次存进一个数组（或双端队列）， 然后用两个下标一头一尾交替取值，重新连成新链表。这样确实能做对， 但用了 O(n) 额外空间——而本题其实可以做到只用几个指针，不开额外数组。

关键洞察：「头尾交替」本质上是「前半段」和「反过来的后半段」一一交错合并。 只要能把链表从中间切成两段，再把后半段原地反转， 那么从前往后同时走前半段和反转后的后半段、轮流接一个结点， 自然就拼出了 L0 → Ln → L1 → Ln-1 → … 的顺序——全程不需要额外的数组。

⚡ 第三步：找中点 → 反转后半段 → 交替合并

具体拆成三步：

  • ① 找中点：用快慢指针 fast/slow 一遍扫描， fast 一次走 2 步、slow 一次走 1 步；fast 走到头时，slow 正好停在前半段的最后一个结点， 从这里把链表切成两段。
  • ② 反转后半段：用 prev/cur 把后半段原地反转， 反转后它的顺序就变成了「从尾到头」。
  • ③ 交替合并：用 p1 指向前半段头、p2 指向反转后的后半段头， 轮流把 p2 接到 p1 后面、再把 p1 原来的下一个接到 p2 后面，两个指针各自前移，直到 p2 走完。
指针 fast / cur / p2 指针 slow / prev / p1 后半段结点（淡色底） 合并完成后的最终连接

试试把链表改成 1,2,3,4（偶数个结点）或 1（只有 1 个结点）， 看看找中点和合并的边界情况分别长什么样。

☕ 第四步：写成 Java 代码

// Definition for singly-linked list.
class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

class Solution {
    public void reorderList(ListNode head) {
        if (head == null || head.next == null) return;

        // 1. 快慢指针找中点：fast 走到头时，slow 停在前半段最后一个结点
        ListNode slow = head, fast = head;
        while (fast.next != null && fast.next.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }

        // 2. 切开，把后半段原地反转
        ListNode second = slow.next;
        slow.next = null; // 前半段在此结尾
        ListNode prev = null;
        while (second != null) {
            ListNode tmp = second.next;
            second.next = prev;
            prev = second;
            second = tmp;
        }

        // 3. p1 走前半段、p2（反转后的后半段头）交替合并
        ListNode p1 = head, p2 = prev;
        while (p2 != null) {
            ListNode n1 = p1.next;
            ListNode n2 = p2.next;
            p1.next = p2;
            if (n1 == null) break; // 前半段用完，合并结束
            p2.next = n1;
            p1 = n1;
            p2 = n2;
        }
    }
}
时间复杂度 O(n)

找中点、反转、合并各只扫一遍链表，三步相加仍是 O(n)。

空间复杂度 O(1)

只用了 slow/fast、prev/cur、p1/p2 这几个指针，不依赖结点数开额外数组。

🚀 小结

套路：链表上想要「从两头往中间」处理，又不想花 O(n) 额外空间存下标时， 就想到快慢指针找中点 + 原地反转后半段这套组合——它把「双向访问」的需求， 转化成了「反转一半之后照样单向遍历」。这个套路在回文链表、链表重排这类题里非常通用，值得记牢。
