链表不能像数组那样直接「倒着数」。看完你会明白： 快慢指针找中点、原地反转后半段、再用两个指针交替拼接——三步组合拳， O(n) 时间、O(1) 额外空间就能把链表掰成头尾交替的样子。

🧩 第一步：题目到底要什么

先说清楚「链表」是什么，因为这道题的所有麻烦都从它来。数组你可以想象成一排挨着放的储物格， 知道格子编号（下标）就能直接跳过去拿；但链表更像一串手拉手排队的人（我们把每个人叫一个 结点），每个人手里只攥着「下一个人在哪」这一个信息，自己压根不知道队伍有多长、 更不知道倒数第几个是谁。要找到某个人，只能从头一个一个地问过去。

回到题目：给你一个单链表 head，把它「对折」起来——第 1 个结点后面接最后 1 个， 再接第 2 个，再接倒数第 2 个……像这样从两头往中间交替地排列， 而且不能只改值，必须真的把结点（那个「人」）接到新的位置上，改的是它手里攥的 「下一个是谁」。

head[1, 2, 3, 4, 5]

head 是 1，tail 是 5 → 先接 1 → 5；接下来 2，对应的尾是 4 → 接 2 → 4； 中间剩下 3，自己结尾。最终变成 [1, 5, 2, 4, 3]。

🐢 第二步：最直接的笨办法

最容易想到的：既然链表没法「跳着数」，那就先老老实实从头走一遍， 把每个结点的值依次抄进一个数组里——数组是可以随便按下标取值的， 这下就能像下标一样，一头一尾交替地读值，再重新穿成一条新链表。这样确实能做对， 但多开了一个存 n个值的数组，也就是 O(n) 的额外空间—— 而本题其实可以做到只借助几个「记路标」的指针，不开额外数组。

关键洞察：「头尾交替」本质上是「前半段」和「反过来的后半段」一一交错合并。 这里的「反过来」指的是把后半段整个掉个头：原来每个结点都指向「下一个」， 掉头之后全都改成指向「上一个」，于是顺着走下去，读出来的顺序正好是从尾到头。 只要能把链表从中间切成两段，再把后半段这样原地掉头（反转）， 那么从前往后同时走前半段和反转后的后半段、轮流接一个结点， 自然就拼出了 L0 → Ln → L1 → Ln-1 → … 的顺序——全程不需要额外的数组， 只是把「找中点」「掉头」「交替接」这三件小事分别做掉。

⚡ 第三步：找中点 → 反转后半段 → 交替合并

在链表里，我们没法用下标去访问某个位置，只能靠几个「指针」来记路标—— 你可以把指针想成一张贴在某个结点上的便利贴，上面写着「我现在站在这里」， 代码每挪一步，就是把这张便利贴撕下来贴到下一个结点身上。整道题只需要几张这样的便利贴， 分三步走：

  • ① 找中点：同时放两张便利贴 fast（快）和 slow（慢）， 两个一起从头出发，但 fast 每次走 2 步、slow 每次只走 1 步——就像赛跑时一个人速度是另一个人的两倍。 等 fast 先跑到终点，slow 必然只跑了一半路程，正好停在前半段的最后一个结点， 从这里把链表切成两段。
  • ② 反转后半段：用 prev（记「上一个」，一开始是空）和 cur（当前正处理的结点）这两张便利贴，一边往后挪一边把 cur 的指向掉头去指 prev，走完整段后半段，它的顺序就变成了「从尾到头」，prev 也就停在了新的头上。
  • ③ 交替合并：用 p1 指向前半段头、p2 指向反转后的后半段头， 轮流把 p2 接到 p1 后面、再把 p1 原来的下一个接到 p2 后面，两个指针各自前移，直到 p2 走完。 这一步就是把两段「拧」在一起，一次插一个，正好拼出头尾交替的效果。
指针 fast / cur / p2 指针 slow / prev / p1 后半段结点（淡色底） 合并完成后的最终连接

☕ 第四步：写成 Java 代码

// Definition for singly-linked list.
class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

class Solution {
    public void reorderList(ListNode head) {
        if (head == null || head.next == null) return;

        // 1. 快慢指针找中点：fast 走到头时，slow 停在前半段最后一个结点
        ListNode slow = head, fast = head;
        while (fast.next != null && fast.next.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }

        // 2. 切开，把后半段原地反转
        ListNode second = slow.next;
        slow.next = null; // 前半段在此结尾
        ListNode prev = null;
        while (second != null) {
            ListNode tmp = second.next;
            second.next = prev;
            prev = second;
            second = tmp;
        }

        // 3. p1 走前半段、p2（反转后的后半段头）交替合并
        ListNode p1 = head, p2 = prev;
        while (p2 != null) {
            ListNode n1 = p1.next;
            ListNode n2 = p2.next;
            p1.next = p2;
            if (n1 == null) break; // 前半段用完，合并结束
            p2.next = n1;
            p1 = n1;
            p2 = n2;
        }
    }
}
时间复杂度 O(n)

找中点、反转、合并各只扫一遍链表，三步相加仍是 O(n)。

空间复杂度 O(1)

只用了 slow/fast、prev/cur、p1/p2 这几个指针，不依赖结点数开额外数组。

🚀 小结

套路：链表上想要「从两头往中间」处理，又不想花 O(n) 额外空间存下标时， 就想到快慢指针找中点 + 原地反转后半段这套组合——它把「双向访问」的需求， 转化成了「反转一半之后照样单向遍历」。这个套路在回文链表、链表重排这类题里非常通用，值得记牢。
