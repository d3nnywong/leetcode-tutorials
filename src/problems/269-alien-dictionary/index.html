一份排好序的外星词典，藏着字母之间「谁排在谁前面」的线索。 把这些线索连成一张图，再做一次拓扑排序，就能还原出整套字母表。

🧩 第一步：题目到底要什么

有一门外星语言，字母表跟英语一样还是 26 个字母，但顺序变了—— 我们不知道是 a、b、c… 这样，还是别的排法。

题目给了一份已经按这门外星语言排好序的词典（一个字符串数组 words）。我们要做的是：从这份排好序的词典里，反推出 字母表的顺序，拼成一个字符串返回；如果词典里的排列本身就自相矛盾（不存在任何 合法顺序），返回空字符串 ""。

words["wrt","wrf","er","ett","rftt"]

"wrt" 排在 "wrf" 前面 → 说明 t 在 f 前面；"wrf" 排在 "er" 前面 → 说明 w 在 e 前面…… 把所有这种「谁在谁前面」的线索拼起来，就能还原出一种合法顺序："wertf"。

💡 第二步：线索藏在「相邻两个单词」里

直觉上好像得把每两个单词两两比较才能挖出所有线索，那是 O(n²) 次比较。 但仔细想想：词典已经整体排好序了，相邻的两个单词之间的比较， 其实已经包含了「为什么这一个排在那一个前面」最直接的证据—— 它们从第一个不同的字母开始分道扬镳，那个字母谁先出现，谁就该排在字母表更靠前。

关键技巧：只需要扫一遍相邻单词对（一共 n−1 对）， 每对找到第一个不同的字母，就能确定一条「字母 a 必须排在字母 b 前面」的关系。 把这种「谁先谁后」的关系画出来，就是一张图——一堆「点」 （这里的点就是 26 个字母）和连着它们的「线」。这里的线还带着方向， 规定了谁必须排在谁前面，就像单行道只能往一个方向走，所以叫有向边 a → b，意思是「字母 a 必须排在字母 b 前面」。不相邻的单词之间的顺序， 会通过这些边在图上自动传递，不需要再单独比较。

还有一种特殊情况要小心：如果两个单词前面所有字母都一样，更长的那个 却排在了更短的（也就是它自己前缀）后面——比如 "abc" 排在 "ab" 前面——这在任何字母表下都不可能成立，直接判定无解。

把所有线索连成图之后，问题就变成了图论里的经典任务：拓扑排序—— 把图里的所有点排成一条队伍，使得每条边 a → b 里 a 都排在 b 前面。 可以类比穿衣服：袜子必须穿在鞋子前面，这就是一条「袜子 → 鞋子」的边； 把所有类似的「谁先谁后」的边都照顾到，就凑出了一份合法的穿衣顺序—— 拓扑排序做的正是这件事，只不过换成了字母。但如果关系里出现「a 必须在 b 前面， b 又必须在 a 前面」这种死循环（图论里管这种死循环叫环）， 那就谁也没法排在最前面，说明压根没有合法顺序。

⚡ 第三步：建图 + 拓扑排序（Kahn 算法）

整体分两步走：

  1. 建图：依次比较相邻两个单词，找到第一个不同的字母对 (a, b)，连一条边 a → b，并把 b 的入度加一。 入度就是「有多少条边指向它」——可以想成这个字母还欠着几笔「债」： 每多一个字母规定必须排在它前面，它的入度（欠的债）就加一； 债没还清（入度不为 0）之前，它就不能被排进结果。 途中若发现「更长的词排在自己前缀后面」，直接判无解。
  2. 拓扑排序（这一整套具体做法有个名字叫 Kahn 算法，记不住名字完全没关系， 记住思路就行）：先把所有入度为 0 （没有别的字母必须排在它前面，也就是「不欠债」）的字母放进队列—— 队列就像排队买票，谁先进去谁先出来（先进先出）。然后每次从队列取出一个字母 加进结果，再把它指向的每个字母的入度都减一（相当于「还了一笔债」）—— 减到 0 就说明「挡着它的字母都已经排完了」，轮到它入队。如果最后还有字母 没能排进结果，说明这些字母之间存在环（互相欠着债、谁也还不清），也就无解。
当前比较 / 当前出队的字母 新连边的目标字母 已入队（入度为 0） 已排进结果顺序 卡住（存在环）

☕ 第四步：写成 Java 代码

class Solution {
    public String alienOrder(String[] words) {
        Map<Character, Set<Character>> adj = new HashMap<>();
        Map<Character, Integer> inDegree = new HashMap<>();
        for (String w : words) {
            for (char c : w.toCharArray()) {
                adj.putIfAbsent(c, new HashSet<>());
                inDegree.putIfAbsent(c, 0);
            }
        }

        for (int i = 0; i < words.length - 1; i++) {
            String w1 = words[i], w2 = words[i + 1];
            int len = Math.min(w1.length(), w2.length());
            boolean foundDiff = false;
            for (int j = 0; j < len; j++) {
                char a = w1.charAt(j), b = w2.charAt(j);
                if (a != b) {
                    if (!adj.get(a).contains(b)) {
                        adj.get(a).add(b);
                        inDegree.put(b, inDegree.get(b) + 1);
                    }
                    foundDiff = true;
                    break;
                }
            }
            if (!foundDiff && w1.length() > w2.length()) return "";
        }

        Queue<Character> queue = new LinkedList<>();
        for (char c : inDegree.keySet()) {
            if (inDegree.get(c) == 0) queue.offer(c);
        }

        StringBuilder sb = new StringBuilder();
        while (!queue.isEmpty()) {
            char u = queue.poll();
            sb.append(u);
            for (char v : adj.get(u)) {
                int d = inDegree.get(v) - 1;
                inDegree.put(v, d);
                if (d == 0) queue.offer(v); // 入度归零，挡着它的字母都排完了
            }
        }

        return sb.length() == inDegree.size() ? sb.toString() : ""; // 没排全说明有环
    }
}
时间复杂度 O(C)

C 是所有单词的总字母数。建图要扫一遍相邻单词对，拓扑排序每条边、每个节点各访问一次。

空间复杂度 O(1)

字母最多 26 个，图、入度表、队列的大小都被字母表大小（常数）卡死。

🚀 小结

套路：看到「一组元素存在某种偏序关系，要还原出一个满足所有关系的排列」， 先想能不能把关系画成一张有向图，再用拓扑排序求解—— 本文用的是靠入度判定的 Kahn 算法（就是「排队买票」那种一层层剥掉入度为 0 的字母的做法）。 另一种常见思路是 DFS：一条路走到底、走不通再退回来试别的路，这个「一直往深处走」 的过程通常靠函数自己调用自己（也就是递归）来实现， 再靠一个「正在这条路径上」的标记来发现环。不管用哪种方法，判断「无解」的关键 都是看能不能把所有节点都排进结果——排不全，必有环。
