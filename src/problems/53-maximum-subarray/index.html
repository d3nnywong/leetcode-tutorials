一道看似简单、实则很容易掉进 O(n²) 陷阱的题。看完你会明白： 只用两个变量边走边记，就能把「找最大连续和」从暴力枚举压成一遍扫描。

🧩 第一步：题目到底要什么

给你一排数字（可能有正有负），要你从中挑出连续的一段 （子数组，至少 1 个元素），让这一段的和最大，返回这个最大和。

nums[-2, 1, -3, 4, -1, 2, 1, -5, 4]

挑出 [4, -1, 2, 1] 这一段，和是 4 − 1 + 2 + 1 = 6， 比其他任何一段连续子数组的和都大 → 答案是 6。

🐢 第二步：最直接的笨办法

最容易想到的：枚举每一个起点 i，再枚举每一个终点 j（ j ≥ i），把 nums[i..j] 的和都求出来，记录最大值。 光是枚举起点终点就是 O(n²) 对组合，再算每段的和又要 O(n)， 总共 O(n³)（如果提前把「从开头累加到每个位置的和」都算好存起来——这招叫 前缀和，任意一段 nums[i..j] 的和就变成两个存好的数相减， 不用每次重新加——能把总耗时压到 O(n²)，但终究还是要两层循环去枚举 起点和终点，跑不出「平方级」）。

慢在哪：数组长度能到 10⁵，O(n²) 就是上百亿次运算， 根本跑不完。问题的本质是——我们对「这一段值不值得要」这件事，每次都从头重新算， 完全没利用上「往前挪一格」时其实只多了一个数这件事。

⚡ 第三步：边走边记——Kadane 算法

换个思路：既然每次都从头重新算很浪费，那能不能把「算过的东西」存进一个变量里， 往前走一步就顺手更新它，而不是每次都推倒重来？我们用一个变量 cur 来记录「以当前元素结尾的那一段，最大能加到多少」。 走到 nums[i] 时，只需要回答一个问题——要不要把前面那段接上来？

如果 cur 已经是负数，说明前面那段对后面只有拖累，不如丢掉它、从 nums[i] 自己重新开始；如果 cur 还是非负的，就把 nums[i] 接到后面继续累加。每走一步，都用当前的 cur 去刷新目前为止见过的最大值 best。

这背后的思路，其实就是动态规划（英文 Dynamic Programming，简称 DP）： 把一个大问题拆成一串更小的子问题，每个子问题的答案只算一次、顺手记下来， 之后要用就直接拿，不用从头重算。这里的「子问题」就是「以 nums[i] 结尾的 最大子数组和是多少」，写成公式是：

dp[i] = max(nums[i], dp[i-1] + nums[i])

意思是：以 nums[i] 结尾的最大和，要么自己单独另起一段 （前面全部扔掉），要么接在「以 nums[i-1] 结尾的最大和」dp[i-1] 后面—— 两者取更大的那个。因为算 dp[i] 只用得到前一步的 dp[i-1]， 根本不需要留着整张表，于是就把 dp 数组压缩成了一个变量 cur， 一路往前滚动更新——这就是经典的 Kadane 算法。

当前累加段 cur 的范围 历史最大子数组 best 的范围 在此处丢弃前面、重新开始

试试把数组改成 5,4,-1,7,8（全程不重启）或 -1,-2,-3 （每步都重启，答案是最大的那个负数），观察 cur / best 的变化。

☕ 第四步：写成 Java 代码

class Solution {
    public int maxSubArray(int[] nums) {
        int cur = nums[0];  // 以当前元素结尾的最大子数组和
        int best = nums[0]; // 全局看到过的最大值
        for (int i = 1; i < nums.length; i++) {
            // cur 已经是负数就丢掉它，从 nums[i] 重新开始；否则接上去
            cur = Math.max(nums[i], cur + nums[i]);
            best = Math.max(best, cur); // 随时刷新最大值
        }
        return best;
    }
}
时间复杂度 O(n)

数组只扫一遍，每步只做常数次比较和加法。

空间复杂度 O(1)

只用了 cur、best 两个变量，没有额外数组。

🚀 小结

套路：当题目要在数组里找「连续一段」的最优值时，先想想能不能维护一个 「以当前元素结尾」的状态，边走边用它去更新目前见过的最优答案——这样就能把朴素枚举的 O(n²)/O(n³) 压成一遍 O(n) 扫描。 Kadane 算法是这个套路最经典的入门例子，后面遇到「最大/最小连续和」「环形数组最大和」 这类题，都可以照着这个思路去想。
