给你一棵树的「先序」和「中序」两份遍历记录，要你把整棵树原样拼回来。 看完你会明白：这两份记录加在一起，其实藏着「谁是根」「左右子树各有几个节点」的全部信息。

🧩 第一步：两份遍历记录到底告诉了我们什么

先序遍历（root → 左子树 → 右子树）每次第一个访问的就是根， 所以 preorder[0] 一定是整棵树的根节点。

中序遍历（左子树 → root → 右子树）里，根节点出现的位置左边全是左子树的节点、 右边全是右子树的节点。也就是说：只要在中序里找到根节点的位置， 就能把树「劈」成左右两半。

preorder[3, 9, 20, 15, 7]
inorder[9, 3, 15, 20, 7]

根是 preorder[0] = 3；在 inorder 里找到 3，下标是 1， 它左边的 [9] 是左子树的中序，右边的 [15, 20, 7] 是右子树的中序。

🔑 第二步：劈开之后，子树怎么办——递归

左右两半各自又是一棵「更小的树」的中序遍历，而它们对应的先序遍历， 也正好紧跟在 preorder 里根节点后面——先是整段左子树，再是整段右子树。 于是同样的套路可以照搬到子树上：拿子树先序的第一个数当子树的根，再去子树中序里劈一刀。

怎么快速定位根在中序里的位置？如果每次都现场遍历查找，定位一个根要 O(n)， 整体退化到 O(n²)。提前用一个哈希表把「值 → 中序下标」记下来，定位就是 O(1)， 整体降到 O(n)。

⚡ 第三步：递归分治动画

preorder 用一个全局指针 pre 从左到右扫，每次取一个数当「这一段」的根； inorder 维护一个不断收缩的区间 [l, r]，根节点的位置把它劈成左右两半， 按「先左后右」的顺序递归下去，树就一层层被拼出来。

当前从 preorder 取出的根 已取出 / 当前中序区间 [l, r] 根节点在 inorder 中的下标 mid

试试改成单节点 preorder = -1, inorder = -1，或自己拼一棵小树，看看左右子树为空时是怎么变成 null 的。

☕ 第四步：写成 Java 代码

class Solution {
    private int preIdx = 0; // preorder 的全局指针
    private Map<Integer, Integer> inorderIndex; // 值 -> 中序下标
    private int[] preorder;

    public TreeNode buildTree(int[] preorder, int[] inorder) {
        this.preorder = preorder;
        inorderIndex = new HashMap<>();
        for (int i = 0; i < inorder.length; i++) {
            inorderIndex.put(inorder[i], i); // 提前记下「值 -> 中序下标」，定位 O(1)
        }
        return build(0, inorder.length - 1);
    }

    // 构造「中序区间 [l, r]」对应的子树，preIdx 隐式标记了对应的 preorder 起点
    private TreeNode build(int l, int r) {
        if (l > r) return null; // 区间为空，这里是 null

        int rootVal = preorder[preIdx++]; // 先序的下一个数就是这一段的根
        TreeNode root = new TreeNode(rootVal);

        int mid = inorderIndex.get(rootVal); // 根在中序里的位置，把区间劈成两半
        root.left = build(l, mid - 1);  // 先序顺序：根之后紧跟整段左子树
        root.right = build(mid + 1, r); // 再是整段右子树
        return root;
    }
}
时间复杂度 O(n)

每个节点只在 preorder 里被取出一次、在哈希表里被查找一次，都是 O(1)。

空间复杂度 O(n)

哈希表存 n 个值；递归栈深度最坏（链状树）也是 O(n)。

🚀 小结

套路：「先序定根，中序分左右」——先序遍历的第一个数永远是当前子树的根， 中序遍历里根的位置把剩下的元素分成左右两半子树。把这个规律套进递归， 配合一个哈希表加速「根在中序里的位置」查找，就能把整棵树原样重建出来。 类似的「两种遍历互相还原」的题（如中序+后序）也是同一个套路。
