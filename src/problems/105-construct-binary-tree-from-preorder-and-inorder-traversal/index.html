给你一棵树的「先序」和「中序」两份遍历记录，要你把整棵树原样拼回来。 看完你会明白：这两份记录加在一起，其实藏着「谁是根」「左右子树各有几个节点」的全部信息。

🧩 第一步：先搞懂「二叉树」和这两份「遍历记录」是什么

先别管算法，二叉树其实就是一张简化版的家谱图：最上面是一个「祖先」， 在树里叫根节点；每个节点往下最多分出两个「孩子」， 分别叫左孩子和右孩子；一个孩子带着的那一整挂后代， 就叫这个孩子对应的左子树或右子树。

题目给的 preorder（先序遍历）和 inorder（中序遍历）， 是同一棵树按两种不同「走法」走一遍、把经过的节点值依次记下来得到的两串数字 （这种按规则走遍所有节点的过程，就叫遍历）：

  • 先序遍历：每到一个节点，先记它自己，再去左子树，最后去右子树（根 → 左 → 右）。
  • 中序遍历：先去左子树，再记它自己，最后去右子树（左 → 根 → 右）。

题目只给了这两串数字，要我们把原来那棵树的形状——谁是根、谁是谁的左右孩子——原样拼回来。 诀窍就藏在上面这两条走法规则里：

先序遍历「先记自己」，所以它记下的第一个数，永远是当前这一段树的根节点， 也就是 preorder[0] 一定是整棵树的根节点。

中序遍历「先左、再自己、再右」，所以根节点在中序记录里出现的位置， 天然把这段记录劈成了两半：左边全是左子树的节点，右边全是右子树的节点。 也就是说：只要在中序里找到根节点的位置，就能把树「劈」成左右两半。

preorder[3, 9, 20, 15, 7]
inorder[9, 3, 15, 20, 7]

根是 preorder[0] = 3；在 inorder 里找到 3，下标是 1， 它左边的 [9] 是左子树的中序，右边的 [15, 20, 7] 是右子树的中序。

🔑 第二步：劈开之后，子树怎么办——递归

劈开后剩下的左边一段、右边一段，各自其实就是一棵「更小的树」的中序遍历——道理跟原来那棵大树完全一样， 只是节点变少了。而且它们对应的先序遍历，也正好紧跟在 preorder 里根节点后面—— 先是整段左子树，再是整段右子树。

这意味着，我们可以把「取根、在中序里劈一刀」这一整套动作，原封不动地套用到子树上， 子树里再套更小的子树……一直套到某段子树只剩 0 个或 1 个节点为止。 这种一个函数在解决问题的过程中，又调用自己去解决一个更小的同类问题的写法， 就叫递归——有点像俄罗斯套娃：打开最外面一层，里面还是同样结构的一个更小的娃娃， 一层层打开下去，直到打开的娃娃里什么都没有为止。

怎么快速定位根在中序里的位置？如果每次都现场从头扫一遍去找，定位一个根要 O(n) 的时间， n 个节点全部定位下来就退化到 O(n²)。更好的办法是：一开始就用一个哈希表—— 你可以把它想成一本「查号簿」，塞进去一对「值 → 位置」，之后凭「值」就能直接翻到对应的位置， 不用再挨个去找——提前把「值 → 中序下标」都记进哈希表，之后每次定位根都是 O(1)， 整体就从 O(n²) 降到了 O(n)。

⚡ 第三步：递归分治动画

preorder 这边维护一个全局指针 pre——你可以把它想成一枚不断往右挪的书签： 每次需要「取根」时，就从书签所在的位置拿一个数当「这一段」的根，再把书签往右移一位； inorder 那边维护一个不断收缩的区间 [l, r]，根节点在区间里的位置把它劈成左右两半， 按「先左后右」的顺序递归下去（也就是重复第二步那套「取根、劈开、再对更小的一段做同样的事」）， 树就这样一层层被拼出来。

当前从 preorder 取出的根 已取出 / 当前中序区间 [l, r] 根节点在 inorder 中的下标 mid

试试改成单节点 preorder = -1, inorder = -1，或自己拼一棵小树，看看左右子树为空时是怎么变成 null 的。

☕ 第四步：写成 Java 代码

class Solution {
    private int preIdx = 0; // preorder 的全局指针
    private Map<Integer, Integer> inorderIndex; // 值 -> 中序下标
    private int[] preorder;

    public TreeNode buildTree(int[] preorder, int[] inorder) {
        this.preorder = preorder;
        inorderIndex = new HashMap<>();
        for (int i = 0; i < inorder.length; i++) {
            inorderIndex.put(inorder[i], i); // 提前记下「值 -> 中序下标」，定位 O(1)
        }
        return build(0, inorder.length - 1);
    }

    // 构造「中序区间 [l, r]」对应的子树，preIdx 隐式标记了对应的 preorder 起点
    private TreeNode build(int l, int r) {
        if (l > r) return null; // 区间为空，这里是 null

        int rootVal = preorder[preIdx++]; // 先序的下一个数就是这一段的根
        TreeNode root = new TreeNode(rootVal);

        int mid = inorderIndex.get(rootVal); // 根在中序里的位置，把区间劈成两半
        root.left = build(l, mid - 1);  // 先序顺序：根之后紧跟整段左子树
        root.right = build(mid + 1, r); // 再是整段右子树
        return root;
    }
}
时间复杂度 O(n)

每个节点只在 preorder 里被取出一次、在哈希表里被查找一次，都是 O(1)。

空间复杂度 O(n)

哈希表存 n 个值；递归栈深度最坏（链状树）也是 O(n)。

🚀 小结

套路：「先序定根，中序分左右」——先序遍历的第一个数永远是当前子树的根， 中序遍历里根的位置把剩下的元素分成左右两半子树。把这个规律套进递归 （一层套一层地对更小的子树重复同一件事），配合一个哈希表加速「根在中序里的位置」查找， 就能把整棵树原样重建出来。类似的「两种遍历互相还原」的题（如中序+后序）也是同一个套路。
