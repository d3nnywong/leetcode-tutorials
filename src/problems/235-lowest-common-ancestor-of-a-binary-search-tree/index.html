普通二叉树找最近公共祖先要老老实实递归两边找；二叉搜索树多了一条「左小右大」的性质， 能直接当导航用——看完你会发现这题其实比想象中简单很多。

🧩 第一步：题目到底要什么

给你一棵二叉搜索树的根节点 root，以及树里两个不同的节点 p 和 q，要你找出它们的最近公共祖先（LCA）。

「最近公共祖先」说人话就是：从 p 和 q 各自往上爬， 第一个同时是它们俩祖先的节点。特别要注意——一个节点可以是它自己的祖先， 所以如果 q 就在 p 的子树里，那答案直接就是 p 本身。

root[6,2,8,0,4,7,9,null,null,3,5]
p, q2, 8

2 在根节点 6 的左子树，8 在右子树，两边的「家谱」只在根节点 6 这里汇合 → 答案是 6。

🐢 第二步：普通二叉树的做法为什么「浪费」

如果这是一棵普通二叉树，标准解法是递归：在左子树里找 p、q， 在右子树里也找，谁都没找到就说明两个节点都在同一侧，往上交给父节点决定； 左右各找到一个，当前节点就是答案。这个做法稳，但要把整棵树都摸一遍， 时间是 O(n)，因为它压根没用上「这是一棵 BST」这个额外信息。

关键洞察：二叉搜索树有个免费的导航仪——左子树全部更小，右子树全部更大。 站在某个节点上，只要比较一下 p.val、q.val 和当前节点值， 立刻就能判断答案在左边、右边，还是就在这里分岔，根本不需要真的去左右子树里「找」。

⚡ 第三步：从根出发，沿着一条路径走到底

从根节点开始，每一步只看当前节点的值：

  • p、q 都比当前节点小 → 它们俩一定都在左子树里，往左走。
  • p、q 都比当前节点大 → 它们俩一定都在右子树里，往右走。
  • 不再同侧（一个 ≤ 当前节点、一个 ≥ 当前节点，包括当前节点正好就是 p 或 q 本身）→ 从这里开始两人就要「分家」了， 当前节点就是最近公共祖先。

全程只沿着一条从根到答案的路径走，不用回溯、不用额外空间，时间只花在树的高度上。

当前正在比较 已走过的路径 最近公共祖先

默认 p = 3、q = 5：先往右走、再往左走才分岔。也可以试试 p = 2、q = 8，一步到位（根节点就是答案）。

☕ 第四步：写成 Java 代码

class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        TreeNode cur = root;
        while (cur != null) {
            if (p.val < cur.val && q.val < cur.val) {
                cur = cur.left; // 两个都更小，答案在左子树
            } else if (p.val > cur.val && q.val > cur.val) {
                cur = cur.right; // 两个都更大，答案在右子树
            } else {
                // 不再同侧（或 cur 正好是 p / q 本身），这里就是分岔点
                return cur;
            }
        }
        return null; // 题目保证 p、q 都存在，不会到这
    }
}
时间复杂度 O(h)

h 是树高，只沿一条路径往下走；平衡树 O(log n)，最坏（链状树）退化到 O(n)。

空间复杂度 O(1)

迭代写法只用了一个指针，没有用到递归栈。

🚀 小结

套路：遇到「二叉搜索树」相关的题，先别急着套通用二叉树的递归模板—— 想想左小右大这条性质能不能省掉一半的搜索。这题就是最典型的例子： 普通二叉树要 O(n) 摸遍全树，BST 靠比大小直接导航，O(h) 沿一条路径走到底。
