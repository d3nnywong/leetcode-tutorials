位运算入门第一课：数一个整数的二进制里有几个 1。 看完你会明白一个特别巧的小技巧——n & (n - 1)， 为什么它每次都能精确清掉最低位的那个 1。

🧩 第一步：题目到底要什么

给你一个正整数 n，要你数一数它写成二进制之后， 一共有几个 1。

先说说什么是「二进制」：我们平时数数用的是「十进制」，逢十进一，每一位是 0~9 里的一个数字。 而计算机里的每一个数字，骨子里都是用只有 0 和 1 两种符号拼出来的， 逢二进一，这就叫二进制。二进制里的每一位（一个 0 或一个 1）叫做一个 位（bit）。比如十进制的 11，换算成二进制就是 1011——从右往左，每一位分别代表 1、2、4、8…… 这样按 2 的次方往上翻倍的「位权」，1×8 + 0×4 + 1×2 + 1×1 = 11，正好对上。

这道题要的就是：数一数 n 的二进制表示里，一共出现了几个 1。 这个「1 的个数」在计算机里有个专门的名字，叫汉明重量—— 名字听着唬人，其实就是「1 的个数」，不用死记。

n11
二进制1011

1011 里有 3 个 1（最高位、第 1 位、第 0 位）→ 返回 3。

🐢 第二步：最直接的笨办法

最容易想到的：既然 n 是由 32 个「0 或 1」拼成的 （Java 里 int 固定占 32 位），那就把这 32 位挨个检查一遍， 数数里面有几个 1。

怎么才能「只看一眼最低位是不是 1」？这就要用到位运算—— 顾名思义，不把数字当成一个整体来算，而是拆开来一位一位地按 0/1 处理。 这里用到的第一个位运算叫按位与 &： 它把两个数按位对齐、逐位比较，两边都是 1 才得 1，只要有一边是 0 就得 0。 于是 n & 1 就是拿 n 去跟只有最低位是 1 的 ...0001 按位与——除了最低位，别的位统统被压成 0，最低位则原样保留， 结果正好就是「n 的最低位到底是 0 还是 1」。

看完最低位，要挪去看下一位，这就要用第二个位运算——右移 >>>： 把整个数所有的位整体往右挪一位，就像把一排小灯泡整体往右推一格， 左边补一个 0，右边挤出去的那一位直接丢掉。挪一次之后，原来「第二低」的那一位 就变成了新的最低位，下一轮 n & 1 就能看到它。

这样「看一眼最低位、整体右移一位」重复 32 次（32 位 int 的每一位都看一遍）， 就能把 1 的个数数完。

慢在哪：不管 n 里实际有几个 1，笨办法永远要把 32 位 全部检查一遍——哪怕 n = 128 这种只有 1 个 1 的数， 也要白白多看 31 次「这一位是 0」。我们能不能只为真正存在的 1 买单？

⚡ 第三步：n & (n - 1) 一步清掉一个 1

关键观察：把 n 减去 1，会发生什么？我们拿一个具体例子看看： n = 12，二进制是 1100。减 1 是什么感觉？ 就像十进制里 100 - 1 要一路往前借位、变成 099 一样， 二进制减 1 也要借位：1100 最低两位都是 0，借不出 1， 于是一路往左找，找到第一个 1（也就是最低位的那个 1），把它变成 0， 它右边原本那一串 0 则全部借位翻转成 1。 算下来 n - 1 = 1011（十进制的 11）。

规律就是：n - 1 会把 n 最低位的那个 1 变成 0， 同时把它右边那一串 0 全部翻转成 1，左边更高的位完全不受影响。

n1100（十进制 12）
n - 11011（十进制 11）
n & (n - 1)1000（十进制 8）

这时候再做 n & (n - 1)，也就是把 n 和 n - 1 按位对齐、逐位相与，会发生三件事：n 原本最低位的那个 1， 在 n - 1 里已经变成了 0，两边一比，只要有一边是 0 结果就是 0， 于是这个 1 就被精确地清成了 0；它右边那一串位， n 本来就是 0、n - 1 借位后变成了 1，两边一比还是 0， 原样不变；再往左那些完全没被减法波及的位，n 和 n - 1 长得一模一样，跟自己按位与当然也是原样保留。三段拼起来，效果就是—— 只清掉了 n 最低位的那一个 1，其余所有位分毫不动。 也就是说，每做一次 n & (n - 1)，就能干净利落地消掉一个 1。

于是只要不断重复「n = n & (n - 1)，计数器加一」，直到 n 变成 0，循环次数就正好是 1 的个数——而且循环次数只跟 1 的个数有关，跟 32 位里有多少个 0 完全无关。

当前还是 1（还没数到） 这一步刚被清掉的 1 已经数过的小计

试试把 n 改成 128，看它一步就清完唯一的那个 1； 再试试 255，看 8 个 1 是怎么一个个被消掉的。

☕ 第四步：写成 Java 代码

class Solution {
    public int hammingWeight(int n) {
        int count = 0;
        while (n != 0) {
            n &= (n - 1); // 精确清掉最低位的那个 1
            count++;
        }
        return count;
    }
}

代码里的 n &= (n - 1) 是 n = n & (n - 1) 的简写 （跟 += 是 = 和 + 的简写是一个道理）： 算出 n & (n - 1) 的结果，再存回 n 自己。

对照一下笨办法：笨办法是 while (n != 0) { count += n & 1; n >>>= 1; }， 每一步都在「挪一位、看一眼」；这里的 n &= (n - 1) 则是「一步直接消掉一个 1」， 循环次数从固定的 32 次，降到了「恰好等于 1 的个数」。

时间复杂度 O(k)

k 是 n 二进制中 1 的个数，最坏情况 k ≤ 32（远好于固定 32 次的笨办法）。

空间复杂度 O(1)

只用了一个计数器，没有额外的数据结构。

🚀 小结

套路：n & (n - 1) 是位运算里最常用的小技巧之一—— 它能把整数二进制中最低位的 1 精确清零。除了数 1 的个数， 它还能用来快速判断一个数是不是 2 的幂（n & (n - 1) == 0）， 记住这个技巧，很多位运算题都会变得轻松不少。
