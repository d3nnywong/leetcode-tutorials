位运算入门第一课：数一个整数的二进制里有几个 1。 看完你会明白一个特别巧的小技巧——n & (n - 1)， 为什么它每次都能精确清掉最低位的那个 1。

🧩 第一步：题目到底要什么

给你一个正整数 n，要你数一数它写成二进制之后， 一共有几个 1。这个「1 的个数」有个专门的名字，叫汉明重量。

n11
二进制1011

1011 里有 3 个 1（最高位、第 1 位、第 0 位）→ 返回 3。

🐢 第二步：最直接的笨办法

最容易想到的：把 n 的 32 个二进制位挨个检查一遍—— 每次看最低位是不是 1（n & 1），是就计数加一， 然后把整个数右移一位（n >>> 1），继续看下一位，一共要做 32 次。

慢在哪：不管 n 里实际有几个 1，笨办法永远要把 32 位 全部检查一遍——哪怕 n = 128 这种只有 1 个 1 的数， 也要白白多看 31 次「这一位是 0」。我们能不能只为真正存在的 1 买单？

⚡ 第三步：n & (n - 1) 一步清掉一个 1

关键观察：把 n 减去 1，会发生什么？ n 最低位的那个 1 会变成 0， 而它右边所有的 0 会全部借位变成 1 （其余更高位完全不变）。

这时候再做 n & (n - 1)：最低位的 1 和它右边的那串 0， 跟 n - 1 里翻转后的位逐一相与，结果正好把那个最低位的 1 精确地清成 0，其他位原样保留。也就是说，每做一次 n & (n - 1)，就能干净利落地消掉一个 1。

于是只要不断重复「n = n & (n - 1)，计数器加一」，直到 n 变成 0，循环次数就正好是 1 的个数——而且循环次数只跟 1 的个数有关，跟 32 位里有多少个 0 完全无关。

当前还是 1（还没数到） 这一步刚被清掉的 1 已经数过的小计

试试把 n 改成 128，看它一步就清完唯一的那个 1； 再试试 255，看 8 个 1 是怎么一个个被消掉的。

☕ 第四步：写成 Java 代码

class Solution {
    public int hammingWeight(int n) {
        int count = 0;
        while (n != 0) {
            n &= (n - 1); // 精确清掉最低位的那个 1
            count++;
        }
        return count;
    }
}

对照一下笨办法：笨办法是 while (n != 0) { count += n & 1; n >>>= 1; }， 每一步都在「挪一位、看一眼」；这里的 n &= (n - 1) 则是「一步直接消掉一个 1」， 循环次数从固定的 32 次，降到了「恰好等于 1 的个数」。

时间复杂度 O(k)

k 是 n 二进制中 1 的个数，最坏情况 k ≤ 32（远好于固定 32 次的笨办法）。

空间复杂度 O(1)

只用了一个计数器，没有额外的数据结构。

🚀 小结

套路：n & (n - 1) 是位运算里最常用的小技巧之一—— 它能把整数二进制中最低位的 1 精确清零。除了数 1 的个数， 它还能用来快速判断一个数是不是 2 的幂（n & (n - 1) == 0）， 记住这个技巧，很多位运算题都会变得轻松不少。
