一棵树要「一层一层」地报数，靠的不是递归的深度优先，而是用一个队列把同层节点 打包处理——这就是广度优先搜索（BFS）在树上最经典的应用。

🧩 第一步：题目到底要什么

先认识一下「二叉树」：它长得有点像一张家谱图——最上面有一个起点， 叫根节点（root）；每个「节点」（说白了就是树上的一个格子， 里面装着一个数字）下面最多能牵出两条线，接到它的「孩子」（左孩子、右孩子）； 孩子下面又能再长出自己的孩子，一层一层往下铺开，直到某个节点没有孩子为止 （这种没有孩子的节点叫「叶子」，是树的末梢）。

这道题要你做的是：把这棵树上所有节点的值，按「一层一层、 每层从左到右」的顺序整理出来，每一层单独放进一个小数组，最后所有小数组拼成一个大数组。

root[3, 9, 20, null, null, 15, 7]

这个数组是 LeetCode 按层记录树形状的写法，null 表示「这个位置没有孩子节点」。 翻译成树就是：根节点 3，它的左右孩子是 9 和 20（9 没有孩子，所以后面跟了两个 null）， 20 的左右孩子是 15 和 7。 按层分组就是：第 1 层 [3]，第 2 层 [9, 20]， 第 3 层 [15, 7] → 答案 [[3],[9,20],[15,7]]。

🐢 第二步：为什么不能直接用递归 DFS

一提到「树的遍历」，很多人第一反应是递归——就是写一个函数， 让它在处理「这个节点」时，直接调用自己去处理「它的孩子节点」，一层套一层， 有点像俄罗斯套娃，或者两面镜子对照出的无穷层影像。这样写通常是深度优先的： 从根节点一路扎到最深处的叶子，碰壁了再退回上一层换个方向接着走 （这个「退回去换方向」的动作就叫回溯）。

问题是：递归天生是「一头扎到底」，它根本不知道「现在走到第几层了」。 想用它分层，就得额外传一个 depth 参数专门记录当前深度， 麻烦还容易绕晕。

问题的本质：层序遍历要的是「按层分批」，这正好是广度优先搜索（BFS） 的拿手好戏——BFS 不是「一头扎到底」，而是像水波纹一样一圈一圈往外扩散： 先处理离根节点最近的一圈，再处理下一圈，每一圈刚好对应树的每一层。 实现 BFS 最简单的方式，就是用一个队列——它就跟排队买票一样， 先进去排队的人先被叫到（先进先出），谁先进队列，谁就先被拿出来处理。

⚡ 第三步：用队列「一层一层」地扫

先把根节点放进队列（此刻队列里只有它一个，也就是第 1 层）。然后反复做这件事： 记下当前队列里一共有多少个节点 levelSize ——这正好就是「这一层」的节点数。 接着出队 levelSize 次：每次都从队伍最前面「请」出一个节点（出队）， 把它的值记下来，再把它的左右孩子（如果存在）排到队伍最后面（入队）， 留给下一轮处理。

关键就在于「先量好这一层有几个节点，再出队这么多次」——这样不管队列后面又混进了 下一层的孩子，这一轮也只会处理到属于当前层的节点，绝不会串层。 一轮处理完，队列里剩下的全是下一层的节点，继续循环，直到队列空。

在队列中等待 正在出队 / 处理 刚被加入队尾的子节点 已处理完

试试把输入改成 1,2,3,4,5,null,6,7，看看节点更多、层数更深时队列是怎么滚动的； 也可以把输入框清空，看看空树是怎么直接返回 [] 的。

☕ 第四步：写成 Java 代码

/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public List<List<Integer>> levelOrder(TreeNode root) {
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result; // 空树，直接返回空列表

        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);

        while (!queue.isEmpty()) {
            int levelSize = queue.size(); // 先量好这一层有几个节点
            List<Integer> level = new ArrayList<>();
            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();
                level.add(node.val);
                if (node.left != null) queue.offer(node.left);
                if (node.right != null) queue.offer(node.right);
            }
            result.add(level); // 这一层处理完，整批加入结果
        }
        return result;
    }
}
时间复杂度 O(n)

每个节点恰好入队、出队一次，n 是节点总数。

空间复杂度 O(n)

最坏情况（比如一棵完全二叉树的最后一层）队列要同时装下接近一半的节点； 加上存结果的空间，总体是 O(n)。

🚀 小结

套路：只要看到「按层」「最短路径」「逐圈扩散」这类字眼，第一反应就该是 BFS + 队列，而不是递归 DFS。核心动作只有一句话： 先量好当前队列的大小再出队那么多次，这一招能保证每一轮恰好处理完一层， 后面遇到「二叉树的锯齿形层序遍历」「图的最短路径」这类题，都是同一个套路的变形。
