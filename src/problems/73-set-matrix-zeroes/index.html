一道经典的「原地修改」题：表面上一遍遍历就能做完，但稍不注意就会用提前清零的 0 污染后面的判断。看完你会明白怎么只用两个布尔变量，把矩阵自己当成记事本。

🧩 第一步：题目到底要什么

给你一个 m 行 n 列的矩阵，只要某个格子的值是 0，就要把它所在的整行和整列都变成 0。 而且要求原地修改——不能新开一个一模一样大小的矩阵来存结果。

matrix[[1,1,1],[1,0,1],[1,1,1]]

中间的 matrix[1][1] = 0，所以第 1 行和第 1 列全部变成 0 → [[1,0,1],[0,0,0],[1,0,1]]。

🔑 第二步：为什么不能扫到 0 就立刻清零

第一反应可能是：扫一遍矩阵，碰到 0 就把它的整行整列都设成 0，不就完了？ 但这样做有个致命问题——你把别的格子也变成 0 了，后面再扫到这些「假 0」， 会被误判成「又发现一个 0」，导致清零范围越滚越大，结果完全错误。

所以标准做法是分两步走：先把「哪些行 / 哪些列该清零」都记下来， 全部记完之后，再统一执行清零。问题变成：用什么记？

朴素做法：开两个长度分别为 m 和 n 的布尔数组 rows[] / cols[]，记下哪些行、哪些列含 0。 这样空间复杂度是 O(m + n)，能过，但不是最优解。

更省空间的想法：矩阵的第 0 行正好有 n 个格子， 第 0 列正好有 m 个格子——数量和 rows[] / cols[] 完全对得上！与其新开数组，不如直接借用矩阵自己的第 0 行当 cols[]， 第 0 列当 rows[]：用 matrix[0][j] 记「第 j 列要不要清零」， 用 matrix[i][0] 记「第 i 行要不要清零」。

唯一要小心的坑：第 0 行 / 第 0 列自己原本是不是也含 0， 这件事会在「打标记」的过程中被覆盖掉，分不清「这是原来就有的 0」还是「后来打的标记」。 解决办法很简单：先用两个布尔变量 row0Zero / col0Zero 单独记下第 0 行 / 第 0 列原本是否含 0，再放心地拿它们当标记位用， 最后再根据这两个变量决定要不要把第 0 行 / 第 0 列也整个清零。

☕ 第四步：写成 Java 代码

class Solution {
    public void setZeroes(int[][] matrix) {
        int m = matrix.length, n = matrix[0].length;
        boolean row0Zero = false, col0Zero = false;

        // 1) 先单独记录第 0 行 / 第 0 列原本是否含 0（马上要被当标记位覆盖）
        for (int j = 0; j < n; j++) {
            if (matrix[0][j] == 0) { row0Zero = true; break; }
        }
        for (int i = 0; i < m; i++) {
            if (matrix[i][0] == 0) { col0Zero = true; break; }
        }

        // 2) 扫内部格子，用第 0 行 / 第 0 列当标记位：matrix[0][j] 记第 j 列，matrix[i][0] 记第 i 行
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][j] == 0) {
                    matrix[i][0] = 0;
                    matrix[0][j] = 0;
                }
            }
        }

        // 3) 再扫一遍内部格子，标记位是 0 就把这个格子清零
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][0] == 0 || matrix[0][j] == 0) {
                    matrix[i][j] = 0;
                }
            }
        }

        // 4) 最后处理第 0 行 / 第 0 列本身
        if (row0Zero) {
            for (int j = 0; j < n; j++) matrix[0][j] = 0;
        }
        if (col0Zero) {
            for (int i = 0; i < m; i++) matrix[i][0] = 0;
        }
    }
}
时间复杂度 O(m·n)

矩阵被完整扫描了常数遍（检查 + 打标记 + 应用 + 收尾），每个格子访问次数是常数。

空间复杂度 O(1)

不算输入矩阵本身，只用了 row0Zero / col0Zero 两个布尔变量。

🚀 小结

套路：需要「先标记、后统一处理」却又想省掉额外数组时， 想想输入数据本身有没有空间可以借用——这里矩阵的第 0 行/列大小正好和 需要的标记数组对得上。借用之前，记得先把会被覆盖的原始信息单独存好（ row0Zero / col0Zero），这是这类「原地算法」题最容易翻车的地方。
