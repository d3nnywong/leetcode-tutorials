一道经典的「原地修改」题：表面上一遍遍历就能做完，但稍不注意就会用提前清零的 0 污染后面的判断。看完你会明白怎么只用两个布尔变量，把矩阵自己当成记事本。

🧩 第一步：题目到底要什么

题目给你一个 matrix：一个 m 行 n 列的矩阵 ——说白了就是一张像 Excel 表格一样的方格表，有若干行、若干列，每个格子里放一个数字。 行号从上往下数、列号从左往右数，都从 0 开始编号：第 0 行就是最上面那行， 第 0 列就是最左边那列。后面不管是讲解还是代码里写的 matrix[i][j]， 意思都是「第 i 行、第 j 列的那个格子」。

题目要求：只要某个格子的值是0，就要把它所在的整行和整列 都变成 0。而且要求原地修改——「原地」的意思是，你不能另外找一张同样大小的 新表格打草稿、算好了再整个搬过去；手里只有这一张表格，答案必须直接写在它自己身上 （最多允许借用几个变量临时记点东西，就像倒水时借用一个小杯子，而不是再买一整个新水桶）。

matrix[[1,1,1],[1,0,1],[1,1,1]]

这是一个 3 行 3 列的表格，每个方括号里的一组数字就是一行。正中间那格 matrix[1][1] = 0（第 1 行第 1 列），所以第 1 行、第 1 列都要整个变成 0， 结果是 [[1,0,1],[0,0,0],[1,0,1]]：中间那一整行变 0，中间那一整列也变 0。

🔑 第二步：为什么不能扫到 0 就立刻清零

第一反应可能是：一边扫矩阵，一边碰到 0 就立刻把它的整行整列都设成 0，扫完不就好了？ 我们拿一个小例子实际走一遍，看看问题出在哪。

matrix[[1,0,1],[1,1,1],[1,1,1]]

这里只有 matrix[0][1] = 0 一个 0。正确答案应该是：第 0 行整行变 0、 第 1 列整列变 0，其余格子都不受影响 → [[0,0,0],[1,0,1],[1,0,1]]。

如果扫到 (0,1) 时立刻把第 0 行、第 1 列清零，矩阵会先变成 [[0,0,0],[1,0,1],[1,0,1]]——到这一步其实还是对的。问题出在接着往后扫： 扫到 (0,2) 这一格时，它现在的值是 0（是刚才被我们清出来的）， 但程序没法区分「这是刚才清出来的 0」还是「这本来就是一个 0」， 于是会把它当成新发现的一个 0，接着又把第 2 列也整个清零—— 可第 2 列本来根本不该受影响！越往后扫，被误判的 0 越多，清零范围就跟滚雪球一样越滚越大， 最终结果完全错误。

所以标准做法是分两步走：第一步只「记账」——先把「哪些行 / 哪些列该清零」 记下来，绝对不能一边记一边动手清零；第二步等全部记完了，再统一执行清零。 这样就不会把「刚清出来的假 0」和「原本真正的 0」搞混了。剩下的问题是：记在哪里？

朴素做法：开两个数组 rows[]（长度 m）和 cols[]（长度 n），每个格子只存「是 / 否」这一种信息—— 这种只有两个取值的类型叫布尔（boolean），就像一个开关， 要么开、要么关，没有第三种状态。rows[i] = true 就表示「第 i 行含 0，要清零」， cols[j] 同理。先扫一遍矩阵把这两个数组填好，再扫一遍照着它们清零。 这样空间复杂度是 O(m + n)，能过，但不是最优解。

更省空间的想法：矩阵的第 0 行正好有 n 个格子， 第 0 列正好有 m 个格子——数量和 rows[] / cols[] 完全对得上！与其额外开两个新数组，不如直接借用矩阵自己的第 0 行来当 cols[] 用，第 0 列来当 rows[] 用：拿 matrix[0][j] 这个格子本身当标记位，记「第 j 列要不要清零」；拿 matrix[i][0] 记「第 i 行要不要清零」。（所谓「标记位」，就是借用一个本来存别的东西的格子， 临时拿它的值当一盏信号灯，用来告诉后面的代码「这里该怎么处理」，用完就不再当原来的用途了。）

唯一要小心的坑：第 0 行 / 第 0 列自己原本是不是也含 0， 这件事会在「打标记」的过程中被覆盖掉——比如第 0 行本来就有一个 0， 后来又拿它当标记位反复改写，就再也分不清「这是原来就有的 0」还是「后来打的标记」了。 解决办法很简单：先用两个布尔变量 row0Zero / col0Zero （同样是只有 true/false 两种取值的开关变量）单独记下第 0 行 / 第 0 列原本是否含 0， 把这个「原始真相」存起来之后，再放心地拿第 0 行 / 第 0 列当标记位随便改， 最后再根据这两个变量决定要不要把第 0 行 / 第 0 列也整个清零。

☕ 第四步：写成 Java 代码

class Solution {
    public void setZeroes(int[][] matrix) {
        int m = matrix.length, n = matrix[0].length;
        boolean row0Zero = false, col0Zero = false;

        // 1) 先单独记录第 0 行 / 第 0 列原本是否含 0（马上要被当标记位覆盖）
        for (int j = 0; j < n; j++) {
            if (matrix[0][j] == 0) { row0Zero = true; break; }
        }
        for (int i = 0; i < m; i++) {
            if (matrix[i][0] == 0) { col0Zero = true; break; }
        }

        // 2) 扫内部格子，用第 0 行 / 第 0 列当标记位：matrix[0][j] 记第 j 列，matrix[i][0] 记第 i 行
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][j] == 0) {
                    matrix[i][0] = 0;
                    matrix[0][j] = 0;
                }
            }
        }

        // 3) 再扫一遍内部格子，标记位是 0 就把这个格子清零
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][0] == 0 || matrix[0][j] == 0) {
                    matrix[i][j] = 0;
                }
            }
        }

        // 4) 最后处理第 0 行 / 第 0 列本身
        if (row0Zero) {
            for (int j = 0; j < n; j++) matrix[0][j] = 0;
        }
        if (col0Zero) {
            for (int i = 0; i < m; i++) matrix[i][0] = 0;
        }
    }
}
时间复杂度 O(m·n)

矩阵被完整扫描了常数遍（检查 + 打标记 + 应用 + 收尾），每个格子访问次数是常数。

空间复杂度 O(1)

不算输入矩阵本身，只用了 row0Zero / col0Zero 两个布尔变量。

🚀 小结

套路：需要「先标记、后统一处理」却又想省掉额外数组时， 想想输入数据本身有没有空间可以借用——这里矩阵的第 0 行/列大小正好和 需要的标记数组对得上。借用之前，记得先把会被覆盖的原始信息单独存好（ row0Zero / col0Zero），这是这类「原地算法」题最容易翻车的地方。
