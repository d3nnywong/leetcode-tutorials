一个字符串里藏着的「最长对称片段」。看完你会明白：与其去检查每一段子串是不是回文， 不如反过来——从每个可能的「对称中心」往外扩，又快又直观。

🧩 第一步：题目到底要什么

「回文」就是正着读和反着读都一样的字符串，比如 "bab"、"aba"、 "bb"。题目给一个字符串 s，要你从里面找出最长的 那一段连续子串，使它本身是回文。

s"babad"

它的子串里 "bab" 和 "aba" 都是回文，长度都是 3， 两个都算正确答案——返回任意一个即可。

🐢 第二步：最直接的笨办法，以及它慢在哪

最容易想到的：把 s 的每一段子串都枚举出来（两层循环定起止 下标），再逐个检查是不是回文（再花一层循环对比首尾字符）。三层循环，时间复杂度 O(n³)，字符串长度到 1000 就要算到十亿次量级，明显跑不动。

慢在哪：我们对「是不是回文」这件事，每一段子串都从头检查一遍， 完全没有利用「短回文能不能拼出长回文」之间的关系。

换个角度想：一个回文串一定是关于某个中心对称的——奇数长度（如 "aba"）的中心是一个字符；偶数长度（如 "bb"）的中心是 两个字符之间的缝。一个长度为 n 的字符串，一共只有 2n − 1 个这样的中心。

于是思路反过来：不去检查子串，而是从每个中心出发往外扩。 用左右两个指针 L、R 从中心位置出发，只要 s[L] === s[R] 就同时向外挪一格，一旦不相等（或者撞到字符串边界）就停下—— 这个中心能扩出的最长回文就确定了。把 2n − 1 个中心都试一遍，记录全程扩出过的 最长结果，就是答案。

⚡ 第三步：中心扩展，一步步看

当前指针 L / R 本次扩展中、匹配成功的区间 比较不相等 目前为止最长的回文

试试把字符串改成 cbbd，看它怎么在第 4 个中心（"b" 和 "b" 之间）扩出 "bb"。

☕ 第四步：写成 Java 代码

class Solution {
    public String longestPalindrome(String s) {
        if (s == null || s.length() < 1) return "";
        int start = 0, end = 0; // 最长回文子串的左右下标（闭区间）
        for (int i = 0; i < s.length(); i++) {
            int len1 = expand(s, i, i);     // 以 i 为中心（奇数长度）
            int len2 = expand(s, i, i + 1); // 以 i, i+1 为中心（偶数长度）
            int len = Math.max(len1, len2);
            if (len > end - start + 1) {
                start = i - (len - 1) / 2;
                end = i + len / 2;
            }
        }
        return s.substring(start, end + 1);
    }

    // 以 left、right 为中心向两边扩展，返回能扩出的回文长度
    private int expand(String s, int left, int right) {
        while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
            left--;
            right++;
        }
        return right - left - 1; // 循环多减了一层，要补回来
    }
}
时间复杂度 O(n²)

2n − 1 个中心，每个中心最坏情况要向外扩 O(n) 步。

空间复杂度 O(1)

只用了几个指针变量，没有额外的数组或表格。

🚀 小结

套路：遇到「回文」相关的题，先想到它的对称性—— 一个回文串总能由更短的回文从中心往外扩得到。中心扩展法把「枚举所有子串再逐个验证」的 O(n³) 降到了 O(n²)，且只用常数额外空间，是这类问题里最朴素也最常用的入门解法。 （如果还想更快，这道题还有 O(n²) 空间的区间 DP 解法，以及更复杂的 O(n) Manacher 算法，可以作为进阶了解。）
