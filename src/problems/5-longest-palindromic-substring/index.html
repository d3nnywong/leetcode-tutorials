一个字符串里藏着的「最长对称片段」。看完你会明白：与其去检查每一段子串是不是回文， 不如反过来——从每个可能的「对称中心」往外扩，又快又直观。

🧩 第一步：题目到底要什么

「回文」就是正着读和反着读都一样的字符串，比如 "bab"、"aba"、 "bb"。题目给一个字符串 s，要你从里面找出最长的 那一段连续子串，使它本身是回文。

s"babad"

它的子串里 "bab" 和 "aba" 都是回文，长度都是 3， 两个都算正确答案——返回任意一个即可。

🐢 第二步：最直接的笨办法，以及它慢在哪

最容易想到的：用两层循环——外层挑一个起点下标，内层挑一个 终点下标（「下标」就是字符在字符串里的位置编号，从 0 开始数）， 把 s 的每一段子串都枚举出来（枚举，就是 一个不漏、一个不重地把所有可能情况都列一遍）；再对每一段子串用第三层循环，从两头往 中间逐个字符比对，检查它是不是回文。三层循环叠在一起，计算量大致跟 n × n × n 成正比，这就是所谓的时间复杂度 O(n³)—— 字符串长度到 1000，就要算到十亿次量级，明显跑不动。

慢在哪：我们对「是不是回文」这件事，每一段子串都从头检查一遍， 完全没有利用「短回文能不能拼出长回文」之间的关系。

换个角度想：一个回文串一定是关于某个中心对称的——就像照镜子一样， 从中心往左看和往右看，字符是一一对应、完全相同的。奇数长度（如 "aba"）的中心是一个字符（这个字符自己就站在「镜子」的位置上）； 偶数长度（如 "bb"）没有正中间的字符，中心就落在两个字符之间的缝里。 一个长度为 n 的字符串，逐个字符数是 n 个中心，字符之间的缝还有 n − 1 个， 加起来一共只有 2n − 1 个这样的中心。

于是思路反过来：不去检查子串，而是从每个中心出发往外扩——就像往平静的 水面丢一颗石子，波纹一圈圈往外扩散。具体做法：设两个「指针」L、 R（指针在这里没什么神秘的，就是两个会不断挪动的位置标记，一左一右）， 从中心位置出发。只要 L、R 这两个位置上的字符相同，就把 L 往左挪一格、R 往右挪一格，继续往外比对；一旦碰到不相同的字符， 或者已经挪到字符串的最前面/最后面、没地方可挪了，就停下来——这时候，这个中心能扩出的 最长回文就确定了。把 2n − 1 个中心都这样试一遍，记录全程扩出过的最长结果， 就是答案。

当前指针 L / R 本次扩展中、匹配成功的区间 比较不相等 目前为止最长的回文

☕ 第四步：写成 Java 代码

把上面的思路翻译成代码。核心是一个 expand 函数：给它一对起始位置 left、right，它就沿着这两个位置向两边扩展，直到扩不动为止， 返回这次扩出来的回文长度。主函数只需要对每个中心分别调用它一次——奇数中心传 (i, i)（左右指针从同一个字符出发），偶数中心传 (i, i + 1) （左右指针从相邻两个字符出发）——记录长度最大的那一次结果就行。 

class Solution {
    public String longestPalindrome(String s) {
        if (s == null || s.length() < 1) return "";
        int start = 0, end = 0; // 最长回文子串的左右下标（闭区间）
        for (int i = 0; i < s.length(); i++) {
            int len1 = expand(s, i, i);     // 以 i 为中心（奇数长度）
            int len2 = expand(s, i, i + 1); // 以 i, i+1 为中心（偶数长度）
            int len = Math.max(len1, len2);
            if (len > end - start + 1) {
                start = i - (len - 1) / 2;
                end = i + len / 2;
            }
        }
        return s.substring(start, end + 1);
    }

    // 以 left、right 为中心向两边扩展，返回能扩出的回文长度
    private int expand(String s, int left, int right) {
        while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
            left--;
            right++;
        }
        return right - left - 1; // 循环多减了一层，要补回来
    }
}
时间复杂度 O(n²)

2n − 1 个中心，每个中心最坏情况要向外扩 O(n) 步。

空间复杂度 O(1)

只用了几个指针变量，没有额外的数组或表格。

🚀 小结

套路：遇到「回文」相关的题，先想到它的对称性—— 一个回文串总能由更短的回文从中心往外扩得到。中心扩展法把「枚举所有子串再逐个验证」的 O(n³) 降到了 O(n²)，且只用常数额外空间，是这类问题里最朴素也最常用的入门解法。 （如果还想更快，这道题还有一种基于动态规划（DP，简单说就是把大问题拆成一个个 小问题，把每个小问题的答案记在一张表里，避免重复计算）的区间 DP 解法，只需要 O(n²) 空间；还有更复杂、专门为回文问题设计的 O(n) Manacher 算法。这两种都可以留到进阶 阶段再深入了解——现在这套中心扩展法，已经足够通过本题，也足够帮你建立起对「回文」问题的直觉。）
