「打家劫舍」的进阶版：房子从一条直线变成了一个圈。看完你会发现， 多出来的「首尾相邻」并不可怕——只要把一个环拆成两条直线， 就能把这道题变回一个经典套路：动态规划 （简单说就是把一个大问题拆成一串小问题，每个小问题的答案只算一次、记下来， 后面要用就直接翻出来，不用重新算一遍）。就算你没做过前一题， 下面也会从头带你推一遍，不需要任何编程基础。

🧩 第一步：题目到底要什么

还是那个小偷，还是「相邻的房子不能同时偷」的规则。唯一的变化：这次所有房子 围成一圈，第一间和最后一间也算是「相邻」的。问能偷到的 最高金额是多少。

nums[2, 3, 2]

不能同时偷 1 号和 3 号（首尾相邻），所以只能二选一：偷 1 号得 2，偷 2 号得 3， 最优是只偷 2 号 → 返回 3。

🐢 第二步：把环拆成两条直线

先想象房子还是排成一条直线（这正是上一题「打家劫舍」原题的样子）。 这类问题的标准解法就是前面提到的动态规划：从左到右一间一间地算， 把「只考虑到第 i 间为止，最多能偷到多少钱」记在一张小表里， 记作 dp[i]（dp 是「动态规划」英文 Dynamic Programming 的缩写，dp[i] 就读作「到第 i 间为止的最优答案」）。 它的算法长这样：dp[i] = max(dp[i-1], dp[i-2] + nums[i])—— 第 i 间要么不偷（沿用上一间算出的最优解 dp[i-1]）， 要么偷（那紧邻的上一间就必须跳过，只能在「上上间的最优解」 dp[i-2] 基础上加上这一间的钱）。

现在题目多了一条规则：第 1 号房和第 n 号房也算相邻 （因为房子围成一圈，头和尾正好挨在一起）。唯一的麻烦就是 这两间不能同时偷，直接套用上面的直线公式会出错。

但反过来想：只要能保证「第 1 号与第 n 号不同时被偷」，剩下的就还是一条普通的直线问题。 而「不同时偷」只有两种互斥的可能（互斥就是这两种情况不会同时发生， 只能二选一）：

  • 情形 A：彻底不偷第 n 号房 —— 问题退化成在 [1, n-1] 这条直线上做标准打家劫舍。
  • 情形 B：彻底不偷第 1 号房 —— 问题退化成在 [2, n] 这条直线上做标准打家劫舍。

这两种情形覆盖了所有不违反「首尾不同时偷」的方案，互不重叠也没有遗漏， 所以答案就是两条直线各自跑一遍一维 DP，再取较大值。

为什么不会漏掉「两个都不偷」的情况：「两个都不偷」属于情形 A （不偷第 n 号）能覆盖到的一种可能——情形 A 的 DP 本来就会在「偷第 1 号更划算」和 「不偷第 1 号更划算」之间自动取较大者，所以不需要单独再分第三种情形。

🧠 第三步：动态规划三件套

1

状态：dp[i] 表示什么？

在「拆出来的某一条直线」上，dp[i] = 偷到第 i 间房（含第 i 间）为止，能拿到的最高金额。

两条直线（情形 A、情形 B）各自维护一份 dp，互不影响。

2

转移：dp[i] 怎么由更小的问题推出来？

站在第 i 间房，要么偷它，要么不偷：

dp[i] = max(dp[i-1], dp[i-2] + nums[i])

不偷第 i 间就沿用 dp[i-1]；偷第 i 间就必须跳过紧邻的第 i-1 间，金额是 dp[i-2] + nums[i]。

3

边界：直线的起点怎么算？

每条直线的第一间房只能直接偷：dp[第一间] = nums[第一间]； 第二间房不能和第一间同时偷，取较大者：dp[第二间] = max(nums[第一间], nums[第二间])。

下面的动画会先点出「首尾相邻」的矛盾，再依次演示情形 A、情形 B 两条直线各自的一维 DP： 被排除的那间房整间变暗，当前在算的房子高亮，它依赖的「前一间」「前两间」标成来源色， 最后两个总额取较大者，就是答案。

本情形不偷这一间 当前正在算的 dp[i] 依赖的 dp[i-1] 与 dp[i-2] 最终答案所在的情形

试试把输入改成 2,3,2（官方示例 1），看两条直线打成平手； 再试试只填 1 个数，看「只有一间房」时是怎么直接跳过首尾矛盾的。

☕ 第四步：写成 Java 代码

每条直线上的一维 DP，算 dp[i] 时只会用到最近两个值 dp[i-1] 和 dp[i-2]，再往前的值就再也用不上了。 所以没必要把整张 dp 表都存下来，只留两个变量， 每算完一步就往前挪一格、覆盖旧值（这种写法叫「滚动变量」， 就像滚轮一样只保留最近的一小段），空间就能从 O(n) 压到 O(1)。 

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 1) return nums[0]; // 只有一间房，没有首尾相邻的问题

        // 情形 A：不偷最后一间，在 [0, n-2] 上跑标准打家劫舍
        // 情形 B：不偷第一间，在 [1, n-1] 上跑标准打家劫舍
        return Math.max(robLinear(nums, 0, n - 2), robLinear(nums, 1, n - 1));
    }

    // 在闭区间 [lo, hi] 这条直线上做标准的「打家劫舍」一维 DP（滚动变量，O(1) 空间）
    private int robLinear(int[] nums, int lo, int hi) {
        int prev2 = 0; // 滚动变量：dp[i-2]
        int prev1 = 0; // 滚动变量：dp[i-1]
        for (int i = lo; i <= hi; i++) {
            int cur = Math.max(prev1, prev2 + nums[i]); // 不偷 i / 偷 i
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
}
时间复杂度 O(n)

两条直线各扫一遍，整体仍是线性。

空间复杂度 O(1)

每条直线只用两个滚动变量，不需要整张 dp 表。

🚀 小结

套路：遇到「环形」的限制时，先别急着发明新算法——想想能不能 去掉那条让问题变复杂的额外限制，把问题拆回你已经会做的「直线」版本。 打家劫舍 II 的核心不是全新的 DP 状态，而是「拆环为直线」这个通用技巧： 只要环上有「两个位置不能同时选」这样的限制，就可以分别假设「先不选其中一个」， 跑两次直线版的 DP，再取较大值。这个技巧不只适用于打家劫舍， 很多「首尾相连」的环形排列问题，都能用同样的思路把环拆开、变回熟悉的直线问题。
