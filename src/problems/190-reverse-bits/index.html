一个整数的 32 个二进制位，怎么「整体左右翻个个儿」？ 看完你会发现：这其实就是一道披着位运算外衣的「数组首尾对调」题。

🧩 第一步：题目到底要什么

先弄清楚「二进制」是什么：我们平时数数用的是十进制，逢十进一， 每一位是 0~9 这十个符号；计算机内部只认两个符号：0 和 1，逢二进一， 这就是二进制。一个「32 位整数」说白了就是用 32 个格子拼出来的 一长串 0/1，每个格子叫一「位」（英文 bit）。

这些格子有个编号习惯：从最右边数起，最右边那一位叫 bit0（第 0 位，也叫「最低位」），往左依次是 bit1、 bit2……一直数到最左边的 bit31（「最高位」）。 后面我们会一直用这套编号来描述「第几位」。

题目要做的事：给你一个 32 位整数 n，把它的二进制表示 整体倒过来：原来排在最左边（bit31，最高位）的那一位， 挪到最右边（bit0）；原来第 2 位的，挪到倒数第 2 位……以此类推， 最后把这串翻转后的二进制再当成一个新的整数返回。

n43261596
二进制00000010100101000001111010011100
翻转后00111001011110000010100101000000

翻转后的二进制再读成十进制，就是 964176192。

💡 第二步：换个角度看「位运算」

先说说标题里的「位运算」是什么：它听起来吓人，其实就是不把这 32 个 0/1 当成一个整体的数字去加减，而是直接一位一位地摆弄它们。最常用的几种操作是： 左移 <<（所有位整体往左挪，右边空出来的位补 0）、 右移（往右挪）、按位与 &（两边都是 1，结果才是 1，常用来 「单独取出」某一位）、按位或 |（只要有一边是 1，结果就是 1，常用来 把某一位「点亮」塞进去）。这道题从头到尾只会用到这几种操作。

32 个二进制位排成一排，「整体翻转」和「翻转一个长度为 32 的数组」完全是同一件事—— 只不过数组里装的是 0/1，而且我们最后要把翻转结果重新读成一个整数。

翻转规律很简单：原数第 i 位（从最低位 bit0 数起）， 翻转之后会变成新数第 (31 - i) 位。 也就是说 bit0 和 bit31 对调、bit1 和 bit30 对调……像数组的双指针那样，从两端往中间逐位「对调」。

代码里怎么实现：不用真的开一个 32 格的数组，而是用两个变量配合 位运算，模拟出「两头往中间搬」的效果，每次循环做三件事：
  • result <<= 1：把 result 已经拼出来的部分 整体左移一位——相当于在最右边「腾出一个空位」，准备塞新的一位进去；
  • result |= (n & 1)：n & 1 是「按位与」， 效果是把 n 除了最右边一位以外全部清零，只留下最右边那一位； 再用 |=「按位或」把这一位塞进 result 刚腾出来的空位；
  • n >>>= 1：把 n 整体右移一位，丢掉刚刚已经 用过的最低位，让下一位挪到最右边等着被取出来。
这里要特别留意用的是 >>>（无符号右移）， 而不是普通的 >>。原因是 Java 的 int 是「有符号数」，如果最高位（bit31）恰好是 1，普通右移会在 左边自动补 1（这是为了保住负数的符号），这样就会凭空多出几个不该有的 1， 翻转结果就错了；无符号右移则老老实实在左边补 0，才是我们真正想要的「按位搬运」。 循环 32 次，n 的每一位都被取出来用过一次—— 第 i 次循环塞进去的那一位，后面还会被再左移 (31-i) 次， 所以它最终正好停在第 (31-i) 位，和上面的翻转规律完全对应。

下面把这个翻转过程逐位演示出来：上排是原数 n 的 32 位（从左到右 bit31 → bit0），下排是正在拼出来的 result。 每一步从原数第 i 位取一位，直接搬到结果的 第 (31-i) 位——上排的「读取头」从右往左走， 下排的「写入头」从左往右走，两边同步推进，正好对称。

☕ 第四步：写成 Java 代码

class Solution {
    public int reverseBits(int n) {
        int result = 0;
        for (int i = 0; i < 32; i++) {
            result <<= 1;          // 给 result 腾出最低位
            result |= (n & 1);     // 把 n 当前最低位塞进去
            n >>>= 1;              // 无符号右移，丢掉刚处理过的那一位
        }
        return result;
    }
}
时间复杂度 O(1)

固定循环 32 次，与输入的数值大小无关。

空间复杂度 O(1)

只用了几个整型变量，不需要额外数组。

🚀 小结

套路：遇到「按位翻转 / 按位重排」类题目，先把二进制位想象成一个 普通数组，套用数组翻转（首尾对调、双指针）的直觉；再回到位运算，用 <<、>>>、&、| 把「数组操作」翻译成「移位 + 位或」即可。如果同一个数要反复调用这个函数， 还可以按 8 位一组做查表法——提前把 0~255 这 256 种可能的 翻转结果算好存进一张表，要用时直接查表拿结果，不用现场逐位算， 这样能把 32 次循环压缩成 4 次查表，这正是题目「进阶」想引导你想到的优化方向。
