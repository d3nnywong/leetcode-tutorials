一个整数的 32 个二进制位，怎么「整体左右翻个个儿」？ 看完你会发现：这其实就是一道披着位运算外衣的「数组首尾对调」题。

🧩 第一步：题目到底要什么

给你一个 32 位整数 n，把它的二进制表示整体倒过来： 原来排在最左边（最高位）的那一位，挪到最右边（最低位）； 原来排第 2 位的，挪到倒数第 2 位……以此类推，最后把这串翻转后的二进制 再当成一个新的整数返回。

n43261596
二进制00000010100101000001111010011100
翻转后00111001011110000010100101000000

翻转后的二进制再读成十进制，就是 964176192。

💡 第二步：换个角度看「位运算」

32 个二进制位排成一排，「整体翻转」和「翻转一个长度为 32 的数组」完全是同一件事—— 只不过数组里装的是 0/1，而且我们最后要把翻转结果重新读成一个整数。

翻转规律很简单：原数第 i 位（从最低位 bit0 数起）， 翻转之后会变成新数第 (31 - i) 位。 也就是说 bit0 和 bit31 对调、bit1 和 bit30 对调……像数组的双指针那样，从两端往中间逐位「对调」。

代码里怎么实现：不用真的开一个 32 格的数组。 每次循环把 result 整体左移一位腾出最低位，再把 n 当前的最低位塞进去，然后把 n 右移一位丢掉这一位、留出下一位。 循环 32 次，n 的每一位都被取出来用过一次—— 第 i 次循环塞进去的那一位，后面还会被再左移 (31-i) 次， 所以它最终正好停在第 (31-i) 位，和上面的翻转规律完全对应。

⚡ 第三步：逐位搬运动画

下面把这个翻转过程逐位演示出来：上排是原数 n 的 32 位（从左到右 bit31 → bit0），下排是正在拼出来的 result。 每一步从原数第 i 位取一位，直接搬到结果的 第 (31-i) 位——上排的「读取头」从右往左走， 下排的「写入头」从左往右走，两边同步推进，正好对称。

☕ 第四步：写成 Java 代码

class Solution {
    public int reverseBits(int n) {
        int result = 0;
        for (int i = 0; i < 32; i++) {
            result <<= 1;          // 给 result 腾出最低位
            result |= (n & 1);     // 把 n 当前最低位塞进去
            n >>>= 1;              // 无符号右移，丢掉刚处理过的那一位
        }
        return result;
    }
}
时间复杂度 O(1)

固定循环 32 次，与输入的数值大小无关。

空间复杂度 O(1)

只用了几个整型变量，不需要额外数组。

🚀 小结

套路：遇到「按位翻转 / 按位重排」类题目，先把二进制位想象成一个 普通数组，套用数组翻转（首尾对调、双指针）的直觉；再回到位运算，用 <<、>>>、&、| 把「数组操作」翻译成「移位 + 位或」即可。如果同一个数要反复调用这个函数， 还可以按 8 位一组做查表法，把 32 次循环压缩成 4 次查表， 这正是题目「进阶」想引导你想到的优化方向。
